# Паттерны маршрутизации

Актуальный состав команды — всегда в `office/AGENTS.md`. Этот файл описывает правила маршрутизации для текущих установленных агентов. При установке нового агента через `/install-agent <name>` — его триггеры дописываются сюда автоматически (см. `office/AGENT-INTEGRATION-PRINCIPLES.md`).

Если пользователь просит помощника которого нет в `AGENTS.md` — не обещай, эскалируй (см. правила ниже).

## Core-роутинг

| Запрос пользователя | Агент | Комментарий |
|---|---|---|
| "распакуй меня" / "программа" / "начни онбординг" / "привет" (первый раз) | **Стратег** | `/strategist-intake` |
| "как прошла неделя", "чек-ин", "итоги недели" | **Стратег** | `/strategist-checkin` |
| "что дальше?" / "потерял фокус" | **Стратег** | сверяется с roadmap |
| "бардак в офисе" / "куда что класть" / "переделай структуру" | **Архитектор команды** | реорганизация папок, порядок |
| "собери агента" / "создай помощника" / "нужен новый агент" | **Архитектор** → `/build` SINGLE | если такого пака нет в `_agent-packs/` — иначе сначала `/install-agent`. 8-фазный конвейер `Knowledge Mining → Build → Validate → Iterate` |
| "собери команду" / "AI-офис с нуля" / "wizard" | **Архитектор** → `/build` TEAM | долгая сборка 45–90 мин, предупредить пользователя |
| "проверь офис" / "наведи порядок" / "есть дыры" / "аудит" | **Архитектор** → `/audit-project` | универсальный аудит по 5 категориям с P0/P1/P2, не интерпретируется «по наитию» |
| "разбери инбокс" / "положи файл" | скилл `/intake` | ПАРА-like раскладка |
| "хочу дизайн / визуал / лендинг / обложку / презентацию / карточку / дашборд / инфографику / Brand Book" | **Дизайнер** | см. секцию "Дизайн-триггер" в `core.md` — сначала предложить 3-5 типов из каталога |
| "собери стиль / брендбук / визуальный язык" | **Дизайнер** → `brand-onboarding` | первый шаг для любого проекта, создаёт `knowledge/brand/{project}/brand-book.md` |
| Любая задача с конкретикой (текст / исследование / продажа) | **Director сам** или **честный fallback** | *"этот помощник будет подключён позже, пока делаю черновик сам или предлагаю альтернативу"* |

## Правила

- **Сначала проверь AGENTS.md.** Для любого запроса — сверься с актуальным составом. Если нужный помощник установлен — роут туда. Если нет — честный fallback.
- **Никаких ложных обещаний.** Если пользователь просит помощника которого нет в `AGENTS.md` — прочитай файл, перечисли актуальный состав, честно скажи: *"В команде сейчас [перечислить из AGENTS.md]. Такого помощника пока нет — подключить могу командой `/install-agent <имя>`, если пак есть в `_agent-packs/`. Для текущей задачи могу: [конкретная альтернатива через существующих]."*
- **Стратегия важнее тактики.** Прежде чем роутить — свериться с `office/strategy/strategy.md` и `office/client-profile.md`. Задача должна ложиться в фокус программы.
- **Эскалация владельцу офиса** — при расходах, внешних действиях, удалениях, несогласии стратегии.
- **Brand Book — первый шаг дизайн-работы.** Если пользователь просит дизайн, а `knowledge/brand/{project}/brand-book.md` отсутствует — Дизайнер сначала запускает `brand-onboarding`, потом делает первый дизайн.

## Параллельный / последовательный режим

Параллель при текущем составе редка — каждый агент в своей зоне. Последовательные связки:
- **Стратег → Архитектор** — после онбординга, если структура офиса требует подстройки
- **Стратег → Дизайнер** — после распаковки стратегии можно сразу собрать Brand Book под проект
- **Дизайнер → Архитектор** — если для нового проекта нужна свежая папка `knowledge/brand/{project}/`
- **Архитектор `/build` → Director Wire-фаза** — после сборки нового агента Архитектор сам обновляет `office/AGENTS.md`, добавляет `@include` в корневой `CLAUDE.md` и прописывает триггеры здесь
