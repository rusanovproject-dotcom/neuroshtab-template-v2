# Экономия контекстного окна

## Цена типичного CLAUDE.md

Средний CLAUDE.md (400 строк) ≈ 1200-1500 токенов.
Повторяется в КАЖДОМ сообщении сессии.
20 сообщений × 1500 = 30,000 токенов только на контекст.

Если CLAUDE.md 800 строк → +30,000 токенов за сессию впустую.

## Техника 1: Lean CLAUDE.md

Target: ≤300 строк, ≤500 токенов.

Удалить:
- Очевидные вещи (Claude уже знает TypeScript)
- Дубли правил (одно и то же разными словами)
- Примеры, если менее 3 реальных кейсов
- Описания процессов других агентов

Проверка: если Claude уже делает правильно без правила — удали правило.

## Техника 2: Progressive Disclosure через Skills

Было: всё в CLAUDE.md (800 строк)
Стало: core CLAUDE.md (200 строк) + skills/ (по запросу)
Экономия: ~15,000 токенов за сессию.

Правило: если инструкция нужна <30% задач — выноси в skill.

## Техника 3: Handle Pattern

Вместо содержимого файла — путь + описание в 1 строку:
```
- `knowledge/brand/positioning.md` — позиционирование продукта
```
Агент загрузит ТОЛЬКО когда задача потребует.

## Техника 4: Summary Pattern

Для файлов >200 строк: summary-версия ≤50 строк.
```
context.md     ← 50 строк (читается всегда)
state.md       ← 500 строк (читается по запросу)
```

## Техника 5: .claudeignore

```
node_modules/
.venv/
dist/
build/
*.lock
*.min.js
*.map
coverage/
*.png
*.jpg
```
Экономия: до 40% токенов на типичном проекте.

## Техника 6: Specialist Pattern

Каждый агент видит ТОЛЬКО свои файлы:
```
CTO:  reads projects/*, ops/log.md
      ignores knowledge/brand/
CMO:  reads knowledge/brand/, knowledge/audience/
      ignores projects/*/src/
```

## Техника 7: Роутинг моделей (из стрима)

- Батч-обработка (50 писем) → дешёвая модель (Qwen/Mistral)
- Оркестрация/reasoning → дорогая модель (Opus)
- Экономия: 30-40% бюджета

## Техника 8: Truncate в тулах

Подключи трейсинг (LangFuse) → найди тулы, жрущие контекст →
добавь truncate. Валера так починил жор токенов в OpenClaw.

## Когда начинать новую сессию

Правило: при 60% контекстного окна (не 90%) → /clear → новая сессия.
Загрузи ТОЛЬКО: context.md + task brief + релевантные файлы.
