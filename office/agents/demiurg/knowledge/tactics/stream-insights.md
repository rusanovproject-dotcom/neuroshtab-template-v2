## Ключевые парадигмы

### Environment Engineering (новая эра)
Раньше: prompt-engineering → context-engineering → **environment-engineering**
Теперь мы создаём окружение для LLM и правильно подаём бизнес-контекст.
> "Мы с вами теперь environment-инженеры" — Валера

### Skills-First Architecture
Бизнес-логику больше не пишут в коде фреймворка. Всё выносится на уровень скиллов.
Ядро агента = чёрный ящик. Ты расширяешь только harness-слои под свой бизнес.
> "Бизнес-логику тащи на уровень скиллов — тогда расшивается вся парадигма" — Леша

### Вайп-кодинг скиллов для всех
Маркетеры, аналитики, нетехнические спецы будут вайп-кодить скиллы.
Это как вайп-кодинг на Cursor, но для бизнес-автоматизаций.

| Ядро | Для чего | Плюсы | Минусы |
|------|----------|-------|--------|
| **Claude Code CLI** | Код, sandbox, sub-агенты | Лучший harness, MCP, скиллы | Vendor lock |
| **Codex CLI** | Код, sandbox | Открытый, хороший sandbox | Новый, мало экосистемы |
| **OpenClaw** | Всё-в-одном ассистент | Полный функционал | Жрёт токены (100$/ночь) |
| **ZeroClaw** | Лёгкий старт | Можно форкнуть, 4000 строк | Базовый функционал |
| **NanoClaw** | Минимальное ядро | Самый простой | Ограничен |

> "Не выходи за рамки этих пяти штучек" — Леша

**Паттерн: форк + расширение.** Не пиши ядро с нуля. Возьми ZeroClaw,
дай Claude его код и скажи: "сделай мне такое же ядро, но с моими тулами".
Ядро = ~4000 строк TypeScript/Go. Всё остальное — скиллы.

## Модели: роутинг по задачам

### Правило: мощная модель прощает ошибки
> "Чем лучше модель, тем больше тебе простят несовершенство фреймворка" — Леша

OpenClaw работает потому что Opus умный. На дешёвой модели он бы не работал.
GPT-5.4 сам починит сломанный скрипт внутри скилла — дешёвые не смогут.

### Практический роутинг

| Тип задачи | Модель | Почему |
|------------|--------|--------|
| Оркестрация, reasoning | Opus 4.6 / GPT-5 | Emergent reasoning |
| Батч (50 писем, парсинг) | Qwen/Mistral дёшево | Скорость + экономия |
| Tool-calling | Проверенный провайдер | OpenRouter кидает между 20 провайдерами! |
| Summary/фильтр | Любая | Не требует reasoning |

### Ловушка OpenRouter
Каждый запрос идёт к разным провайдерам, с разной квантизацией модели.
Tool-calls ломаются, потому что у одного провайдера правильный chat-template,
а у другого — нет. **Решение:** запросить конкретного провайдера в конфиге.

## Экономия токенов

1. **Батч на дешёвом, решения на дорогом.** 50 писем → выгрузи через Llama →
   передай summary в Opus для решений.

2. **Специализированные агенты.** Не один жирный агент на всё —
   лайт-агент (Qwen) для рутины + heavy (Opus) для reasoning.

3. **Транкейт в тулах.** Подключи трейсинг (LangFuse) → увидишь какие тулы
   жрут контекст → добавь truncate. Валера так починил OpenClaw.

4. **Не доверяй OpenRouter tool-calls** — скрытый множитель расходов.

## Отладка агентов

### "Учись читать логи" (из Яндекса)
Логи агента = единственный trust-source. Каждая строка кода должна логировать.

### Цикл отладки (из OpenAI Harness статьи)
```
Агент → выполняет → пишет логи/трейсы → ты читаешь →
→ видишь ошибку → даёшь фикс-промт → деплой → проверка спанов → повтор
```

Когда агент видит своё окружение (логи, спаны, метрики) — он может
себя отлаживать в цикле без человека.

### Слои инструкций (Harness Pattern)
```
System instruction (ядро, не трогай)
  → Environment layer (API, БД, логи, метрики)
    → Skills layer (бизнес-логика, тулы)
      → Safety layer (guardrails, ограничения)
```

## Итеративное улучшение скиллов

Паттерн от Леши:
1. Напиши первую версию скилла (кривую)
2. Агент идёт делать задачу, набивает шишки
3. Скажи: "пробеги по своей истории, посмотри что было неоптимально"
4. Агент предлагает улучшения
5. Обнови скилл → следующая итерация лучше

**Паттерн от Валеры:** дай агенту ссылку на статью про архитектуру скиллов
и скажи: "напиши скилл, проверь что использованы все паттерны из статьи".

---

## Паттерн: Memory через скиллы

Когда агент обучается новому действию (например, парсит маркетплейс):
1. Агент тыкается, находит путь
2. Скажи: "напиши список инструкций для себя на будущее"
3. Он генерит step-by-step
4. Положи в CLAUDE.md или отдельный скилл
5. В следующий раз он уже знает маршрут

> Это самообучающийся паттерн через файловую систему.

## Красные флаги

- ❌ Писать своё ядро с нуля (форкни готовое)
- ❌ Дешёвая модель для reasoning (Opus/GPT-5 прощают ошибки)
- ❌ OpenRouter без конкретного провайдера (tool-calls ломаются)
- ❌ Нет логирования (летишь вслепую)
- ❌ Граф-RAG на день 1 ("а вы хоть что-то построили?" — Валера)
- ❌ Бизнес-логика в коде ядра (выноси в скиллы)
- ❌ Один агент на всё (разделяй: дешёвый для рутины + дорогой для решений)
