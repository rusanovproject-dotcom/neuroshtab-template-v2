# Протокол передачи задач между агентами

Как агенты передают задачи друг другу. Единый формат для всей команды.

---

## 1. Task Brief — формат передачи задачи

Один агент передаёт другому ровно 4 поля:

```
Задача: {что сделать — 1 предложение}
Контекст: {какие файлы прочитать, что уже сделано}
Результат: {формат, объём, куда положить}
Приоритет: {сегодня / на неделе / когда будет время}
```

**Пример — Director передаёт CMO:**
```
Задача: Напиши пост для Telegram про запуск мастер-группы
Контекст: knowledge/brand/positioning.md, knowledge/audience/avatars.md
Результат: текст 150-200 слов, хук + история + CTA, файл в drafts/
Приоритет: сегодня
```

**Пример — Director передаёт CTO:**
```
Задача: Добавь эндпоинт /api/schedule в <your-app>
Контекст: <your-app>/CLAUDE.md, Issue #42 (там спека)
Результат: рабочий эндпоинт + тест, Draft PR в ветке feat/schedule
Приоритет: на неделе
```

**Правило:** нет Task Brief — нет задачи. Устные "сделай что-нибудь" не работают.

---

## 2. Протокол передачи (3 шага)

1. **Director определяет получателя** — по роутинг-таблице в AGENTS.md.
   Если задача на стыке двух агентов — Director решает кто ведущий.

2. **Director формирует Task Brief** — заполняет 4 поля выше.
   Контекст = конкретные файлы (Handle-паттерн), не "посмотри сам".

3. **Получатель выполняет** — читает Task Brief, загружает контекст
   по Progressive Disclosure (только нужные файлы), делает, возвращает результат.

---

## 3. Формат результата

Агент возвращает после выполнения:

```
Статус: готово | нужна доработка | заблокировано
Результат: {ссылка на файл / текст / PR}
Что сделал: {1-2 строки}
```

**Пример — CMO возвращает Director'у:**
```
Статус: готово
Результат: drafts/tg-mastergroup-launch.md
Что сделал: пост 180 слов, хук через боль ЦА, CTA на запись
```

**Пример — заблокировано:**
```
Статус: заблокировано
Результат: —
Что сделал: нужен файл knowledge/audience/avatars.md, но его нет
```

---

## 4. Эскалация

Когда агент НЕ МОЖЕТ выполнить задачу — не молчит, а эскалирует.

**Правило:** 2 попытки → эскалация наверх.

**Формат эскалации:**
```
Не могу: {причина}
Пробовал: {что делал}
Варианты: {1, 2}
Рекомендую: {N}
```

**Пример:**
```
Не могу: нет доступа к API Telegram для проверки статистики
Пробовал: искал токен в .env, спросил у CTO
Варианты: 1) пользователь даст токен, 2) работаем без статистики
Рекомендую: 1
```

---

## 5. Cross-agent — задача для нескольких агентов

Когда задача требует работы 2+ агентов:

1. **Director разбивает** задачу на подзадачи — каждому свой Task Brief
2. **Каждый работает** в своей зоне ответственности
3. **Director собирает** результаты и синтезирует финальный ответ

**Пример — запуск нового продукта:**
```
CMO:  Task Brief → текст лендинга + воронка
CTO:  Task Brief → страница + форма оплаты
Head of Program: Task Brief → структура программы
Director: собрал всё → проверил → передал пользователю
```

**Правило:** агенты не передают задачи друг другу напрямую.
Всё через Director. Это точка координации и единый источник правды.
