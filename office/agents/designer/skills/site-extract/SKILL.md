# site-extract — извлечение дизайна с сайта

## Когда запускать
- Пользователь присылает ссылку на сайт и хочет «сделай в таком стиле»
- Нужно быстро получить цвета/шрифты любого сайта для референса
- Создать Brand Book на основе существующего бренда пользователя (его же сайт)

## Что делает
Использует CLI-инструмент **dembrandt** (github.com/dembrandt/dembrandt, MIT) — одной командой извлекает дизайн-токены сайта в DESIGN.md.

## Шаги

1. Получи URL сайта от пользователя
2. Проверь есть ли Node.js 18+: `node --version`
   - Если нет → скажи пользователю: *«Нужен Node.js 18+, установи с nodejs.org и попробуй снова. Или пришли скриншоты — разберу по ним.»*
3. Создай папку `knowledge/brand/{project}/references/` если нет
4. Запусти: `npx dembrandt@latest <url> --design-md --save-output knowledge/brand/{project}/references/{домен}.md`
5. Прочитай полученный DESIGN.md
6. Вытащи: палитру (hex), шрифты, отступы, логотип URL
7. Положи суммари в Brand Book пользователя (через brand-revise если уже есть, или в brand-onboarding если собираем)

## Output
Пользователю: *«Извлёк стиль с [домен]. Цвета: [3-5 hex]. Шрифты: [пара]. Хочешь взять всё как есть для своего Brand Book — или подкрутим?»*

## Fallback
Если CLI не работает / нет Node.js → попроси скриншот сайта, опиши стиль глазами, используй как референс без автоматики.
