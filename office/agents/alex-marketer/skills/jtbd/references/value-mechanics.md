## 1. Формула ценности

**Job Solution Value = Job Solution Benefit − Job Solution Cost**

- **Benefit** = насколько хорошо решение выполняет работу (функционально + эмоционально)
- **Cost** = инвестиции клиента (деньги + время + энергия + когнитивные усилия + эмоциональные затраты)

Мозг при покупке: показ оффера → удовольствие и предвкушение. Показ цены → инсула (боль). Решение → medial prefrontal cortex взвешивает. Если польза > инвестиций → покупает.

**Следствие:** работать с обеими сторонами. Не только «вот польза», но и снижать ощущение боли от цены (рассрочка, гарантии, контекстуализация — «одна консультация стоит 50к, а тут целая система за 200к»).

### Ожидания как фактор

Ценность зависит не только от абсолютных цифр пользы/инвестиций, но от их соотношения с ожиданиями:
- **Позитивная ошибка предсказания** = получил больше чем ждал → вау-эффект, рекомендации
- **Негативная ошибка** = получил меньше → разочарование, потеря клиента
- **Оптимум:** обещать то, что можешь выполнить, и чуть-чуть занижать → клиент получает больше → вау

## 2. Граф работ

Граф работ = иерархия работ для удовлетворения потребностей. Именно ГРАФ (связи многие-ко-многим), не дерево.

**Структура:**
- Big Job — верхнеуровневая работа сегмента
- Core Jobs — основные работы (3-7 штук)
- Small/Micro Jobs — работы ниже уровнем
- Предыдущие работы — что делал ДО текущей
- Следующие работы — что делает ПОСЛЕ

**CJM vs Граф работ:**
- CJM = проекция графа работ на временную шкалу
- Граф работ даёт 50+ механик решения бизнес-задач, CJM — нет

**Критическая последовательность** — работы, которые клиент НЕ МОЖЕТ не выполнить, чтобы Big Job сработала. Фокус оффера — на этих работах.

## 3. ~20 механик создания ценности

Механики ценности — это способы преобразования графа работ клиента, которые увеличивают Job Solution Value. Скилл выбирает 2-3 самые применимые для фокусного сегмента.

### Механики через снижение Cost:

1. **Убить работы** — клиент ждал что придётся делать Х, а не пришлось. Пример: ChatGPT убил работу «знать испанский» для написания письма на испанском
2. **Ускорить работы** — та же работа, но быстрее. Пример: AI-копирайтер — текст за 5 минут вместо 2 часов
3. **Упростить работы** — снизить когнитивную нагрузку. Пример: Canva vs Photoshop
4. **Убрать налоговые работы** — устранить побочные работы от текущего решения
5. **Снизить входной барьер** — бесплатный период, пробный доступ, демо
6. **Снизить риск** — гарантия возврата, «попробуй бесплатно»

### Механики через увеличение Benefit:

7. **Выполнить больше работ одним решением** — «всё в одном». Пример: Notion = заметки + база + проект-менеджмент
8. **Выполнить работу качественнее** — по критериям клиента (не своим!)
9. **Выйти на работу выше уровнем** — выполнить Big Job, а не Core. Пример: LiFT для топ-менеджеров — не «набор услуг по личному бренду», а «прокачать личный бренд за меня». Чек x3
10. **Выйти на предыдущую работу** — забрать работу до текущей. Пример: Aviasales = работа «найти дешёвые билеты» ДО работы «купить билеты»
11. **Добавить эмоциональную ценность** — не только функция, но и ощущение. Пример: Apple — не просто ноутбук, а ощущение принадлежности
12. **Создать привычку** — встроиться в регулярный цикл клиента
13. **Персонализировать под контекст** — адаптировать решение под конкретную ситуацию клиента

### Механики через структуру:

14. **Выполнить критическую последовательность** — закрыть все обязательные работы в графе, не оставляя пробелов
15. **Объединить точки контакта** — один интерфейс вместо 5 сервисов
16. **Автоматизировать переключения** — клиенту не нужно вручную переходить между шагами

### Два типа «выхода наверх по графу»:

**Тип А — работы ниже УМИРАЮТ как класс.** Продукт поднимается на Big Job, нижние работы перестают существовать. Пример: ChatGPT убил работу «знать язык» для перевода. Следствие: кардинально дороже конкурентов, позиционирование через отсутствие работ.

**Тип Б — работы ОСТАЮТСЯ, но внутри продукта.** Продукт выполняет их за клиента. Пример: SaaS для контент-креаторов — тренды, сценарий, генерация, публикация — всё внутри. Следствие: дороже на фактор экономии времени, позиционирование через «быстрее и проще».

## 4. Consideration Set и аукцион решений

**Consideration Set** — набор решений в голове клиента для данной работы. Включает:
- Прямых конкурентов (аналогичные продукты)
- Заменители (YouTube, книги, менторы, самообучение)
- «Ничего не делать» (всегда в наборе!)

**Аукцион решений:** когда работа активируется, мозг проводит внутренний аукцион. Приоритет — привычное решение. Новое побеждает только если:
- Привычное недоступно
- Привычное перестало давать ценность
- Новое заметно выгоднее

**Следствие:** информацию о продукте нужно загружать в голову ДО активации работы.

## 5. Стратегические ходы

### Подняться на Big Job выше
Выполнить работу выше уровнем → другой уровень ценности, другой чек.
- Пример: РосКо — от «продать запчасти» к «зарабатывать на обслуживании клиентов» → CRM для СТО. Рост +5% выручки на клиента.

### Выйти на предыдущую работу
Забрать работу, которую клиент делает ДО нашей.
- Пример: трейд-ин у автодилеров — работа «продать старую машину» ДО «купить новую».
- Пример: Домклик — «рассчитать кредит» ДО «оформить ипотеку».

### Сместиться в соседний сегмент
Добавлять сегменты с бо́льшим доходом и требованиями.
- Пример: Яндекс Такси — от эконома к бизнесу и премиуму.

## 6. Порождённые работы

Каждое новое решение на рынке порождает новую работу у клиентов:

1. Появление нового решения (ChatGPT, электросамокат)
2. Первые пользователи формируют новую работу
3. Работа тиражируется через социальное заражение
4. Новая работа порождает запрос на новые решения
5. Старые решения обесцениваются

**Для инфобиза JTBD-методологии критично:**
- ChatGPT породил работу «решать задачи через AI»
- Курс без AI выглядит устаревшим
- LLM-агенты породили работу «делегировать рутину AI»

**Применение:** при анализе проверять, какие новые работы порождены на рынке и учтены ли они.

## 7. B2B-специфика (кратко)

**Два слоя работ для ЛПР:**
- Бизнес-работа («вырастить продажи»)
- Личная работа («не потерять работу», «получить бонус», «быть повышенным»)

**90% мотивации — личные работы.** Бизнес-работы — социально приемлемый фасад.

**Много ролей в сделке:** Бизнес-заказчик, ЛПР, ЛВР, Держатель бюджета, Саботажники, Конечные пользователи — у каждой роли своя работа.

**Ожидаемая польза шире:** простота внедрения, TCO, надёжность поставщика, гарантии поддержки.

## 8. Подсегменты и масштабирование

Внутри любого сегмента всегда есть подсегменты.

**Масштабирование** = постепенный захват подсегментов внутри фокусного.

**Возражения клиентов = карта подсегментов:**
- «А подойдёт ли для моей ниши?» → подсегмент по нише
- «А если нет техкоманды?» → подсегмент по техуровню
- «А хватит ли времени?» → подсегмент по занятости

Фиксировать → группировать → превращать в продуктовые гипотезы → приоритизировать.

## 9. Ошибки сегментации (5 ошибок JTBD-методологии)

1. **Слишком широкий сегмент.** «Все предприниматели» — не сегмент
2. **Слишком узкий сегмент.** Маленький убыточный — потерянные годы
3. **Сегментация по демографии.** Пол/возраст/город не дают причинно-следственных связей
4. **Следствия вместо причин.** Работать с симптомами, а не с работами
5. **Один сегмент навсегда.** Сегмент нужно пересматривать по мере роста

**Радикальная формулировка JTBD-методологии:** «от большинства сегментов нужно отказаться». Выбор сегмента = фокус. Без отказа от остальных — размывание.

## Эвристика «не тот сегмент»

Если клиенты продерутся через кривой онбординг — сегмент правильный, ценность создана.

Если клиенты не отвечают, сливаются, не идут на контакт — сегмент не тот или ценность недостаточна.

**Бенчмарки отклика:**
- Email → 10% конверсия в интервью (стандарт), до 40% (пик)
- WhatsApp → 20-40% в интервью, 50% в любой ответ
