# Протокол разбора инбокса

Правила классификации для скилла `/intake`. Пользователь кидает всё в `inbox/` — скилл раскладывает по папкам согласно таблице ниже.

## Таблица классификации

| Что пришло | Признаки | Куда | Как |
|------------|----------|------|-----|
| Задача / ТЗ | "надо сделать X", дедлайн, requirements | `projects/<relevant>/` | добавить в существующий проект или завести через `/new-project` |
| Бриф / идея проекта | описание концепции, цели, ЦА | `projects/<name>/brief.md` | если проекта нет — создать через `/new-project` |
| Пользователь / лид | имя + компания, контакты, запрос | `clients/<slug>/` | создать через `/new-client`, контент положить внутрь карточки |
| Знание / референс | статья, методика, конспект, шаблон | `knowledge/<slug>.md` | создать через `/new-knowledge` |
| Черновик / текст | пост, письмо, фрагмент копирайта | `projects/<relevant>/artifacts/` | положить в артефакты нужного проекта |
| Скриншот / картинка | изображение | `projects/<relevant>/artifacts/` или оставить в inbox | если контекст ясен — в проект, если нет — спросить |
| Ссылка без текста | только URL | посмотреть контент → классифицировать по содержимому | — |
| Неясно | нет явных признаков | остаётся в `inbox/` + запись в `_questions.md` | спросить пользователя что это |

## Приоритеты при конфликте

Если файл подходит под несколько категорий — выбирай в таком порядке:

1. **Пользователь** перекрывает знание (если есть имя живого человека в бизнес-контексте → это пользователь, даже если там есть "полезная методика").
2. **Проект** перекрывает знание (если явно привязано к задаче → в проект, а не в общую базу).
3. **Знание** — если переиспользуемо и не привязано к конкретной задаче/пользователю.

## Правила

- Не перемещай файлы без подтверждения пользователя. Сначала покажи план таблицей, жди "ок".
- Если создаёшь сущность через `/new-project`, `/new-client`, `/new-knowledge` — перенеси содержимое файла в соответствующее место карточки, оригинал удали из inbox.
- После разбора обнови `INDEX.md` в `clients/` и `knowledge/` (если затронуты).
- `README.md` в inbox — не трогать, это инструкция для пользователя.

## Что делать с `_questions.md`

Если что-то осталось неразобранным — в `inbox/_questions.md` добавь строку:

```
- <дата> — <имя файла> — непонятно что это, контекста не хватает
```

Пользователь сам посмотрит и уточнит в следующий запуск `/intake`.
