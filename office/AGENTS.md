# Команда офиса

**Это единственный источник правды о составе команды.** Новые агенты устанавливаются через `/install-agent <name>` и добавляются сюда автоматически. Все остальные упоминания в корневом `CLAUDE.md`, `director/core.md`, `demiurg/core.md` ссылаются сюда.

Не нужно писать «команда из N» — количество меняется по мере установки агентов.

## Как работать с агентами

**Обычная задача** — через Director:
> "Нужно разобраться со стратегией на квартал"

Director сам позовёт нужного помощника из текущей команды.

**Онбординг программы:**
```
/strategist-intake     # первичная распаковка
/strategist-unpack     # детальный разбор документов
/strategist-discovery  # уточняющий диалог
/strategist-prepare    # сверка собранного и подготовка офиса
/strategist-roadmap    # финал: стратегия + 6-недельный план + HTML-роадмап
/strategist-checkin    # еженедельно
```

**Прямое обращение** к агенту: `@director`, `@strategist`, `@demiurg`, `@designer`, `@alex` (или `@alex-marketer`).

**Онбординг визуала (Дизайнер):**
```
Собери мне Brand Book    # первый шаг — 10 минут диалога, артефакт в knowledge/brand/
Хочу лендинг / обложку / презентацию   # Дизайнер предложит тип и сам соберёт (или даст промт если попросишь)
```

## Layered memory

У каждого агента — 4 файла:
- `core.md` — ядро, обновляется из template при будущих апдейтах
- `overrides.md` — твои персональные правила, приоритет над core, **не перезаписывается** при апдейтах
- `memory.md` — append-only, агент пишет сам после задач (decisions / patterns / context)
- `failures.md` — append-only, агент пишет сам о своих ошибках чтобы не повторять
- `CLAUDE.md` — склейка `@core.md` + `@overrides.md` (Claude Code читает это)

Пиши свои кастомизации в `overrides.md`. Так ты не потеряешь их когда обновится ядро.

## Добавление нового агента

Новые помощники поставляются готовыми паками в `_agent-packs/<name>/`. Установка — одной командой:

```
/install-agent <name>
```

Что произойдёт автоматически:
1. Файлы агента скопируются в `office/agents/<name>/`
2. Добавится строка в таблицу **Установленные агенты** выше
3. Добавится `@office/agents/<name>/core.md` в layered include корневого `CLAUDE.md`
4. Обновятся триггеры в `director/core.md` и `director/knowledge/routing-patterns.md`

Какие паки уже готовы к установке — смотри `_agent-packs/`.

Принципы интеграции и архитектура паков — `AGENT-INTEGRATION-PRINCIPLES.md`.
