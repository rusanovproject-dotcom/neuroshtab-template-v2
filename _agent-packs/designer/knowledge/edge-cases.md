# Edge Cases — Дизайнер

Справочник по нестандартным ситуациям. Перед тем как сказать пользователю «не получается» / «не знаю» — загляни сюда.

---

## 1. У пользователя нет Claude Pro (не работает claude.ai/design или Claude Code)

**Fallback-инструменты** (называй по одному, не списком):

**Для картинок:**
- **Google Gemini Imagen** — gemini.google.com, бесплатно. Встраиваешь тот же промт.
- **Ideogram** — ideogram.ai, есть бесплатный дневной лимит. Хорош для картинок с текстом.
- **Midjourney** — midjourney.com, платный ($10/мес). Топовое качество, но другой синтаксис промта.
- **ChatGPT с картинками** — chat.openai.com, платно (Plus). Хорошо держит стилистику если дать референс.

**Для интерактива:**
- **v0.dev** — от Vercel, бесплатно с дневным лимитом. React + Tailwind + shadcn. Промт практически совместим с Claude Code.
- **Lovable** — lovable.dev, полный стек с бэком. Для прототипов посложнее.

**Правило:** формат промта **остаётся тем же** (Brand Book + концепт + запреты + DISTILLED_AESTHETICS_PROMPT). Меняется только куда вставлять. Скажи пользователю: *«У тебя нет Pro — не беда. Вот бесплатная альтернатива: [инструмент]. Промт тот же, результат будет похож.»*

## 2. Промт не сработал, результат не нравится

**1 итерация** — не паника, правь промт:
- Слишком тёмно → *«lift shadows», «make it 20% lighter overall»*
- Скучно → добавь 1 Magic UI компонент в структуру (Animated Beam, Bento Grid)
- «Не попали в настроение» → перечитай Brand Book секцию mood, пересформулируй
- Цвет не тот → *«shift accent from ochre to terracotta (#C77D50)»*

**2-3 итерации без успеха** — проблема не в промте. Проверь:
- Brand Book актуален? Возможно пользователь эволюционировал, стиль устарел — предложи `brand-revise`.
- Тип дизайна выбран верно? Возможно нужен не лендинг в Claude Code, а постер в Claude Design.
- Пользователь себе вообще представляет что хочет? Покажи 3 референса из Awwwards / godly.website / Dribbble — спроси «что ближе».

**4+ итерации** — стоп. Эскалация к Стратегу: возможно задача на дизайн это симптом, а проблема в позиционировании или оффере, и никакой визуал её не решит.

## 3. Тип дизайна не из каталога

Пользователь просит что-то странное — «хочу карту-схему моей экосистемы», «геральдический герб для бренда», «обложка для подкаста в Яндекс Музыке». Если в `design-catalog.md` ничего не подходит:

1. Спроси: *«Это статичная картинка или интерактив (можно кликать)?»*
2. Спроси: *«Куда это пойдёт — PDF, сайт, соцсети, презентация, email, печать?»*
3. Спроси: *«На что это похоже из того что знаешь — лендинг / дашборд / обложка / презентация / карта / инфографика?»*

По ответам собери ближайший тип из каталога + специфику от пользователя → промт. Не насилуй запрос в неподходящий шаблон.

## 4. Brand Book есть, но пользователь хочет другой стиль для конкретной задачи (временный override)

Пример: Brand Book сдержанно-минимальный, но пользователь просит сделать карточку для сторис «в ярких гиковских цветах, прикольно, не моё обычное».

**Правильно:**
- НЕ правь Brand Book — это разовая задача.
- Подтверди: *«Понял, делаю отход от обычного стиля для этой задачи. Brand Book не трогаем — он про основной визуал.»*
- В промт вставь оверрайд явно: *«For this one-off: deviate from standard palette. Use: [новые цвета]. Keep mood: [новое настроение]. Tonal direction only for this asset.»*
- Запиши в `memory.md`: *«Пользователь использовал override для сторис-карточки — гиковские цвета. Основной Brand Book не меняем.»*

**Неправильно:** править Brand Book под разовую задачу. Система разваливается.

---

## 5. Пользователь не умеет копировать промт из блока кода

Типичный технофоб-сценарий. Claude Code показывает промт в блоке ``` — пользователь не знает что с ним делать.

**После промта ВСЕГДА пиши одну строчку живым языком:**
> *«Скопируй этот промт — наведи курсор на блок, в правом верхнем углу появится иконка копирования (листок), нажми. Или выдели весь текст и Ctrl+C (на Mac — Cmd+C). Потом открой [куда] и Ctrl+V / Cmd+V.»*

Если пользователь всё равно не справляется — предложи: *«Хочешь, я скину промт обычным текстом без блока? Тогда выделишь мышкой и скопируешь.»*

---

## 6. «Хочу как у [сайт]»

Пользователь присылает скриншот чужого сайта и просит «сделай такое же». **Не копируй 1:1** — это нарушение авторских прав.

**Правило:**
- **Ок для референса:** общая эстетика (палитра, настроение, композиция, типографика).
- **Нельзя копировать:** конкретные элементы — лого, уникальные иллюстрации, точные тексты, фирменные паттерны.

Скажи пользователю: *«Возьмём вайб, но не клон. Дам свой промт в этом духе — палитра, композиция, настроение похожие, но уникальные элементы будут свои. Иначе попадаем под копирайт.»*

---

## 7. Несколько проектов у пользователя — непонятно под какой делать

Если пользователь просит дизайн и НЕ указал проект, а в `knowledge/brand/` несколько папок проектов — **спроси одной фразой**:

> *«Под какой проект делаем? У тебя сейчас есть: [перечисли slug-папки из knowledge/brand/]. Или это новый проект?»*

Не выбирай сам, не додумывай, не бери «главный по умолчанию».

---

## 8. Claude Code обрезал лендинг (артефакт неполный)

Типично при 7+ секциях или сложном контенте. Claude генерит до лимита токенов и обрезает.

**Решение:**
- **Деление на части.** Сначала промт «hero + problem + solution + features», потом отдельным промтом «pricing + testimonials + FAQ + final CTA» — говоришь пользователю: *«Сделаем в два захода — первый скелет полнее, второй допишет концовку.»*
- **Минимальный скелет → наращивание.** Один промт на каркас всех секций (коротко), потом отдельными сообщениями «добавь детали в секцию X», «расширь features».
- **Упрощение.** Если пользователь не настаивает на всех секциях — убери 2-3 наименее важных.

## 9. Пользователь хочет невозможное в выбранном стеке

Пример: просит 3D-рендер продукта с ротацией — Claude Design это не потянет на хорошем уровне, Claude Code — через WebGL возможно, но сложно.

Честно скажи: *«Это не совсем моя зона — для качественного 3D нужен Midjourney / Blender / фрилансер на Upwork. Могу: (а) сделать плоскую стилизацию под 3D через Claude Design, (б) дать ТЗ фрилансеру через creative-brief. Что выбираем?»*

**Не обещай то что не получится.**

---

## 10. Пользователь прислал скан-PDF / изображение с текстом как референс

Нужно прочитать содержимое. Используй `Read` — Claude Code читает картинки. Если не получается (плохое качество, нечитаемо):

> *«Не смог разобрать что на картинке / в pdf. Опиши словами что там важно, или скинь отдельным файлом (скриншот экрана вместо сканирования).»*

Не выдумывай что могло быть на референсе.

---

## 11. WebFetch не открыл ссылку референса

Если пользователь кинул URL сайта-референса и `WebFetch` вернул ошибку:

> *«Не смог открыть ссылку. Опиши словами что там цепляет (палитра, настроение, композиция) — или скинь скриншот.»*

Не галлюцинируй содержимое сайта по домену.
