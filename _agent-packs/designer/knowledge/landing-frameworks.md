# Смысловая и логическая структура лендингов 2026

> Назначение файла: база знаний Designer-агента для AI-офиса учеников.
> Designer должен не просто нарисовать красиво — а выбрать правильную структуру смыслов под конкретный оффер/нишу/аудиторию.

---

## 1. TL;DR — топ-5 фреймворков 2026

Если нужно выбрать один стек для работы и ничего больше не читать: **StoryBrand SB7** даёт скелет истории (пользователь = герой, бренд = гид); **PAS** — самый быстрый способ написать убедительный блок для любой нише с болью; **Hormozi Value Equation** — единственный фреймворк который объясняет *почему* ваш оффер должен стоить именно столько; **Schwartz Awareness Levels** — диагностика с чего начинать лендинг (аудитория не знает что болит vs аудитория уже сравнивает варианты); **Jobs-to-be-Done** — инструмент чтобы вообще понять ЧТО писать, пока остальные фреймворки говорят КАК. Все остальные — вариации или углубления этих пяти.

## 2. Таблица 20+ фреймворков

| Фреймворк | Когда применять | Сильная сторона | Ниша-пример |
|-----------|----------------|-----------------|-------------|
| **AIDA** | Аудитория уже в режиме покупки, знает проблему | Энергичный путь к конверсии, работает в ads + hero | SaaS free trial, e-commerce |
| **PAS** | Боль очевидна и острая | Максимальная эмпатия, затем выход | Health, EdTech, коучинг, B2B |
| **PASTOR** | Нужна история + глубокое убеждение | Покрывает весь путь от боли до трансформации | Онлайн-курсы, услуги, личный бренд |
| **BAB** | Короткий формат, история трансформации | Создаёт эмоциональный momentum в 3 движения | Social ads, короткие лендинги, кейсы |
| **StoryBrand SB7** | Любой продукт где нужно позиционировать себя как гида | Чёткая роль бренда, пользователь = герой | Консалтинг, services, сложный SaaS |
| **4Ps** (Promise-Picture-Proof-Push) | Продажа трансформации | Визуализация будущего состояния | Курсы, коучинг, онлайн-программы |
| **Hormozi Value Equation** | Обоснование цены, борьба с "дорого" | Математика ценности: outcome × likelihood / time × effort | Премиум-офферы, мастер-группы |

| **QUEST** | Сложный продукт, нужно сначала квалифицировать | Начинает с отбора, убирает нерелевантных | B2B SaaS, дорогие программы |
| **FAB** (Features-Advantages-Benefits) | Продукт с конкретными характеристиками | Переводит фичи в пользу для пользователя | SaaS, tech products, e-commerce |
| **ACCA** (Awareness-Comprehension-Conviction-Action) | Сложные или инновационные решения | Ведёт через понимание к убеждению | B2B, новые категории продуктов |
| **Star-Chain-Hook** | Нужен сильный нарратив вокруг Big Idea | Идея → доказательства цепочкой → захват | Инфопродукты, presell pages |
| **BAB + AIDA (гибрид)** | Длинный лендинг, сложный оффер | BAB строит контекст, AIDA закрывает | Курсы + SaaS |
| **Jobs-to-be-Done (JTBD)** | До написания структуры, для диагностики | Выявляет истинную мотивацию покупки | Любая ниша |
| **Four Forces JTBD** | Когда нужно снять тревогу покупки | Push + Pull + Anxiety + Habit = структура возражений | Switching products, re-engagement |
| **Schwartz Awareness (5 уровней)** | Диагностика аудитории перед выбором структуры | Определяет с какого места начинать лендинг | Любая ниша |
| **April Dunford Positioning** | Когда продукт непонятен рынку | Создаёт контекст сравнения (market category) | Новые SaaS, инновационные продукты |

| **Peak-End Rule** | Проектирование опыта страницы | Запоминается пик + финал, не середина | Любой длинный лендинг |
| **Message-Market-Mechanism** | Запуск нового продукта | Уникальный механизм как differentiator | Инфопродукты, новые методологии |

## 3. Детальные карточки топ-10 фреймворков

### 3.1 PAS — Problem, Agitation, Solution

**Механика:**
1. Problem — назови боль конкретно, словами аудитории
2. Agitation — усиль боль (последствия, эмоции, что будет если ничего не менять)
3. Solution — покажи выход как логичное следствие

**Когда работает:**
- Боль очевидна и острая (усталость, деньги, время)
- Аудитория: problem-aware или solution-aware (см. Schwartz)
- Ниши: здоровье, финансы, продуктивность, EdTech, B2B operations

**Когда НЕ работает:**
- Unaware аудитория (они ещё не знают что болит)
- Luxury-продукты где нет боли, есть желание

**Структура лендинга на PAS:**
```
Hero: [Боль в заголовке — точными словами аудитории]
Block 2: [Развёртывание боли — 3-5 конкретных проявлений]
Block 3: [Последствия если ничего не менять]
Block 4: [Разворот — "Есть другой путь"]
Block 5: [Решение + механизм]
Block 6: [Социальное доказательство]
Block 7: [Оффер + CTA]
```

**Промт-паттерн:**
```
Ты — прямой маркетолог. Напиши лендинг на основе PAS-структуры.

Продукт: [название]
Целевая аудитория: [кто]
Главная боль: [в их словах, не в твоих]
Решение: [механизм]

Структура:
1. Hero — заголовок = боль (5-8 слов, словами аудитории)
2. Problem — 3 конкретных проявления боли с деталями
3. Agitation — что будет через год если ничего не изменить
4. Solution — как продукт решает (механизм, не фичи)
5. Proof — 2-3 трансформационных отзыва (до/после)
6. CTA — одно действие, ориентированное на выгоду

Тон: прямой, без воды, без корпоративного языка.
```

### 3.2 AIDA — Attention, Interest, Desire, Action

**Механика:**
1. Attention — захват внимания (hook, заголовок, статистика)
2. Interest — удержание через релевантность и любопытство
3. Desire — создание желания (выгода, трансформация, visuals)
4. Action — чёткий призыв к действию

**Когда работает:**
- Аудитория solution-aware или most-aware
- Короткие/средние лендинги, ads, email subject lines
- E-commerce, SaaS free trial, быстрые конверсии

**Когда НЕ работает:**
- Дорогой оффер (>$500) — AIDA слишком быстра, нужен PASTOR/StoryBrand
- Unaware аудитория — нет точки входа через внимание

**Структура лендинга на AIDA:**
```
Hero: [Hook/заголовок — сюрприз/вопрос/статистика]
Subhero: [Почему это важно для тебя]
Block 2: [Что ты получишь — конкретные выгоды]
Block 3: [Социальное доказательство/демо/визуал]
CTA: [Одно действие, срочность]
```

**Промт-паттерн:**
```
Напиши hero-секцию лендинга по AIDA-структуре.

Продукт: [название + одно предложение что делает]
ЦА: [сегмент]
Желаемое действие: [регистрация/покупка/звонок]

Дай 5 вариантов заголовка (Attention) — разных по подходу:
- Вопрос с болью
- Статистика-сюрприз  
- Конкретный результат
- Контраст (до/после)
- Прямое обещание

Для лучшего варианта напиши полную AIDA-последовательность:
Attention (заголовок) → Interest (2-3 предложения) → Desire (3 bullet-point выгоды) → Action (CTA кнопка)
```

### 3.3 StoryBrand SB7 — Donald Miller

**Механика (7 элементов):**
1. A Character — пользователь = герой (не компания)
2. Has a Problem — внешняя + внутренняя + философская проблема
3. And Meets a Guide — ваш бренд = гид (не герой)
4. Who Gives Them a Plan — конкретные шаги 1-2-3
5. And Calls Them to Action — прямой призыв
6. That Helps Them Avoid Failure — что случится если не действовать
7. And Ends in Success — трансформация, жизнь после

**Ключевой принцип:** Бренд который позиционирует себя героем — проигрывает. Пользователь = герой, вы = Йода.

**Когда работает:**
- Консалтинг, коучинг, услуги
- Любой сложный продукт где нужно объяснить ценность
- Личный бренд эксперта
- B2B SaaS с несколькими стейкхолдерами

**Структура лендинга на StoryBrand:**
```
Hero: [Проблема пользователя — их слова] + [Что получат = transformation]
Block 2: Мы понимаем твою боль (empathy + authority как гид)
Block 3: Наш план — 3 шага к результату
Block 4: Что ты получишь (outcomes, не фичи)
Block 5: Без нас: что потеряешь / риски провала
Block 6: С нами: картина успеха в деталях
Block 7: Social proof — история трансформации
Block 8: CTA + гарантия
```

**Промт-паттерн:**
```
Напиши лендинг по фреймворку StoryBrand SB7.

Заполни Brand Script:
- Герой (пользователь): [кто он, его контекст]
- Внешняя проблема: [что не работает]
- Внутренняя проблема: [как он себя чувствует из-за этого]
- Философская проблема: [почему это несправедливо]
- Гид (мы): [наш авторитет + эмпатия в 1 предложении]
- Наш план: [3 конкретных шага]
- Прямой CTA: [одно действие]
- Failure (без нас): [конкретные потери]
- Success (с нами): [трансформация в деталях]

Напиши текст для каждой секции лендинга, используя Brand Script выше.
Тон: человечный, как разговор с другом-экспертом.
```

### 3.4 PASTOR — Problem, Amplify, Story, Transformation, Offer, Response

**Механика:**
1. Problem — боль аудитории
2. Amplify — усиление + последствия
3. Story — история трансформации (кейс)
4. Transformation — что стало возможным
5. Offer — конкретный оффер с ценностью
6. Response — CTA

**Когда работает:**
- Самый полный фреймворк для длинных лендингов
- Курсы, мастер-группы, высокочековые услуги
- Когда нужна история + трансформация + продажа
- Аудитория: problem-aware или solution-aware

**Структура лендинга на PASTOR:**
```
Hero: [Проблема + кому адресован лендинг]
Block 2: Amplify — углубляем боль, последствия
Block 3: Story — кейс/история трансформации (до/после/как)
Block 4: Transformation — что теперь возможно
Block 5: Оффер в деталях (что входит, что получишь)
Block 6: Social proof + кейсы
Block 7: Pricing + гарантия
Block 8: FAQ (возражения)
Block 9: CTA
```

**Промт-паттерн:**
```
Напиши лендинг для [продукт] по фреймворку PASTOR.

Данные:
- Продукт: [название + суть]
- ЦА: [сегмент + их контекст]
- Боль: [конкретная]
- Трансформационная история: [имя пользователя + до/после/как]
- Оффер: [что получат, формат, срок]
- Цена: [сумма + что обосновывает]

Напиши каждый блок:
P — Problem (заголовок + 3 конкретных проявления)
A — Amplify (что будет если не действовать)
S — Story (история пользователя 200-300 слов)
T — Transformation (картина жизни после)
O — Offer (полный состав, как это работает)
R — Response (CTA + что происходит после клика)
```

### 3.5 Hormozi Value Equation

**Механика (формула):**
```
Ценность = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)
```

**Четыре рычага:**
1. Dream Outcome — насколько грандиозен желаемый результат
2. Perceived Likelihood — насколько верят что получат результат
3. Time Delay — как быстро получат результат
4. Effort & Sacrifice — сколько усилий потребует путь

**Для лендинга:** каждый блок должен двигать хотя бы один из 4 рычагов.

**Когда работает:**
- Обоснование высокой цены
- Конкурентные рынки где все говорят одно и то же
- Запуск премиум-офферов

**Структура лендинга на Value Equation:**
```
Hero: [Dream Outcome в заголовке — самый большой желаемый результат]
Block 2: [Perceived Likelihood — доказательства что получат: кейсы, гарантии, цифры]
Block 3: [Time Delay — как быстро начнут видеть результат + быстрые победы]
Block 4: [Effort minimization — мы берём сложное на себя, им не придётся...]
Block 5: [Grand Slam Offer — полный состав с ценностью каждого элемента]
Block 6: [Risk Reversal — гарантия убирает последний барьер]
Block 7: [Urgency/Scarcity + CTA]
```

**Промт-паттерн:**
```
Создай лендинг-оффер по Hormozi Value Equation.

Заполни уравнение для [продукта]:
- Dream Outcome: [самый большой результат который получит пользователь]
- Likelihood proof: [почему верить что получит — 3 доказательства]
- Time to first result: [когда почувствует первый результат]
- What they DON'T need to do (effort removed): [что ты берёшь на себя]
- Price: [сумма]

Напиши:
1. Заголовок который продаёт Dream Outcome (не продукт)
2. 3 блока которые поднимают Perceived Likelihood (кейсы/гарантии/данные)
3. "Быстрые победы" — что получат в первую неделю
4. Breakdown оффера: что входит + ценность каждого элемента
5. Гарантию которая убирает риск
6. CTA с urgent trigger
```

### 3.6 BAB — Before, After, Bridge

**Механика:**
1. Before — жизнь без продукта (боль, ограничение)
2. After — жизнь с продуктом (желаемое состояние)
3. Bridge — как перейти из Before в After (ваш продукт)

**Когда работает:**
- Короткие блоки и секции внутри длинного лендинга
- Ads, email, case study-блоки
- Ниши с чёткой трансформацией (похудение, финансы, карьера)

**Промт-паттерн:**
```
Напиши BAB-блок для [продукта].

Before: [конкретная ситуация пользователя сейчас — в деталях]
After: [конкретная жизнь после — в деталях, с цифрами если есть]
Bridge: [как продукт создаёт переход — механизм в 2-3 предложениях]

Дай 3 варианта разной длины:
- Короткий (для ad copy, 50-80 слов)
- Средний (для лендинг-блока, 150-200 слов)  
- Длинный (для email/case study, 300+ слов)
```

### 3.7 QUEST — Qualify, Understand, Educate, Stimulate, Transition

**Механика:**
1. Qualify — отбор (вопрос: "Это для вас если...")
2. Understand — показываем понимание их ситуации
3. Educate — обучаем новому способу думать о проблеме
4. Stimulate — создаём желание решения
5. Transition — переход к действию

**Когда работает:**
- Сложные продукты требующие изменения убеждений
- Дорогие B2B решения
- Аудитория скептичная или solution-aware

**Структура лендинга на QUEST:**
```
Hero: [Квалифицирующий вопрос — "Вы [X]?"]
Block 2: Мы понимаем твою ситуацию (specific details)
Block 3: Почему привычный подход не работает (reframe)
Block 4: Новый способ + механизм
Block 5: Желание — трансформация + кейсы
Block 6: CTA + следующий шаг
```

### 3.8 FAB — Features, Advantages, Benefits

**Механика:**
- Feature → Advantage → Benefit (для пользователя)
- "У нас X [фича], это значит Y [преимущество], поэтому ты получишь Z [выгода]"

**Когда работает:**
- Продукт с конкретными характеристиками
- Блоки с фичами внутри любого лендинга
- SaaS, tech, e-commerce

**Промт-паттерн:**
```
Преобразуй список фич [продукта] в FAB-формат.

Список фич: [фича 1, фича 2, фича 3...]

Для каждой фичи напиши:
- Feature: [техническая характеристика]
- Advantage: [что это даёт в сравнении с альтернативами]
- Benefit: [как это меняет жизнь пользователя — конкретно]

Затем выбери ТОП-3 наиболее убедительных FAB и напиши для них
карточки-блоки для лендинга (заголовок + 2-3 предложения + иконка-идея).
```

### 3.9 4Ps — Promise, Picture, Proof, Push

**Механика:**
1. Promise — обещание трансформации (что изменится)
2. Picture — визуализация желаемого состояния
3. Proof — доказательства что это возможно
4. Push — мотивация действовать сейчас

**Когда работает:**
- Образовательные продукты, курсы
- Коучинг, программы изменений
- Когда главный барьер — "я не верю что это возможно для меня"

---

### 3.10 ACCA — Awareness, Comprehension, Conviction, Action

**Механика:**
1. Awareness — осознание существования проблемы/решения
2. Comprehension — понимание как работает решение
3. Conviction — убеждение что это работает (для меня)
4. Action — призыв к действию

**Когда работает:**
- Сложные или новые категории продуктов
- B2B где несколько стейкхолдеров принимают решение
- Аудитория unaware или problem-aware

## 4. Schwartz Awareness Levels — полная разводка

Источник: "Breakthrough Advertising" Eugene Schwartz (1966), актуальность 2026 подтверждена Leadgen Economy, Game of Conversions, B-PlanNow.

**Принцип:** Структура лендинга меняется полностью в зависимости от того, насколько аудитория осознаёт проблему и решение. Один и тот же продукт — разные лендинги для разных сегментов.

---

### Уровень 1: UNAWARE (не знает что болит)

**Кто это:** Человек который живёт с проблемой, но не идентифицирует её как проблему. Не ищет решения.

**Откуда приходит:** Таргетированная реклама, контент, соцсети — холодный трафик.

**С чего начинать лендинг:**
- Не с продукта — с истории или провокационного вопроса
- Истории и кейсы других людей в похожей ситуации
- Эмоциональные триггеры ("узнай себя в этом")
- Никаких упоминаний продукта выше склейки

**Структура:**
```
1. Hook: провокационный вопрос или история (без упоминания продукта)
2. "Узнай себя" — 5-7 симптомов/ситуаций
3. Объяснение почему так происходит (edutainment)
4. История трансформации другого человека
5. Мягкий переход: "Есть способ изменить это"
6. Lead magnet или low-commitment CTA
```

**Пример заголовка для Unaware:**
- "Почему умные люди годами работают больше, но денег не становится больше" (не "купи курс по продуктивности")

---

### Уровень 2: PROBLEM AWARE (знает боль, не знает решения)

**Кто это:** Знает что что-то не так, активно страдает, но не знает как называется решение.

**Откуда приходит:** Поисковые запросы по симптомам ("как перестать прокрастинировать"), соцсети с контентом по боли.

**С чего начинать:**
- С боли (эмпатия + агитация)
- PAS — идеальный фреймворк
- Создать "таинственность" вокруг решения

**Структура:**
```
1. Hero: Боль в заголовке — их словами
2. Агитация: 3-4 следствия боли
3. Reframe: "Это не ваша вина — вот почему это происходит"
4. Введение решения (концепция, не продукт)
5. Как работает механизм
6. Социальные доказательства: люди с такой же болью изменились
7. Оффер + CTA
```

**Пример заголовка для Problem Aware:**
- "Вы каждый день начинаете и не заканчиваете. Вот почему это происходит и как это остановить"

---

### Уровень 3: SOLUTION AWARE (знает тип решения, не знает вас)

**Кто это:** Знает что такие продукты/услуги существуют, сравнивает варианты.

**Откуда приходит:** Поиск по категории ("лучший курс по управлению временем"), сравнительные запросы.

**С чего начинать:**
- С уникальности вашего подхода (mechanism)
- Testimonials и case studies
- Почему ваш способ лучше других

**Структура:**
```
1. Hero: Уникальный механизм или обещание vs. конкуренты
2. "Почему [обычный подход] не работает" (reframe)
3. Ваш уникальный метод/механизм
4. Детальные кейсы с числами
5. Сравнение с альтернативами (позиционирование)
6. Детали оффера
7. CTA + гарантия
```

**Пример заголовка для Solution Aware:**
- "Не ещё один курс по тайм-менеджменту. Метод [X] который работает потому что [механизм]"

---

### Уровень 4: PRODUCT AWARE (знает вас, не решился)

**Кто это:** Видел вас раньше, рассматривает, есть возражения.

**Откуда приходит:** Ретаргетинг, повторные визиты, email последовательность.

**С чего начинать:**
- С преимуществ именно перед конкурентами
- Со снятия конкретных возражений
- С эксклюзивных бонусов/условий

**Структура:**
```
1. Hero: Ваше ключевое преимущество/отличие
2. FAQ как антивозражения (конкретные барьеры этой аудитории)
3. Расширенные кейсы с деталями
4. Сравнительная таблица (вы vs альтернативы)
5. Специальный оффер или гарантия
6. Urgency/scarcity trigger
7. CTA
```

---

### Уровень 5: MOST AWARE (готов купить, нужен толчок)

**Кто это:** Лоялен, ждёт правильного момента или предложения.

**Откуда приходит:** Email-лист, прямые поиски по бренду.

**С чего начинать:**
- Сразу с оффера
- Минимум убеждения — максимум деталей

**Структура:**
```
1. Hero: Оффер/акция/дедлайн
2. Что конкретно получают (детальный breakdown)
3. Цена и условия
4. Бонусы за решение сейчас
5. Urgency (время/количество)
6. CTA
```

## 5. Нишевые структурные скелеты

### 5.1 SaaS (free trial → paid)

**Аудитория:** solution-aware, сравнивает варианты
**Фреймворк:** AIDA + FAB

```
1. Hero: [Outcome] без [боль/friction] — Product Screenshot
2. Social proof лента: логотипы компаний или "X пользователей"
3. Как это работает: 3 шага (Setup → Use → Win)
4. Bento-grid фичей: каждая карточка = Feature → Benefit
5. Детальный кейс с числами
6. Pricing: 3 тира, средний выделен, toggle monthly/annual
7. FAQ: топ-6 вопросов перед покупкой
8. Final CTA + "No credit card required"
```

**Примеры с реальными URL:**
- linear.app — hero: "Linear is a better way to build software" + немедленный product UI
- notion.so — "Write, plan, share. With AI" + demo embed в hero
- vercel.com — terminal-style hero, performance metrics как социальное доказательство

---

### 5.2 Онлайн-курс / образовательный продукт

**Аудитория:** problem-aware → solution-aware
**Фреймворк:** PASTOR или StoryBrand

```
1. Hero: [Специфичный outcome] за [срок] — для [кого]
2. "Для кого этот курс" — квалификация (3-5 портретов)
3. Боль + агитация (почему сейчас критично)
4. Трансформационная история студента (до/после)
5. Программа: модули с конкретными темами
6. Кто ведёт: bio автора + credentials + "почему я"
7. Детальные отзывы (трансформационные, не "хороший курс")
8. Pricing: тарифы + что входит в каждый
9. Гарантия возврата
10. FAQ: возражения (успею ли, подойдёт ли, как доступ)
11. Final CTA
```

**Принцип:** Hero-ритм = "outcome + instructor trust + curriculum proof + specific testimonials + final CTA"

---

### 5.3 Консалтинг / услуги

**Аудитория:** problem-aware, доверие = главный барьер
**Фреймворк:** StoryBrand + 4Ps

```
1. Hero: [Проблема пользователя] → [Результат с нами] (не "мы лучшие")
2. Empathy: "Мы понимаем как это" (специфика их ситуации)
3. Наш подход: 3-4 шага что мы делаем
4. Кейсы: 2-3 с конкретными числами и именами
5. Кто мы: команда + credentials (доверие, не самореклама)
6. Что вы получите: deliverables в деталях
7. Testimonials: статус + цитата + результат
8. Pricing или "Запишитесь на бесплатный аудит"
9. FAQ
10. CTA: конкретное следующее действие (звонок/аудит/форма)
```

---

### 5.4 Personal brand / эксперт

**Аудитория:** problem-aware + brand-aware (знают эксперта из контента)
**Фреймворк:** StoryBrand + Authority-first

```
1. Hero: [Трансформация которую создаёт] — минимум про себя
2. "Для кого": аудитория узнаёт себя
3. История эксперта: не CV, а путь + поворотный момент
4. Метод/подход: уникальный механизм
5. Результаты: кейсы учеников/пользователей
6. Продуктовая линейка или один главный оффер
7. В медиа / credentials
8. CTA: self-assessment quiz или консультация
```

---

### 5.5 Mobile App

**Аудитория:** unaware → problem-aware, случайный трафик
**Фреймворк:** BAB + AIDA

```
1. Hero: App store badge + [Одна фраза что делает] + Screenshot
2. Проблема в 3 словах
3. Решение — как работает (3 экрана-шага)
4. Ratings: звёзды + число отзывов
5. Feature карточки: 3-4 ключевых функции
6. Social proof: пресса / featured in
7. Download CTA (две кнопки: App Store + Google Play)
```

---

### 5.6 E-commerce — один продукт

**Аудитория:** solution-aware или most-aware
**Фреймворк:** BAB или AIDA

```
1. Hero: Продуктовое фото + headline = главная выгода + цена + CTA
2. "Почему этот" — 3-4 differentiators
3. Lifestyle фотографии: продукт в использовании
4. Ingredients/specs: FAB-формат
5. Reviews: rating + конкретные отзывы с именами/фото
6. Urgency: наличие, скидка, дедлайн
7. Guarantee + возврат
8. Final CTA
```

---

### 5.7 B2B Lead Magnet

**Аудитория:** unaware → problem-aware
**Фреймворк:** PAS (мягкий) → QUEST

```
1. Hero: [Получи бесплатно: конкретный артефакт] что решает [боль]
2. Для кого (квалификация)
3. Что внутри: конкретный preview/glimpse
4. Почему это ценно: 3 bullet outcomes
5. Форма: максимум email + имя
6. Social proof: X компаний уже используют
7. CTA: скачать/получить
```

**Правило:** форма с 3+ полями теряет 60% конверсии vs. форма с 1 полем.

---

### 5.8 Event / вебинар / запуск

**Аудитория:** problem-aware → solution-aware
**Фреймворк:** PAS + urgency

```
1. Hero: [Конкретное обещание] за [срок вебинара] + дата/время
2. Для кого: 3-5 типов участников с их болью
3. Что получишь: bullet takeaways (конкретные, не "узнаешь о X")
4. Кто ведёт: краткое bio с credentials
5. Testimonials прошлых участников
6. Urgency: "осталось X мест" / дедлайн регистрации
7. FAQ
8. CTA: "Занять место" (не "Зарегистрироваться")
```

**Данные:** 73% B2B маркетологов говорят вебинары = лучший источник leads (источник: my-outreach.com).

---

### 5.9 Информационный продукт / presell page

**Аудитория:** problem-aware → unaware (холодный трафик через контент)
**Фреймворк:** Star-Chain-Hook + PASTOR

```
1. Hook: Big Idea (провокационный тезис или counterintuitive fact)
2. "Меня зовут X и это изменило мою жизнь" (story начало)
3. Почему привычный подход не работает (reframe)
4. Новый механизм: уникальный способ думать об этом
5. История трансформации (детальная, 500+ слов)
6. Что вы узнаете: curriculum в деталях
7. Кейсы других
8. Оффер + stacking (bonus stack)
9. Гарантия
10. FAQ
11. CTA (несколько по странице)
```

---

### 5.10 Community / мемберсайт

**Аудитория:** solution-aware, ищет принадлежность
**Фреймворк:** Identity-based + StoryBrand

```
1. Hero: "Join [identity] who [shared goal]" + превью коммьюнити
2. Preview: скриншоты/gif реальных обсуждений
3. Трансформационные истории участников (до/после участия)
4. Что получишь: пространства, события, контент
5. Кто создал: история фаундера + миссия
6. Pricing: 2-3 тира максимум, ценность до цены
7. FAQ: частые вопросы об отмене, доступе, форматах
8. CTA: "Стать участником" (не "Купить доступ")
```

### Промт 2. Hormozi Grand Slam Offer — структура под высокочековый оффер

```

6. FULL OFFER BLOCK — текст блока оффера для лендинга (готовый к копированию)

Принцип: каждый элемент двигает хотя бы один рычаг Value Equation.
```

### Промт 6. Лендинг через JTBD — структура от работы которую нанимают

```
Ты применяешь Jobs-to-be-Done методологию для построения лендинга.

ДАННЫЕ:
- Продукт: [что]
- Контекст: [когда и почему люди ищут этот продукт]
- Four Forces (заполни):
  PUSH: [что толкает от текущей ситуации — "устал от..."]
  PULL: [что притягивает к новому — "хочу..."]  
  ANXIETY: [страхи и сомнения — "боюсь что..."]
  HABIT: [инерция — "привык к..."]

СОЗДАЙ СТРУКТУРУ ЛЕНДИНГА:

1. Hero — начинается с Push ("Устал от X?")
2. Desire — усиливает Pull ("Представь Y...")
3. Ease section — снимает Anxiety ("Не нужно менять всё. Просто...")
4. Quick win — борется с Habit ("Первый результат через 48 часов")
5. Social proof: люди которые были в такой же ситуации
```

## 9. Anti-patterns — чего избегать

| Ошибка | Данные | Исправление |
|--------|--------|-------------|
| Carousel с отзывами | < 1% видят второй отзыв | Статичные testimonials или grid |
| Форма с 9+ полей | Конверсия 3.6% vs 10.1% у 3-польной | Максимум 3 поля, остальное после |
| Competing CTAs | Два primary CTA режут друг друга | Один primary + один secondary (less prominent) |
| Заголовок про себя ("Мы лидеры...") | Аудитория не доверяет | Заголовок про них и их результат |
| Hero без product visual | Абстракция не продаёт | Скриншот / GIF / embedded demo |
| Длинная navigation в header | 16% потеря primary CTA кликов | Sticky минимальный nav или убрать совсем |
| Gradient/animated CTA кнопки | -4% конверсия vs solid high-contrast | Высококонтрастная solid кнопка |
| "Submit" на кнопке | Passivе language | "Получить [конкретную вещь]" |
| Второй person CTA ("Start Your Trial") | -14% vs first-person | "Start My Free Trial" |
| Общий generic CTA | -31% vs specific outcome | "Get My Website Audit" не "Get Started" |

## 10. Quick-reference для Designer

**Вопрос: "Мне нужен лендинг для онлайн-курса, аудитория частично осознанная"**

Диагностика: аудитория = solution-aware (знают что курсы по теме существуют, выбирают).

Фреймворк: **PASTOR** (длинная история трансформации убирает сомнения).

Структура:
```
1. Hero: [Outcome за срок] — для [кого конкретно]
2. "Для кого" — квалификация (3 портрета)
3. Боль + Amplify (что будет через год без изменений)
4. История трансформации студента (300-400 слов, с именем и числами)
5. Программа модулей (с конкретными темами, не абстрактными)
6. Bio автора (путь + поворотный момент + credentials)
7. Testimonials (трансформационные: было/стало/как долго)
8. Pricing: 2-3 тира + гарантия возврата
9. FAQ: 6-8 реальных возражений
10. Final CTA
```

Промт: используй промт #4 из раздела 6.

Референсы: kajabi.com, maven.com, любой хороший course creator лендинг на lapa.ninja → категория "education".
