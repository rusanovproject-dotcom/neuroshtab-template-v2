# design-critique — самокритика перед отдачей

## Когда запускать
- ВСЕГДА перед отдачей готового промта пользователю (самопроверка)
- Когда пользователь прислал результат и жалуется «не то»
- Когда хочется критически оценить свой черновик

## 11 blacklist-паттернов AI-slop (из gstack/design-review, MIT)

1. **Фиолетовый/индиго по умолчанию** — покажи конкретный hex из Brand Book, а не "purple"
2. **3-колоночная сетка преимуществ** — клише, предложи Bento или Z-pattern
3. **Emoji в заголовках** — режет премиальность, удали
4. **Centered everything** — без ритма. Используй F-pattern, асимметрию
5. **Синий градиент-голограмма** — стоковая AI-эстетика, замени на палитру бренда
6. **Стоковые улыбающиеся люди** — явный запрет в промте
7. **"Lorem ipsum" или generic копи** — всегда real content
8. **Glossy corporate vibe** — если не банк, убрать
9. **Inter/Helvetica без повода** — Brand Book диктует шрифт, не дефолт
10. **"Innovative / cutting-edge / seamless"** в тексте — AI-слоп лексика
11. **"Smooth animations" без цифр** — замени на ms и easing-кривые (см. motion-principles.md)

## Шаги

1. Прочитай готовый промт / результат
2. Прогони по 11 паттернам: какие присутствуют?
3. Выставь оценки:
   - **Design Score** (A-F): насколько хорошо использованы Brand Book + фреймворки + стек
   - **AI Slop Score** (A-F): инверсия — A значит нет AI-slop
4. Если найдены паттерны из blacklist → переписать промт с конкретикой
5. Только после A/A или B/B по обоим — отдавать пользователю

## Output
Краткий отчёт: оценки + список найденных slop-паттернов + исправленный промт.
