# Creative Brief — универсальный визуальный бриф

Бриф — это структурированный артефакт, в котором зафиксирована задача на дизайн.

**Когда запускать этот скилл:**
- **Для внешнего дизайнера / фрилансера** — бриф заменяет ТЗ, человек-исполнитель получает формализованную задачу.
- **Для сложной многокомпонентной задачи в AI** (серия из 5+ обложек, лендинг с 7+ секциями, презентация 10+ слайдов) — бриф как промежуточный шаг перед промтом.
- **Когда пользователь сам попросил** «зафиксируй ТЗ», «собери бриф».

**Когда НЕ запускать (default — делаешь сам):**
- Пользователь хочет одну картинку / лендинг / обложку → ты **сам делаешь** через Claude Code артефакт или Claude Design. Бриф не нужен, этот скилл не запускаешь. Default — меньше шагов, больше результата.
- Если пользователь попросил промт для внешнего AI (v0 / Midjourney) — запускай `claude-design-prompt` или `claude-code-artifact` (опция), не creative-brief.

**Правило разведения:** `creative-brief` — для **внешнего человека-дизайнера** (фрилансер / Figma вне пайплайна). Для работы через наш офис (сам делаешь в Claude Code / Claude Design) — бриф не нужен, иди делать.

## Вход

- **Что делаем** — тип дизайна (из `knowledge/design-catalog.md`)
- **Для чего** — контекст (пост / лендинг / реклама / презентация)
- **Brand Book** — `knowledge/brand/{project}/brand-book.md` (обязателен)
- **Текст на визуале** — если есть, от пользователя или копирайтера. Сам не придумываешь.

## Шаги

### 1. Собери данные через диалог

Спрашивай по одному если чего-то не хватает:
- Тип дизайна + формат
- Куда пойдёт (платформа, размещение)
- Главный посыл / что должен человек понять за 3 секунды
- Текст (если есть)
- Есть ли особые требования (например, "должно быть читаемо на тёмном фоне за 2 метра")

Если Brand Book не создан — **останови и запусти `brand-onboarding`**.

### 2. Собери концепцию

В голове или черновиком:
- **Настроение** — 3-5 слов
- **Главный элемент** — что в центре кадра
- **Композиция** — минимализм / симметрия / диагональ / сетка
- **Палитра** — hex из Brand Book (основной / акцент / фон / текст)
- **Шрифты** — из Brand Book
- **Запреты** — из Brand Book + специфика задачи

### 3. Заполни шаблон брифа

Используй `office/agents/designer/templates/design-brief.md`. Или выведи бриф прямо в чат если он нужен быстро.

**Структура:**

```markdown
# Визуальный бриф: [название задачи]

**Дата:** YYYY-MM-DD
**Проект:** [slug]
**Тип дизайна:** [из каталога]
**Формат:** [размер px, ratio]
**Куда пойдёт:** [платформа / контекст]

## Концепция
- Настроение: [3-5 слов]
- Главный элемент: [что в центре]
- Композиция: [минимализм / симметрия / и т.д.]
- Свет / атмосфера: [описание]

## Стиль (из Brand Book)
- Палитра:
  • Основной: #XXX ([название словами])
  • Акцент: #XXX
  • Фон: #XXX
  • Текст: #XXX
- Шрифты:
  • Заголовки: [название] (Google Fonts ссылка)
  • Текст: [название] (ссылка)

## Текст на визуале
[Если есть — написать дословно. Максимум N слов. Если нет — "без текста, пространство оставить под наложение".]

## Запреты
- [Из Brand Book]
- [Специфика задачи]

## Стек / инструмент
[Claude Design / Claude Code / фрилансер / Figma]

## Референсы
[Если были присланы — вставить ссылки]
```

### 4. Что дальше

После брифа — выбор:
- **Если для фрилансера** — отдай пользователю, он копирует и отправляет
- **Если Claude Design** — запусти скилл `claude-design-prompt` с брифом как вводными
- **Если Claude Code** — запусти скилл `claude-code-artifact`

Спроси пользователя: *"Бриф собран. Куда дальше — я напишу промт для Claude Design / Claude Code, или это отдаёшь фрилансеру?"*

## Правила

- **Бриф не обязателен для каждой задачи.** Для простых вещей (одна обложка) можно сразу писать промт. Бриф нужен если:
  - Задача сложная (многокомпонентная, несколько экранов)
  - Отдаём фрилансеру
  - Пользователь хочет зафиксировать ТЗ перед тем как "покупать" дизайн
- **Не вываливай шаблон сразу.** Сначала диалог (шаг 1), потом заполненный бриф.
- **Не придумывай палитру если Brand Book уже есть.** Копируй hex как есть.

## Выход

Заполненный бриф в markdown — либо в чат, либо сохранённый в `projects/{slug}/artifacts/briefs/YYYY-MM-DD-{task}.md`.
