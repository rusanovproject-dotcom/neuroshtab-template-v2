# Шаг 03. Генерация jobStory

**⛔ ПРАВИЛО КОММУНИКАЦИИ: пользователь видит ТОЛЬКО результат с вопросом. НЕ видит: внутренние рассуждения, промежуточные проверки, термины JTBD. Визуальное форматирование обязательно — заголовки, выделения, не полотно текста.**

## Цель

Сгенерировать полную jobStory для сегмента по формуле «Когда / Хочу / Чтобы».

**Связь назад:** шапка сегмента (шаг 02) — берём Big Job сегмента и критерии. Big Job сегмента = блок «Чтобы» (или его основа).
**Связь вперёд:** jobStory → Точки А/Б (шаг 05), граф работ (шаг 06), Consideration Set (шаг 07), Оффер (скилл №6), Лендинг (скилл №10). Это ядро — из него потом строится ВСЁ.

---

## Что СКАЗАТЬ пользователю (обязательно)

**ПЕРЕД результатом:**
> «Сейчас опишем конкретного типичного клиента из этого сегмента: в какой он ситуации, что подтолкнуло к действию, чего хочет и зачем. Это ядро — из него потом собираются лендинг, оффер, реклама.»

**ПОСЛЕ результата (вместе с проверкой из шага 04):**
> «Вот описание клиента. На что обрати внимание: 1) Узнаёшь своих людей в блоке "ситуация"? 2) Критерии — это то, по чему клиенты реально оценивают? 3) Итоговая цель — ради этого они приходят? Если что-то не бьётся — скажи.»

**НЕ ПОКАЗЫВАЙ:**
- Термин «jobStory» — скажи «описание клиента»
- Свои 9 шагов раскопки
- Термин «Core Job» — скажи «то что продукт реально делает»
- Термин «декомпрессия» — не используй вообще

---

## Полная формула работы

```
КОГДА:
- контекст: [конкретная жизненная ситуация, не обобщение]
- психологические особенности: [что за человек, что знает, чего боится, внутренний конфликт]
- прошлый опыт: [что уже пробовал, с каким результатом]
- активирующее знание: [что узнал нового, что изменило картину мира] (опционально)
- триггер: [конкретное событие, запускающее действие — НЕ «задумался»]

ХОЧУ: [глагол + существительное + КАК (критерии)]
- критерий 1: [как поймёт что результат получен]
- критерий 2
- критерий 3

ЧТОБЫ: [работа выше уровнем — зачем в жизни/бизнесе]
- чувствовать себя по-другому: [позитивные эмоции]
- не чувствовать: [негативные эмоции, от которых хочет избавиться]
```

---

## Алгоритм раскопки джобы (9 шагов)

### Шаг 1. Собрать список работ
«Какой результат хотел получить?» → 3-5 формулировок на разном уровне.

Не торопись с первой формулировкой. Из фактуры обычно видно несколько уровней:
- Уровень инструмента: «научиться настраивать рекламу»
- Уровень результата: «начать зарабатывать на фрилансе»
- Уровень жизни: «не зависеть от мужа финансово»

Запиши все — потом определяем какой Core, какой Big.

### Шаг 2. Поднять до Big Job через «зачем»
Итерация вверх: «Зачем тебе [работа], чтобы что?» — пока не выйдешь за границы продукта.

**Пример пошаговой раскопки:**
```
«Научиться настраивать Яндекс Директ»
→ Зачем? → «Чтобы брать клиентов на фрилансе»
→ Зачем? → «Чтобы зарабатывать 30-50к удалённо»
→ Зачем? → «Чтобы не зависеть от мужа и чувствовать себя самостоятельной»
→ Зачем? → «Чтобы быть уверенной что если что-то случится — я не пропаду»
```

Где остановились? «Не зависеть от мужа» — продукт напрямую не делает, но ведёт к этому = Big Job. «Зарабатывать 30-50к удалённо» — продукт это делает (навык + поиск клиентов) = Core Job.

**Пример раскопки (B2B, упаковщик):**
```
«Упаковать продукт для лендинга»
→ Зачем? → «Чтобы лендинг конвертил без моего участия»
→ Зачем? → «Чтобы масштабировать школу, не увеличивая личное время»
→ Зачем? → «Чтобы пробить 5 млн/мес и нанять команду»
```

Core = «Упаковать продукт так, чтобы лендинг конвертил» (делаем полностью).
Big = «Масштабировать до 5+ млн/мес» (вносим вклад, но не полностью).

### Шаг 3. Граница Core vs Big
«Наш продукт полностью это делает?» Да → Core. Нет → Big, спускайся.

**Два признака булшита:**
- Обобщающие слова: «комфорт», «успех», «свобода», «трансформация» — если в Core Job, ты залез в Big
- Невыполнимость гарантии: «ты заработаешь миллион» — если продукт не может это гарантировать, это Big

**Ошибка снизу — Core Job = инструмент:**
❌ «Освоить навык настройки рекламы» — инструмент, не результат.
Тест: «а ЗАЧЕМ навык?» → «чтобы зарабатывать» — вот Core Job.

**Ошибка сверху — Core Job = Big Job:**
❌ «Стать финансово свободной» — курс полностью не делает.
Тест: «мы это ПОЛНОСТЬЮ делаем?» → нет → Big Job.

**Правильная Core Job = между инструментом и Big Job.** Результат в жизни, который продукт полностью выполняет.

### Шаг 4. Декомпрессия критериев
«По каким критериям поймёшь что результат получен?»

**Пример 1 (инфобиз):**
«Восполню компетенции в маркетинге» → «Какие?» → «Смогу пересобирать упаковку сам / понимать кого нанимать / использовать AI по максимуму» — три критерия вместо одного абстрактного.

**Пример 2:**
«Освою профессию» → «Как поймёшь?» → «Сама настрою рекламу клиенту / знаю где искать клиентов / есть первый кейс»

**Пример 3 (B2B):**
«Хочу эффективный маркетинг» → «Эффективный — это какой?» → «Заявки ≤500 руб / конверсия лендинга >3% / ROI положительный за 3 мес»

### Шаг 5. Контекст
«В какой ситуации находился?» — конкретика, не обобщение.
❌ «Предприниматель в сфере онлайн-образования»
✅ «В декрете, ребёнку 1.5 года, живёт на деньги мужа, раньше работала медсестрой»

### Шаг 6. Триггер
Конкретное событие, не «задумался».
❌ «Задумалась о развитии»
✅ «Ребёнок пошёл в сад, появилось 3 часа — увидела рекламу у блогера»

### Шаг 7. Позитивные эмоции
«Как хочешь себя чувствовать?» — уверенность, самостоятельность, контроль, гордость, спокойствие.

### Шаг 8. Негативные эмоции
«Что испытываешь пока не выполнил?» — тревога, неполноценность, обесценивание, страх упустить, стыд.

### Шаг 9. Собрать в формулу
Все компоненты → «Когда / Хочу / Чтобы». Проверить что ничего не потеряно.

## Декомпрессия абстрактных слов

ЗАПРЕЩЕНЫ в формулировке. Каждое = контейнер, скрывающий 3-5 подсегментов.

Вопросы: «А [слово] — это какое?» / «По каким критериям поймёшь?» / «Какой считаешь [слово]? Какой НЕ считаешь?»

**Пример 1:** «качественная мебель» → 20 лет прослужит / немецкая фурнитура / выдержит арендаторов. Три сегмента за одним словом.

**Пример 2 (инфобиз):** «восполню компетенции в маркетинге» → упаковка / трафик / воронки / аналитика? Это 4 разных продукта.

**Пример 3:** «хочу развиваться» → в профессии? в доходе? личностно? Три Big Job.

**Правило:** слово, которое разные люди понимают по-разному — декомпрессируй ДО генерации.

## Паттерны B2C vs B2B

### B2C
- КОГДА: жизненная ситуация (декрет, выгорание, переезд)
- ХОЧУ: эмоционально заряженный, критерии из жизни
- ЧТОБЫ: функциональный + эмоциональный уровни
- Обязательно: «чувствовать себя по-другому» / «не чувствовать»

### B2B
- КОГДА: контекст роли (должность, выручка, команда)
- ХОЧУ: операционные критерии (цифры, метрики, сроки)
- ЧТОБЫ: бизнес-результат + карьерный/личный
- Обязательно: роль в принятии решения

**Антипаттерн:** B2C без эмоций (сухо) или B2B без цифр (размыто).

---

## Связь ХОЧУ ↔ ЧТОБЫ

ОБЯЗАТЕЛЬНО на разных уровнях. Если одинаковые — не раскопал.

❌ ХОЧУ: зарабатывать удалённо. ЧТОБЫ: иметь удалённый доход — перефраз
✅ ХОЧУ: зарабатывать удалённо. ЧТОБЫ: не зависеть от мужа — другой уровень

❌ ХОЧУ: упаковать продукт. ЧТОБЫ: иметь хорошую упаковку — перефраз
✅ ХОЧУ: упаковать продукт. ЧТОБЫ: масштабировать школу — другой уровень

---

## Принцип антиинфоцыганства

- Core Job → прямые обещания: «ты освоишь», «ты соберёшь»
- Big Job → через цель: «чтобы», «что позволит»
- Красный флаг: «ты заработаешь миллион» — продукт этого не гарантирует
- Красный флаг: «трансформация жизни» — продукт даёт навык, не трансформацию

## Примеры jobStory

### B2C хорошая:
```
КОГДА:
- контекст: в декрете, ребёнку 1.5 года. Своего дохода нет — живу на деньги мужа.
  Раньше сама обеспечивала, а сейчас на всё прошу.
- психология: чувствую что «деградирую». Боюсь что после декрета никому не нужна.
  Привыкла с 16 лет сама зарабатывать — зависимость унижает.
- прошлый опыт: перепродажа на Авито — максимум 5к. Курс маркетплейсов — «не доучилась»
- триггер: ребёнок пошёл в сад, появилось 3 часа. Увидела рекламу у блогера.

ХОЧУ: начать зарабатывать удалённо, чтобы:
- не выходить из дома (ребёнок маленький)
- по шагам, с поддержкой — я не из маркетинга
- с реальными перспективами дохода в 2-3 месяца

ЧТОБЫ: вернуть финансовую независимость
- чувствовать: «это мои деньги, я сама заработала»
- не чувствовать: стыд от просьб, тревогу «а что если с мужем что-то случится»
```

### B2B хорошая:
```
КОГДА:
- контекст: онлайн-школа по трафику, 500к-2млн/мес, команда 3-5
- психология: технарь, не умею упаковывать, стесняюсь «продавать»
- триггер: конкурент с худшим продуктом продаёт в 3 раза больше

ХОЧУ: упаковать продукт так, чтобы:
- клиент за 30 секунд понимал зачем ему это
- я мог делегировать продажи менеджеру
- лендинг работал без моего участия

ЧТОБЫ: масштабировать школу до 5+ млн/мес не увеличивая время на продажи
```

### Плохая:
```
КОГДА: предприниматель в онлайн-образовании
ХОЧУ: улучшить маркетинг
ЧТОБЫ: увеличить продажи и чувствовать уверенность
```
Нет контекста, критериев, триггера. «Увеличить» абстрактно. «Уверенность» — эмоция-следствие.

## Если не хватает фактуры

НЕ выдумывай. Помечай «требует уточнения». Вопросы пользователю — на доказательства:

✅ «Был клиент, который купил именно потому что [причина]? Расскажи про него.»
✅ «Что клиенты говорят после покупки — что оказалось самым ценным?»
❌ «Опиши целевую аудиторию» — абстракция
❌ «Какие боли у клиентов?» — маркетинговый штамп

1. Собери компоненты по 9 шагам раскопки
2. Декомпрессируй абстрактные слова (минимум 3 проверки)
3. Собери в формулу Когда/Хочу/Чтобы
4. Проверь: Core Job = результат, не инструмент? Core ≠ Big?
5. Калибруй по эталонам (references/etalon-jobstories.md)
6. **НЕ показывай пользователю — переходи к шагу 04 (проверка)**
