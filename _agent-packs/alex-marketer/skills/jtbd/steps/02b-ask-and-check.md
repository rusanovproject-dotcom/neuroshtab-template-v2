# Шаг 02b. Задать вопросы пользователю и проверить ответы

**⛔ ПРАВИЛО КОММУНИКАЦИИ: пользователь видит ТОЛЬКО вопросы и уточняющие переспросы. НЕ видит: классификацию ответов, проверочные таблицы, внутренние рассуждения, промежуточные выводы.**

## Где ты сейчас (контекст)

Ты определяешь **главную задачу в жизни** (Big Job) для конкретного сегмента. В фактуре не хватает информации — клиенты говорят о запросах к продукту (процессы, средства, эмоции), но не о том, чего хотят достичь В ЖИЗНИ.

Определение Big Job состоит из трёх подшагов:
1. **02a** (сделано) — извлёк запросы, классифицировал, вывод: данных не хватает
2. **02b (ТЫ ЗДЕСЬ)** — задать пользователю вопросы «зачем? чтобы что?», проверить ответы. **Может повторяться 2-3 раунда** — это нормально.
3. **02c** — сформулировать Big Job из проверенных целей

Результат этого подшага: в черновике записаны проверенные цели клиентов (с моментом достижения, за пределами продукта). Достаточно для формулировки Big Job.

**Связь назад:** классификация из шага 02a (в `big-job-черновик.md`).
**Связь вперёд:** проверенные цели → шаг 02c (формулировка Big Job).

---

## Алгоритм

### 1. Перечитать черновик

**tool call: view** `big-job-черновик.md`

Найти блоки «запросы клиентов» и «классификация» по этому сегменту. Определить: по КАКИМ клиентам не хватает информации (те, у кого тип = процесс/средство/эмоция/Core Job).

### 2. Сформировать вопросы

**Техника JTBD-методологии — итерация «зачем? / чтобы что?»:**

Клиент говорит «хочу X» → «Зачем ему X? Чтобы что?»
Итерируем вверх пока не выйдем ЗА ГРАНИЦЫ продукта в жизнь клиента.
Граница: «Продукт может это полностью сделать?» ДА → ещё раз «зачем?». НЕТ → это Big Job.

Адаптация: спрашиваем не клиента, а владельца продукта ПРО клиента.

### 3. Задать вопросы в чате

Формат — привязано к конкретным цитатам:

> «Мне нужна твоя помощь по этой группе клиентов.
>
> 1. [Имя] говорит: "[цитата]". А зачем ему/ей это? Чтобы что в итоге?
> 2. [Имя] говорит: "[цитата]". А зачем? Чтобы что?
> ...»

**Правила:**
- 2-4 клиента за раз, не больше
- Вопрос привязан к цитате, не абстрактный
- Формулировка: ТОЛЬКО «зачем?» или «чтобы что?» — две техники JTBD-методологии
- Можно подсказать направления из контекста клиента
- НЕ давать примеры из конкретной ниши
- НЕ использовать термины JTBD

**Жди ответ. Не переходи дальше пока не получишь.**

### 4. Записать ответ в черновик

**tool call: str_replace** → дописать в `big-job-черновик.md`:
```
=== СЕГМЕНТ: [название] — ответ пользователя (раунд [N]) ===
[Имя]: «[дословный ответ пользователя]»
[Имя]: «[дословный ответ пользователя]»
```

### 5. Проверить каждый ответ

**tool call: view** `big-job-черновик.md` — перечитать только что записанные ответы.

По каждому ответу — проверка:

1. **Процесс?** (нет конечной точки) → переспросить: «А к какому конкретному результату это должно привести?»
2. **Средство?** → переспросить: «А [средство] — зачем? Чтобы что?»
3. **Эмоция?** → переспросить: «А что конкретно должно измениться в жизни, чтобы он/а это почувствовал/а?»
4. **Продукт может полностью?** → это Core Job, переспросить: «А зачем ему это? Чтобы что за пределами продукта?»
5. **Есть момент достижения? Продукт не может полностью?** → ✅ кандидат на Big Job

### 6. Записать проверку в черновик

**tool call: str_replace** → дописать:
```
=== СЕГМЕНТ: [название] — проверка ответов (раунд [N]) ===
[Имя]: «[ответ]» → [тип] — [почему] → [✅ кандидат / ❌ переспросить]
[Имя]: «[ответ]» → [тип] — [почему] → [✅ кандидат / ❌ переспросить]
```

## Что дальше

- **Все ответы прошли проверку** (✅ кандидаты на Big Job) → **переходи к шагу 02c**
- **Часть не прошла** → **ещё раунд вопросов** (вернись к пункту 2, но ТОЛЬКО по тем, кто не прошёл). Максимум 3 раунда.
- **После 3 раундов всё ещё не хватает** → работай с тем что есть. Пометь Big Job как 🔴 AI-генерация (слабая база). Переходи к 02c.

---

## ЗАПРЕТЫ

- **НЕ формулируй Big Job на этом шаге.** Только сбор и проверка целей. Формулировка — в 02c.
- **НЕ принимай размытый ответ как финальный.** Если ответ не прошёл проверку — переспроси. Не бойся задавать 2-3 раунда вопросов.
- **НЕ переспрашивай по ВСЕМ клиентам заново.** Только по тем, у кого ❌.
- **НЕ придумывай цели за пользователя.** Если не знает — запиши «нет данных», пометь 🔴.

## Что НЕ ПОКАЗЫВАТЬ пользователю

Пользователь видит ТОЛЬКО вопросы (пункт 3) и уточняющие переспросы. Не видит: классификацию ответов, проверочные таблицы, внутренние рассуждения, термины JTBD, номера раундов.
