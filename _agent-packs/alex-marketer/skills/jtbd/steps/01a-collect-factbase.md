# Шаг 01a. Собрать фактуру

**⛔ ПРАВИЛО КОММУНИКАЦИИ: весь этот шаг выполняется МОЛЧА. Пользователь НЕ видит: карточки клиентов, цитаты, промежуточные выводы, сколько файлов прочитал. Пользователь видит ТОЛЬКО короткое сообщение в начале и в конце.**

## Цель

Прочитать ВСЮ фактуру из проекта. Вытащить из каждого источника ключевое про клиентов: кто, ситуация, запрос, что получил (если есть). Записать в черновик.

Это подготовительный шаг. Результат — черновик с выжимкой из всех источников. На следующем шаге (01b) — группировка в кластеры.

**Связь вперёд:** черновик → шаг 01b (группировка).

---

## Что СКАЗАТЬ пользователю (одно сообщение)

> «Сейчас прочитаю всю фактуру — все файлы, которые ты загрузил. Это займёт время, но мне важно ничего не пропустить. Когда закончу — покажу результат.»

---

## Алгоритм

### 1. Составить список всех источников

**tool call: view** — посмотреть содержимое папки проекта.

**НЕ начинай читать файлы. Сначала — список.** Запиши в черновик ВСЁ что видишь:

**tool call: create_file** `сегменты-черновик.md`:
```
=== ИСТОЧНИКИ ФАКТУРЫ ===
1. [файл или папка] — тип: [диагностика / кейс / описание / анкета / отзыв / другое]
2. [файл или папка]
3. ...
Всего источников: [N]
```

Если видишь папки — загляни внутрь (`view`), допиши содержимое в список. Если видишь архивы — распакуй (`bash`), допиши. Если что-то непонятное — всё равно запиши, разберёшься при чтении.

### 2. Читать порциями, записывать после каждой

Читай по 5-7 файлов за раз. **Если файл больше 500 строк — читай его порциями по 500 строк (view с range: [1,500], [501,1000], [1001,1500] и т.д.). НЕ пытайся прочитать весь файл за один вызов. Частичное чтение = мусор на выходе.**

После КАЖДОЙ порции — два обязательных действия:

**Действие А.** Записать в черновик что нашёл:

**tool call: str_replace** → дописать в `сегменты-черновик.md`:
```
=== ПОРЦИЯ [N]: [какие файлы] ===
- [кто]: ситуация — [кратко], желаемый результат — «[что хочет получить — дословно из карточки]», получил — «[если есть, из кейса]»
- [кто]: ...
```

**Поле «желаемый результат» ОБЯЗАТЕЛЬНО.** Без него невозможно группировать по запросу на шаге 01b. Берётся из полной карточки клиента (поле «Желаемый результат»), не выдумывается. Если в карточке нет — записать «не указано».

**Действие Б.** Перечитать черновик:

**tool call: view** `сегменты-черновик.md`

Посмотреть список источников из пункта 1. Какие ещё НЕ прочитаны? → следующая порция. Все прочитаны? → пункт 3.

**ЧИТАЙ ФАЙЛЫ ЦЕЛИКОМ.** Не «первые 30-40 строк». Весь файл.

### 3. Финальная сверка

**tool call: view** `сегменты-черновик.md`

Пройдись по списку источников (пункт 1). Каждый файл/папка должен быть упомянут хотя бы в одной порции. Если что-то пропущено — дочитай сейчас.

### 4. Сказать пользователю

> «Прочитал всю фактуру: [N] источников, [N] клиентов. Перехожу к группировке.»

**Переходи к шагу 01b.**

---

## ЗАПРЕТЫ

- **НЕ пропускай источники.** Все файлы, все папки, все архивы. Без исключений.
- **НЕ читай только «первые строки».** Весь файл.
- **НЕ решай что «хватит, паттерны повторяются».** Читай до конца списка.
- **НЕ показывай пользователю** карточки клиентов, цитаты, промежуточные выводы.
- **НЕ начинай группировку** пока не прочитал всё. Группировка — шаг 01b.
