# Sales Meetings — модуль встреч и созвонов

Опциональное расширение Маркетолога для бизнесов где есть **продающие или диагностические встречи 1-на-1**.

## Когда подключать

✅ Подключай если:
- Делаешь диагностики/продающие созвоны перед сделкой
- Программа high-ticket (с высоким чеком) с разговором перед оплатой
- B2B-продажи через Sales-цикл
- Консалтинг, коучинг, наставничество

❌ Не подключай если:
- Бизнес работает через автоворонку без созвонов
- Контент-маркетинг с прямыми оплатами через Stripe/Tinkoff
- E-commerce, SaaS с self-service подпиской
- Курсы продаются через лендинг без диалога

## Что добавляет в проект

При активации скиллом `/marketer-enable-meetings` копируется в `projects/<main>/`:

```
customers/                          🫁 Печень — база клиентов с историей
├── INDEX.md                        тонкая таблица: 1 строка на клиента
├── refusals/{name}-{date}.md       отказники + причины
└── {name}/                         папка на клиента
    ├── _summary.md                 агрегат last-state + Pain Evolution
    ├── transcripts/
    │   ├── {date}-{type}-raw.md    полный текст
    │   └── {date}-{type}.md        выжимка с таймкодами
    ├── {date}-diag.md              карточка после диагностики (5 полей)
    └── {date}-deal.md              карточка после сделки (8 полей)
```

В `funnel/scripts.md` активируется секция «банк возражений + скрипты».

В `metrics.md` добавляются строки: Diagnostic calls, конверсия Diag→Deal, средняя actuality оффера.

## Регистрируемые скиллы маркетолога

| Скилл | Триггеры | Что делает |
|-------|----------|------------|
| `/marketer-log-deal` | «провёл диагностику с {имя}», «{имя} оплатил», «{имя} отказался», «вот транскрипт» | Создаёт/обновляет карточку клиента, заполняет 5 или 8 полей |
| `/parse-meeting` | «вот транскрипт», «разбери созвон с {имя}» | Парсит транскрипт в 7 артефактов параллельно: карточка, voc, expert-bank, конкуренты, возражения, идеи, метрики |

## 8 полей карточки сделки

1. **Main Pain** — главная боль ЦИТАТОЙ
2. **Main Desire** — главное желание ЦИТАТОЙ
3. **Возражения** — каждое отдельной цитатой
4. **Триггер покупки/отказа** — одна ключевая фраза
5. **Про конкурентов / альтернативы** — что сказал
6. **Оффер дословно** (как был озвучен)
7. **Actuality оффера 1-10** — клиент сам называет цифру + «чего не хватает до 10»
8. **Канал первого касания + Lead time**

После диагностики (без сделки) заполняются 5 полей: Main Pain, Main Desire, Возражения, Про конкурентов, Канал.

## Actuality — главный прибор силы оффера

Средняя actuality < 7 → **оффер требует пересборки**. Маркетолог выносит как проблему №1 в чек-ине.

## Pain Evolution

В `_summary.md` каждого клиента — трек как менялась его боль между встречами. Не перезаписываем, а добавляем колонку «Дата → Pain цитатой → что изменилось».

## Связь с базовым организмом

Транскрипт встречи → парсинг → 7 артефактов:
- Карточка → `customers/{name}/{date}-{type}.md`
- Цитаты ЦА → `audience/voice-of-customer.md`
- Голос владельца → `brand/expert-bank.md`
- Упоминания конкурентов → `brand/competitors.md`
- Возражения → `funnel/scripts.md`
- Идеи → `hypotheses.md → Backlog`
- Метрики → `metrics.md`

## Деактивация

Если решишь что встречи больше не нужны — `customers/` остаётся в проекте как архив (не удаляется). Скиллы перестают триггериться. Снова активировать — `/marketer-enable-meetings`.
