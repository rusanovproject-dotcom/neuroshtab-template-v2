# Guardrails (Security & Safety)

## Безопасность файлов

- **Секреты**: не пиши .env, токены, API ключи в файлы (контекст + логи видимы)
- **Sandbox**: не запускай user-code без sandbox'а
- **Boundaries**: пиши ТОЛЬКО в целевую директорию, не вне её
- **Validation**: проверяй все пути перед write (нет path traversal, noop symlinks)
- **No overwrite**: scaffold.sh не перезаписывает существующие файлы (`if [ ! -f ...]`)

## Лимиты

| Параметр | Лимит | Действие при превышении |
|----------|-------|------------------------|
| Агенты в команде | Max 10 | СТОП: "Нужна Hierarchical архитектура" |
| Warning порог | 7+ агентов | Предупреди пользователя |
| Вложенность папок | Max 3 уровня | Выноси в отдельную подсистему |
| CLAUDE.md размер | Max 200 строк | Вынеси детали в knowledge/ |
| Identity блок | Max 300 токенов | Сократи до формулы |
| Review-Refine петля | Max 3 раунда | Стоп при delta < 0.5 |
| Субагент timeout | Max 10 мин | Kill, используй partial result |

## Целевые директории

НИКОГДА не создавай scaffold в:
- `/` (корень системы)
- `~` (домашняя директория)
- `.claude/` (конфигурация Claude Code)
- Существующий git-репозиторий без подтверждения пользователя

## Lethal Trifecta (Simon Willison)

Никогда не давай агенту все три одновременно:
1. Private data (БД, секреты, персональные данные)
2. Untrusted content (пользовательский ввод, веб-скрейпинг)
3. Exfiltration tools (email, API calls, file upload)
