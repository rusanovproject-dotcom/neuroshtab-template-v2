# Продвинутые паттерны — лайфхаки от практиков

## Стиль написания

### Негация → позитив
LLM борется с отрицанием. "НЕ используй X" активирует концепцию X.
- ❌ "Не создавай новые файлы если существуют"
- ✅ "Переиспользуй существующие файлы когда возможно"
Источник: Brandon Casci, From Chaos to Control

### Правило наращивания
CLAUDE.md растёт ТОЛЬКО из ошибок. Добавляй правило когда Claude накосячил дважды.
Если Claude уже делает правильно без правила — правило не нужно.
Цель: 30-200 строк, идеал 50-100.
Источник: shanraisshan/claude-code-best-practice

### Why > MUST (расширенное)
Claude лучше обобщает когда понимает причину. ALWAYS/NEVER в caps — жёлтый флаг.
Переформулируй: "ALWAYS use TypeScript" → "Используй TypeScript — проект типизирован,
нетипизированный код ломает автодополнение всей команде."

### Третье лицо в descriptions
Description скилла → всегда третье лицо. Claude лучше матчит.
- ❌ "I create dashboards"
- ✅ "This skill creates dashboards for data visualization"

## Архитектурные хаки

### Трёхслойный scope CLAUDE.md
```
~/.claude/CLAUDE.md        → глобальные правила (для всех проектов)
.claude/CLAUDE.md          → проектные правила (для команды)
.claude/local.md           → личные переопределения
```
Приоритет: local > project > global.

### Worktree isolation
Параллельные агенты → каждый в своём worktree.
Изолированная копия репо + своя ветка. Автоочистка после завершения.

### Research + Action в разных контекстах
Не делай всё в одном контексте:
- Explore agent: читает файлы, понимает архитектуру
- Task agent: исправляет код, не читает лишнего
- Reviewer agent: проверяет результат
Каждый специализирован → экономия токенов.

### Git-первый подход
Вместо "не удаляй файлы" (полагаемся на LLM):
- Hook при старте → git snapshot
- Hook перед пушем → diff review
- Hook после завершения → git finish
Детерминировано, не зависит от доброй воли LLM.

## Контекст

### Compaction strategy
- 0-50%: свободная работа
- 50-70%: начинай обращать внимание
- 70-90%: /compact с инструкцией что сохранить
- 90%+: /clear и новая сессия

### Selective compaction
`/compact "Keep database schema, 4 API endpoints, test results. Remove exploration code."`
Сохраняет 95% релевантности, освобождает 77% контекста.

### Одна сессия = одна задача
Feature A → Feature B → обратно к A = засорённый контекст.
/clear между несвязными задачами.

## Мета-паттерны

### /insights еженедельно
Команда `/insights` покажет паттерны ошибок Claude за неделю.
Если видишь паттерн → добавь rule в CLAUDE.md. Loop: ошибка → rule → меньше ошибок.

### Самообучение через файловую систему
Агент научился новому → скажи: "напиши инструкцию для себя на будущее"
→ положи в CLAUDE.md или скилл → следующий запуск уже знает маршрут.

### Hooks вместо инструкций
Форматирование, линтинг, git — в hooks (детерминировано), не в CLAUDE.md (вероятностно).
PreToolUse = гейткипер, PostToolUse = валидация, Stop = финализация.

## Золотые правила (топ-5)

1. CLAUDE.md ≤ 200 строк, растёт только из реальных ошибок
2. Description скилла = "Use when..." (pushy), не "What is..."
3. Hooks для детерминированных вещей, CLAUDE.md для judgment
4. Субагенты для isolated work, главный для координации
5. /compact при 70%, /clear при 90%
