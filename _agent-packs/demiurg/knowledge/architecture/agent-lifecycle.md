# Agent Lifecycle — добавление, изменение, удаление

## Добавить агента в существующую систему

### Чеклист (7 шагов):

1. **Проверь нужность** — нет ли агента с такой же зоной?
   ```
   grep -r "Отвечает за" agents/ --include="*.md"
   ```
2. **Определи тип** → загрузи `data-packs.md` → собери данные (score >= 5)
3. **Создай файлы** по шаблону `agent-design.md`:
   - CLAUDE.md (identity <= 300 tok, уникальный принцип)
   - skills/ (если есть процессы)
   - knowledge/ (если есть данные)
4. **Пропиши связи:**
   - reports_to = кому подчиняется
   - "НЕ отвечает за → передай {кому}"
   - В CLAUDE.md вышестоящего добавь роутинг на нового агента
5. **Обнови AGENTS.md** — добавь строку с описанием
6. **Обнови context.md** — если меняются приоритеты/структура
7. **Запусти validate-office.sh** — проверь что ничего не сломал

### Quality gate:
- validate-office.sh PASS
- Нет пересечений зон
- DRY-RUN: новый агент получает задачу за <= 2 шага

## Изменить агента

### Когда менять:
- Зона ответственности расширилась/сузилась
- Добавился новый скилл
- Изменились данные (обновить knowledge/)

### Чеклист (5 шагов):
1. Прочитай текущий CLAUDE.md агента
2. Внеси изменения (сохраняй identity <= 300 tok)
3. Проверь: не сломались ли ссылки?
4. Обнови AGENTS.md если изменилась роль
5. validate.sh на папку агента

---

## Удалить / архивировать агента

### Чеклист (6 шагов):
1. **Проверь зависимости** — кто ссылается на этого агента?
   ```
   grep -r "agent-name" . --include="*.md"
   ```
2. **Перенаправь ссылки** — все "передай {удаляемый}" → замени на нового получателя
3. **Удали из AGENTS.md**
4. **Удали из роутинга Director'а**
5. **Перемести папку** в `archive/` (не удаляй — вдруг понадобится)
6. **validate-office.sh** — проверь что нет broken links

### Правило: НЕ удаляй, а архивируй
```
mv agents/old-agent.md archive/agents/old-agent.md
```

## Масштабирование: когда менять паттерн

| Сигнал | Действие |
|--------|----------|
| 5-6 агентов | Проверь: не пора ли full-office? |
| 7+ агентов | WARNING: подумай о hierarchical |
| 10+ агентов | СТОП: переходи на hierarchical (departments/) |
| Пересечения зон | Разбей на sub-teams с Head'ами |

### Миграция small-team → full-office:
1. Создай departments/ структуру
2. Раздели agents/ по отделам
3. Назначь Head каждому отделу
4. Обнови Director — роутинг только на Heads
5. validate-office.sh
