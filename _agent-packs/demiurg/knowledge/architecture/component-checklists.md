# Покомпонентные чеклисты — глубокая проверка каждого слоя

Demiurg проверяет КАЖДЫЙ компонент агента ОТДЕЛЬНО, обращаясь к нужной
базе знаний для каждого. Не "общая проверка" — а детальная по слоям.

---

## 1. Проверка CLAUDE.md

**База знаний:** `knowledge/architecture/agent-design.md`
**Критерий прохождения:** все 12 пунктов = YES

| # | Проверка | Как проверить | YES/NO |
|---|---------|---------------|--------|
| 1 | Identity ≤ 300 tok | Посчитай слова × 1.3 ≈ токены | |
| 2 | Identity уникальный | Сравни с другими агентами — есть ли свой принцип? | |
| 3 | Первое предложение = роль | "Ты — {Роль}." — не абзац, не история | |
| 4 | Есть принцип работы | Не "делай хорошо", а конкретный принцип домена | |
| 5 | "Отвечает за" — 3-5 пунктов | Глаголы: "пишет", "создаёт", "проверяет" — не "отвечает за всё" | |
| 6 | "НЕ отвечает за" — с redirect | Каждый пункт → куда передать (путь к файлу агента) | |
| 7 | Контекст: ≤5 обязательных | Больше 5 = перегрузка. Остальное → "по запросу" | |
| 8 | Handle-паттерн | Ссылки на файлы, не inline текст знаний | |
| 9 | Все пути существуют | `ls path/to/file.md` для каждой ссылки | |
| 10 | 60-150 строк | `wc -l CLAUDE.md` | |
| 11 | Позитивные формулировки | Найди "НЕ делай" / "НИКОГДА" без объяснения — замени | |
| 12 | Решения: 3 уровня | Сам / С обсуждением / Эскалация — с примерами | |

**Если NO на пунктах 1, 2, 6, 9 → не переходи к следующему компоненту. Исправь сначала.**

## 2. Проверка Skills

**База знаний:** `knowledge/skill-mastery/anatomy.md` + `knowledge/skill-mastery/evaluation.md`
**Критерий прохождения:** все 10 пунктов = YES

| # | Проверка | Как проверить | YES/NO |
|---|---------|---------------|--------|
| 1 | Description "pushy" | Есть синонимы, ситуации, "use when" | |
| 2 | Near-miss прописан | "DO NOT use when..." — чтобы не путался с другими скиллами | |
| 3 | Body ≤ 500 строк | `wc -l SKILL.md` (без frontmatter) | |
| 4 | ≥2 примера Input → Output | Полные, копируемые, не абстрактные | |
| 5 | Фазы с quality gates | Каждая фаза: условие входа, шаги, условие выхода | |
| 6 | Exit criteria | Конкретное: "файл создан И чеклист пройден И score ≥8" | |
| 7 | Error recovery | На каждый шаг: "если упало → делай X" | |
| 8 | Progressive Disclosure | Тяжёлый контент → references/, body → навигация | |
| 9 | Knowledge routing | Таблица: фаза → какой файл загрузить | |
| 10 | Composability | Результат можно передать другому скиллу/агенту | |

**Красные флаги → немедленное исправление:**
- Description < 20 слов
- 0 примеров
- Body > 800 строк без references/
- Нет exit criteria (скилл "никогда не заканчивается")

## 3. Проверка Knowledge

**База знаний:** `knowledge/skill-mastery/writing-guide.md` (принципы)
**Критерий прохождения:** все 8 пунктов = YES

| # | Проверка | Как проверить | YES/NO |
|---|---------|---------------|--------|
| 1 | INDEX.md существует | Если >3 файлов в папке — INDEX обязателен | |
| 2 | INDEX имеет колонку "Когда загружать" | Не просто список файлов, а условия | |
| 3 | Один факт = одно место | grep ключевых фраз — нет дублей между файлами | |
| 4 | Файлы ≤ 300 строк | `wc -l *.md` | |
| 5 | Нет процессов в knowledge | Процессы → skills/. Knowledge = факты и паттерны | |
| 6 | Нет inline в CLAUDE.md | Содержимое knowledge не скопировано в CLAUDE.md | |
| 7 | evolving/ подготовлен | Есть learnings.md, failures.md для самообучения | |
| 8 | Файлы связаны | Из CLAUDE.md и skills/ есть ссылки на knowledge файлы | |

**Критичный пункт: #3.** Дублирование — самая частая ошибка.
Как проверить: возьми 3 ключевые фразы из каждого файла → grep по всей системе.

## 4. Проверка Связей

**База знаний:** `knowledge/architecture/context-patterns.md`
**Критерий прохождения:** все 7 пунктов = YES

| # | Проверка | Как проверить | YES/NO |
|---|---------|---------------|--------|
| 1 | Есть "reports_to" | Прописано кому подчиняется | |
| 2 | Есть "передай → {агент}" | В "НЕ отвечает за" указаны конкретные агенты | |
| 3 | Пути к агентам валидны | `ls agents/cto.md` — файлы существуют | |
| 4 | Нет пересечений зон | Сравни "Отвечает за" с другими агентами — нет дублей | |
| 5 | Прописан в AGENTS.md | Агент есть в корневом реестре | |
| 6 | Нет циклов | A → B → A — это баг. Проверь граф передач | |
| 7 | Общие точки определены | context.md (читает), ops/log.md (пишет) | |

**Как найти пересечения:**
1. Выпиши "Отвечает за" для всех агентов в таблицу
2. Ищи одинаковые задачи у разных агентов
3. Если нашёл → реши: кто главный, кто ассистирует

## Порядок проверки (конвейер)

```
1. CLAUDE.md   → agent-design.md        → чеклист 12 пунктов
   ↓ PASS?
2. Skills      → anatomy.md + evaluation.md → чеклист 10 пунктов
   ↓ PASS?
3. Knowledge   → writing-guide.md       → чеклист 8 пунктов
   ↓ PASS?
4. Связи       → context-patterns.md    → чеклист 7 пунктов
   ↓ PASS?
5. Memory      → (встроенный)           → чеклист 5 пунктов
   ↓ PASS?
6. validate.sh → автоматическая проверка broken links, размеров, TODO
   ↓ PASS?
7. DRY-RUN     → 2 сценария мысленно    → ≤3 шага каждый
```

**Правило:** НЕ переходи к следующему слою, пока текущий не PASS.
Ошибка в CLAUDE.md каскадно сломает skills и knowledge.
