# Migration Protocol — Миграция между паттернами AI-офиса

Практический гайд: как перейти между уровнями организации.
Три сценария вперёд + три отката назад.

---

## 1. Solo → Small Team (самый частый)

**Когда мигрировать:**
- CLAUDE.md > 200 строк (агент перегружен контекстом)
- context.md > 50 строк (слишком много состояния)
- 4+ проектов в одном агенте
- Задачи из разных доменов (код + контент + стратегия)

**Чеклист миграции:**

```
[ ] 1. Сохранить текущий CLAUDE.md
      cp CLAUDE.md agents/original-role.md

[ ] 2. Создать новый CLAUDE.md (Director)
      — Только роутинг: таблица "тип задачи → агент"
      — Никакой доменной логики
      — Ссылка на AGENTS.md

[ ] 3. Разбить старый на 2-3 агента
      — Выдели домены: код (CTO), контент (CMO), стратегия (Director)
      — Каждый агент = отдельный .md в agents/
      — Формат: роль, зона, инструменты, knowledge-ссылки

[ ] 4. Создать AGENTS.md
      — Таблица: имя | роль | зона | файл
      — Граф зависимостей (кто кому передаёт)

[ ] 5. Перенести knowledge/ ссылки
      — Для каждого агента: какие файлы из knowledge/ он читает
      — Убрать дубли — один файл = один владелец (остальные read-only)

[ ] 6. DRY-RUN: 3 тестовых задачи
      — Одна на каждого агента
      — Проверить: роутинг работает? Контекст достаточный?

[ ] 7. Валидация
      validate-office.sh
```

---

## 2. Small Team → Full Office (рост команды)

**Когда мигрировать:**
- AGENTS.md > 5 агентов
- Пересечение зон ответственности (два агента делают одно)
- 5+ проектов, появились явные отделы

**Чеклист миграции:**

```
[ ] 1. Добавить ops/ (операционный слой)
      mkdir -p ops/
      — ops/context.md — текущее состояние
      — ops/log.md — лог действий
      — ops/decisions/ — решения с датами

[ ] 2. Добавить memory/ (общая память)
      mkdir -p memory/
      — Общие факты, профили, стиль
      — То что нужно ВСЕМ агентам

[ ] 3. Обновить AGENTS.md
      — Добавить колонку "отдел"
      — Сгруппировать агентов по отделам

[ ] 4. Настроить канбан-доску
      — Статусы: ideas → week → today → working → testing → done
      — Привязка Issue к агентам через лейблы

[ ] 5. Добавить archive/ (если накопилось)
      mkdir -p archive/
      — Перенести неактуальные планы, старые решения

[ ] 6. Валидация
      validate-office.sh
```

## 3. Full Office → Hierarchical (масштабирование)

**Когда мигрировать:**
- 10+ агентов
- Director-контекст > 200 строк (не может удержать всё)
- Теряются задачи между агентами
- Роутинг стал сложнее чем "посмотри таблицу"

**Чеклист миграции:**

```
[ ] 1. Создать departments/
      mkdir -p departments/{product,content,operations}
      — Каждый отдел = папка с CLAUDE.md + agents/

[ ] 2. Назначить Heads
      — Head of Product, Head of Content, Head of Operations...
      — Head = агент с правом роутинга внутри отдела

[ ] 3. Двухуровневый роутинг
      — Director → Head → Agent
      — Director знает только Heads (5-7 строк, не 20)
      — Head знает агентов своего отдела

[ ] 4. Обновить AGENTS.md
      — Секция на каждый отдел
      — Граф: Director → Heads → Agents

[ ] 5. Разделить knowledge/
      — Общее: knowledge/ (все читают)
      — Отдельное: departments/product/knowledge/ и т.д.

[ ] 6. Валидация
      validate-office.sh --hierarchical
```

---

## Откат (downgrade)

### Full Office → Small Team
```
[ ] Удалить ops/, memory/ (или слить в один context.md)
[ ] Убрать канбан-процесс
[ ] Объединить агентов: 10 → 3-4
[ ] Упростить AGENTS.md до плоской таблицы
```

### Small Team → Solo
```
[ ] Слить всех агентов обратно в один CLAUDE.md
      cat agents/*.md > CLAUDE.md  # потом отредактировать
[ ] Удалить AGENTS.md, agents/
[ ] Вернуть knowledge/ ссылки в единый CLAUDE.md
[ ] Проверить: CLAUDE.md < 200 строк? Если нет — сократить
```

### Hierarchical → Full Office
```
[ ] Убрать departments/, вернуть плоскую структуру agents/
[ ] Director снова роутит напрямую на агентов
[ ] Объединить department-knowledge обратно в общий knowledge/
```

**Правило:** откат — это нормально. Лучше простая рабочая структура,
чем сложная сломанная. Если Director справляется — не усложняй.
