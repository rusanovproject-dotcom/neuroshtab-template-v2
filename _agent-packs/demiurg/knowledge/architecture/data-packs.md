# Data Packs Blueprint — Справочник данных для сборки агентов

> Справочник для Demiurg: какие данные собирать при создании какого типа агента.
> Без правильного data pack агент = пустая оболочка с красивым CLAUDE.md.

---

## Принцип

Агент = CLAUDE.md (мозг) + Skills (процессы) + **Knowledge (данные)**.
CLAUDE.md и Skills можно написать за час. Knowledge — нет.
Knowledge = то, что делает агента СПЕЦИАЛИСТОМ, а не генеричным GPT-ботом.

**Формула:** `Качество агента = Качество CLAUDE.md × Качество данных`

Если данные = 0, результат = 0. Неважно насколько хорош промт.

---

## Как пользоваться этим документом

1. Определи тип агента (или комбинацию типов)
2. Открой соответствующий раздел
3. Пройди по чеклисту "Обязательные данные"
4. Собери данные по процессу сбора
5. Положи в `knowledge/` агента в указанном формате
6. Только после этого — пиши CLAUDE.md

## 1. Копирайтер / Контент — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Авторский стиль (DNA)** — анализ 30-50+ текстов автора → `knowledge/domain/style-dna.md`
  - Tone of voice (исповедальный, дерзкий, академический...)
  - Ритм текста (длина абзацев, чередование, паузы)
  - Лексика (сленг, мат, разговорные формы, слова-маркеры)
  - Структура постов (хук → история → инсайт → CTA)
  - Юмор (тип, частота, примеры)
  - Уязвимость / самоирония (как показывает)
- [ ] **Антипаттерны стиля** — что автор НИКОГДА не скажет → `knowledge/domain/anti-patterns.md`
  - Запрещённые фразы ("друзья, рад сообщить")
  - Запрещённые структуры (стерильный инфостиль, чеклисты без души)
  - AI-slop маркеры для русского языка
- [ ] **Примеры текстов (эталоны)** — 10-20 лучших постов с разметкой → `knowledge/references/best-posts.md`
  - По типам: история, исповедь, провокация, обучающий, продающий
  - Для каждого: почему это сработало, что цепляет
- [ ] **Портрет аудитории** — язык клиента → `knowledge/domain/audience.md` (или ссылка на общий)
  - Дословные фразы из анкет ("знаю что делать, но не делаю")
  - Боли, мотивы, возражения — цитатами
  - Аватары с процентами
- [ ] **Продукт / контекст** — что продвигаем → `knowledge/domain/product.md` (или ссылка)
  - Терминология, тарифы, механики
  - Что решено, что нет

### Желательные данные (усиливают на 30%+):

- [ ] **Библиотека хуков** — 20-30 работающих первых строк → `knowledge/references/hooks.md`
- [ ] **Формулы текстов** — шаблоны по типам (дневник, поток, метафора) → `knowledge/references/formulas.md`
- [ ] **Метафоры и образы** — любимые метафоры автора → `knowledge/references/metaphors.md`
- [ ] **Конкуренты-стилисты** — 2-3 автора чей стиль близок, для калибровки → `knowledge/domain/style-benchmarks.md`
- [ ] **Anti-slop чеклист** — адаптированный под язык → `knowledge/references/anti-slop.md`

### Процесс сбора:

1. **Собери корпус текстов** — все посты автора (Telegram export, блог, рассылки). Минимум 30 текстов.
2. **Проанализируй паттерны** — прочитай 20 лучших постов, выпиши: повторяющиеся приёмы, слова-маркеры, структуру, ритм.
3. **Интервью с автором** (15-20 мин): "Какие твои посты ты считаешь лучшими? Что тебя бесит в текстах других? Какие слова ты никогда не используешь?"
4. **Портрет аудитории** — анкеты, CustDev, дословные фразы.
5. **Тест** — напиши 3 поста, дай автору вслепую оценить "это я или AI?"

### Пример из реальной системы:

В текущем AI Office копирайтер имеет: DNA стиля (500+ строк), антипаттерны, формулы текстов по 8 типам, Anti-slop секцию, чеклист перед отправкой. Это позволяет ему писать посты, которые неотличимы от авторских. **Ключевой файл:** `knowledge/brand/brief-for-methodologist.md` + inline style DNA в самом CLAUDE.md.

## 2. Маркетолог / Growth — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Портрет ЦА с цитатами** — не абстрактный, а с дословными фразами → `knowledge/domain/audience-portrait.md`
  - Аватары (Застрявший 55%, Выгоревший 25%, Ищущий 20%)
  - Иерархия болей с цитатами из анкет
  - Триггеры покупки — что должно случиться
  - Jobs To Be Done — на какую "работу" нанимают продукт
- [ ] **Продукт и тарифы** — что продаём, за сколько → `knowledge/domain/product.md`
  - Тарифная линейка (Стандарт 50-80к, Премиум 120-150к)
  - Что входит в каждый тариф
  - Unit-экономика: CAC, LTV, чек
- [ ] **Каналы и воронки** — где аудитория, как дойдёт до оплаты → `knowledge/domain/channels.md`
  - Текущие каналы (Telegram, Reels, ВК)
  - Путь: трафик → прогрев → заявка → демо → оплата
  - Конверсии на каждом шаге (если есть)
- [ ] **Ограничения** — бюджет, команда, время → `knowledge/domain/constraints.md`
  - Бюджет на маркетинг (руб/мес)
  - Кто делает руками (один человек / команда)
  - Сколько часов в неделю на маркетинг

### Желательные данные (усиливают на 30%+):

- [ ] **CustDev результаты** — сырые данные интервью → `knowledge/references/custdev.md`
- [ ] **Конкурентный анализ** — кто на рынке, цены, позиционирование → `knowledge/references/competitors.md`
- [ ] **История запусков** — что уже пробовали, что сработало/нет → `knowledge/evolving/launch-history.md`
- [ ] **Таблица возражений** — типичные возражения + ответы → `knowledge/references/objections.md`
- [ ] **Метрики-бенчмарки** — CPL, конверсии, ROI в нише → `knowledge/references/benchmarks.md`

### Процесс сбора:

1. **Анкеты** — минимум 50 ответов от ЦА (Google Forms, Typeform). Дословные фразы — золото.
2. **CustDev** — 5-10 глубинных интервью. Записать, транскрибировать, выделить паттерны.
3. **Аудит текущих каналов** — что уже есть, какие метрики, где узкое место.
4. **Конкурентный скан** — 5-10 конкурентов, их цены, воронки, позиционирование.
5. **Unit-экономика** — посчитать CAC, LTV, payback period для каждого канала.

## 3. Дизайнер — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Брендбук / визуальный код** — цвета, шрифты, стиль → `knowledge/domain/brand-code.md`
  - Цветовая палитра с HEX кодами (фон, акцент, текст, предупреждение)
  - Типографика (заголовки: Gilroy Bold, текст: Inter Regular)
  - Запрещённые цвета и шрифты
- [ ] **Визуальный стиль** — как выглядит бренд → `knowledge/domain/visual-style.md`
  - Минимализм / максимализм / технологичный / тёплый
  - Метафоры в визуале (космос, архитектура, свет в темноте)
  - Тон визуала (уверенный, трансформирующий, дерзкий)
  - Запрещённые визуальные приёмы (стоковые люди, мотивашки)
- [ ] **Форматы** — размеры для каждой площадки → `knowledge/domain/formats.md`
  - Telegram: 1280x720px (16:9)
  - Stories: 1080x1920px (9:16)
  - Квадрат: 1080x1080px (1:1)
  - Баннер: 1440x600px
- [ ] **Инструменты** — чем генерируем → `knowledge/domain/tools.md`
  - Midjourney v6+ (основной)
  - DALL-E 3 (быстрые итерации)
  - Ideogram (текст в изображениях)

### Желательные данные (усиливают на 30%+):

- [ ] **Референсы** — 10-20 изображений-эталонов → `knowledge/references/` (папка с файлами)
  - Лучшие обложки, баннеры, креативы
  - С аннотацией: почему это хорошо
- [ ] **Портрет аудитории** — для кого дизайн → ссылка на общий audience.md
- [ ] **Антипримеры** — что НЕ делать (скриншоты или описания) → `knowledge/references/anti-examples.md`
- [ ] **Логотип и айдентика** — файлы логотипа, паттерны → `knowledge/assets/`
- [ ] **Шаблон промта** — проверенная структура промта для Midjourney → `knowledge/references/prompt-template.md`

### Процесс сбора:

1. **Брендбук** — если есть PDF/документ — извлечь цвета, шрифты, правила. Если нет — провести мини-интервью: "покажи 5 визуалов которые тебе нравятся и 5 которые бесят".
2. **Цветовая палитра** — определить 4-5 основных цветов с HEX. Проверить на контрастность.
3. **Референсы** — собрать 10-20 изображений в стиле бренда. Пометить что именно в каждом хорошо.
4. **Форматы** — уточнить все площадки где нужен визуал, зафиксировать размеры.
5. **Тест** — сгенерировать 3-5 изображений, показать клиенту: "это похоже на твой бренд?"

## 4. CTO / Разработчик — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Стек технологий** — языки, фреймворки, версии → `knowledge/domain/stack.md`
  - Python 3.11, FastAPI, aiogram 3, PostgreSQL, Docker
  - Node.js, Next.js, PM2
  - Что используем, что категорически нет
- [ ] **Архитектура проектов** — как устроена система → `knowledge/domain/architecture.md`
  - Список проектов с портами и стеком
  - Схема взаимодействия сервисов
  - База данных: схема, миграции
- [ ] **Деплой и инфраструктура** — как деплоим → `knowledge/domain/deploy.md`
  - VPS: IP, SSH, кто где живёт
  - Docker compose конфиги (без секретов)
  - PM2 конфиги
  - CI/CD pipeline (если есть)
- [ ] **Код-стайл и конвенции** — как пишем → `knowledge/domain/code-style.md`
  - Naming conventions
  - Структура проекта (папки, модули)
  - Git flow (ветки, PR, ревью)
  - Тесты: что покрываем, как запускаем

### Желательные данные (усиливают на 30%+):

- [ ] **API документация** — endpoints, форматы → `knowledge/references/api.md`
- [ ] **Известные баги / tech debt** — что накопилось → `knowledge/evolving/tech-debt.md`
- [ ] **Шаблоны** — boilerplate для нового сервиса → `knowledge/assets/templates/`
- [ ] **Мониторинг** — как следим за здоровьем системы → `knowledge/domain/monitoring.md`
- [ ] **Ограничения** — один человек, AI-кодинг, бюджет → `knowledge/domain/constraints.md`

### Процесс сбора:

1. **Сканирование кода** — `find . -name "*.py" -o -name "*.js" | head -50`, docker-compose.yml, package.json, requirements.txt.
2. **Документация деплоя** — SSH конфиг, порты, systemd/PM2 конфиги.
3. **Интервью с разработчиком** (10 мин): "Какой стек? Как деплоишь? Что бесит в текущей архитектуре? Какие конвенции?"
4. **Git history** — последние 50 коммитов для понимания стиля и активности.

## 5. Коуч / Методолог — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Методология** — система работы → `knowledge/domain/methodology.md`
  - Фреймворк (4 слоя AI-OS, формула, пульс)
  - Этапы трансформации клиента
  - Ключевые инструменты (чекпоинт, нейрофундамент)
- [ ] **Программа** — структура по неделям → `knowledge/domain/program.md`
  - Что на какой неделе
  - Задания и чекпоинты
  - Критерии прохождения
- [ ] **Профиль клиента** — кого коучим → `knowledge/domain/client-profile.md`
  - Психотип (визионер, исполнитель, аналитик)
  - Циклы энергии (интенсив → провал → восстановление)
  - Триггеры энергии и истощения
  - Паттерны прокрастинации
- [ ] **Научная база** — доказательная основа → `knowledge/domain/science.md`
  - Нейронаука привычек
  - Психология изменений (Прохаска, BJ Fogg)
  - Биохакинг: сон, движение, нейрохимия

### Желательные данные (усиливают на 30%+):

- [ ] **Кейсы клиентов** — результаты с цитатами → `knowledge/references/cases.md`
- [ ] **FAQ / типичные возражения** — и ответы на них → `knowledge/references/faq.md`
- [ ] **Шаблоны сессий** — форматы check-in, разбора затора → `knowledge/references/session-templates.md`
- [ ] **Ограничения** — что коуч НЕ делает (не терапия, не бизнес-консалтинг) → в CLAUDE.md
- [ ] **Карта трансформации** — визуальная карта пути клиента → `knowledge/references/transformation-map.md`

### Процесс сбора:

1. **Интервью с методологом** (30-45 мин): "Опиши свою систему. Какие этапы проходит клиент? Что работает лучше всего? Что ты НИКОГДА не советуешь?"
2. **Программа** — расписание по неделям, задания, чекпоинты.
3. **Научная база** — статьи и книги которые лежат в основе методологии.
4. **Кейсы** — 5-10 историй клиентов с цитатами и результатами.
5. **Тест** — дать агенту провести пробную сессию, оценить точность рекомендаций.

## 6. Community Manager — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Правила группы** — что можно, что нельзя → `knowledge/domain/rules.md`
  - Формат общения (на "ты", без формальностей)
  - Что поощряется (уязвимость, вопросы, победы)
  - Что запрещено (спам, самопродвижение, токсичность)
- [ ] **Структура программы** — что когда происходит → ссылка на program.md
  - Расписание по неделям
  - Точки контакта (созвоны, задания, чекпоинты)
  - Критические моменты (неделя 3 — медовый месяц заканчивается)
- [ ] **Механики retention** — как удерживаем → `knowledge/domain/retention.md`
  - Онбординг: первые 48 часов (приветствие, первое задание, быстрая победа)
  - Форматы активации: горячее кресло, ротация пар, стена побед
  - Работа с молчунами: алгоритм по дням (3 дня → 5 дней → 7 дней)
  - Сигналы тревоги: дистанцирование, жалобы, скептицизм
- [ ] **Портрет аудитории** — кто в группе → ссылка на audience.md
  - Возраст, занятость, уровень самостоятельности
  - Что ценят, что бесит
  - Почему пришли, чего боятся

### Желательные данные (усиливают на 30%+):

- [ ] **Шаблоны сообщений** — готовые тексты для ситуаций → `knowledge/references/message-templates.md`
  - Приветствие новичка
  - Личное сообщение молчуну
  - Запуск активности в группе
  - Реакция на негатив
- [ ] **KPI и метрики** — что измеряем → `knowledge/domain/metrics.md`
  - Retention: 80%+ дошли до конца = хорошо
  - Активность: % написавших за неделю
  - NPS после завершения
- [ ] **Календарь ивентов** — план активностей по неделям → `knowledge/references/event-calendar.md`

### Процесс сбора:

1. **Правила** — записать явные и неявные правила сообщества.
2. **Интервью с основателем** (15 мин): "Какой тон в группе? Что делаешь когда кто-то молчит? Что было самым эффективным для вовлечения?"
3. **История группы** — если были прошлые потоки, собрать данные: кто ушёл и почему, что сработало.
4. **Шаблоны** — написать 5-7 шаблонов сообщений для типичных ситуаций.
5. **Тест** — дать агенту сценарий "участник молчит 4 дня" — проверить адекватность рекомендаций.

## 7. Аналитик — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Источники данных** — откуда берём информацию → `knowledge/domain/sources.md`
  - Внутренние: анкеты, CRM, аналитика бота, метрики Telegram
  - Внешние: конкуренты, тренды, рыночные отчёты
  - Формат данных (xlsx, json, csv, Notion)
- [ ] **Метрики и KPI** — что измеряем → `knowledge/domain/metrics.md`
  - Продуктовые: DAU, retention, completion rate
  - Маркетинговые: CPL, конверсия, ROI
  - Бизнес: MRR, LTV, churn
  - Для каждой: формула расчёта, бенчмарк, red flag
- [ ] **Портрет ЦА** — сырые данные → ссылка на audience.md + сырые анкеты
  - Дословные цитаты (главный актив аналитика)
  - Сегментация с процентами
  - JTBD формулировки
- [ ] **Фреймворки анализа** — как думать → `knowledge/domain/frameworks.md`
  - ФАКТ → ИНСАЙТ → ДЕЙСТВИЕ (обязательный)
  - Гипотеза → Проверка → Метрика → Критерий
  - Конкурентная матрица

### Желательные данные (усиливают на 30%+):

- [ ] **Конкурентный ландшафт** — кто на рынке → `knowledge/references/competitors.md`
- [ ] **Шаблоны отчётов** — форматы выхода → `knowledge/references/report-templates.md`
- [ ] **История решений** — что уже пробовали → `knowledge/evolving/decision-log.md`
- [ ] **Доступ к данным** — где лежат анкеты, как подключиться → `knowledge/domain/data-access.md`

### Процесс сбора:

1. **Инвентаризация данных** — что уже есть? Анкеты, CRM, метрики бота, аналитика каналов.
2. **Определение KPI** — с основателем определить 3-5 ключевых метрик и их бенчмарки.
3. **Конкурентный скан** — 5-10 конкурентов, базовая информация.
4. **Настройка доступа** — как агент получает данные (файлы, API, скрипты).
5. **Тест** — дать реальный вопрос ("какой аватар самый конверсионный?"), проверить ход рассуждений.

## 8. Product Designer / UX — Data Pack

### Обязательные данные (без них агент — пустышка):

- [ ] **Платформа и ограничения** — где проектируем → `knowledge/domain/platform.md`
  - Telegram Bot API: inline-кнопки, reply-keyboard, media, форматирование
  - Mini App: iframe, ограничения API
  - Мобильный контекст: 95% с телефона, палец не мышь
- [ ] **Дизайн-система** — визуальные правила → ссылка на brand-code.md или свой
  - Цвета, шрифты (для Mini App)
  - Компоненты: кнопки, карточки, прогресс-бары
  - Тексты интерфейса: тёплые, короткие, как от друга
- [ ] **User flows** — ключевые сценарии → `knowledge/domain/flows.md`
  - Онбординг → первый чекпоинт → привычка
  - Ежедневный цикл: пуш → кнопка → готово
  - Пропуск → мягкое возвращение
- [ ] **Привычкообразующие механики** — как формируем привычку → `knowledge/domain/habit-mechanics.md`
  - Hook-модель: триггер → действие → награда → инвестиция
  - Стрики, уровни, прогресс-бары
  - Прогрессивное раскрытие (неделя 1 = минимум)
- [ ] **Аудитория и антипаттерны** — для кого, что пробовали и бросили → ссылка на audience.md
  - Notion не прижился (слишком сложно)
  - ChatGPT-коуч работал но был отдельно
  - Бросают после 2-3 недель

### Желательные данные (усиливают на 30%+):

- [ ] **Бенчмарки** — как решено в лучших продуктах → `knowledge/references/benchmarks.md`
  - Duolingo (геймификация)
  - Fabulous (привычки)
  - Finch (эмоциональный UX)
  - Headspace (онбординг)
- [ ] **Прототипы** — текущие экраны бота → `knowledge/references/current-screens.md`
- [ ] **Результаты тестирования** — что пробовали, что работает → `knowledge/evolving/ux-tests.md`

### Процесс сбора:

1. **Аудит текущего продукта** — пройти весь путь пользователя, зафиксировать каждый экран.
2. **Ограничения платформы** — задокументировать все лимиты Telegram Bot API.
3. **Бенчмарки** — пройти онбординг 3-5 похожих продуктов, зафиксировать лучшие практики.
4. **Интервью с пользователями** (5-10 чел): "Что бесит? Где застреваешь? Что заставляет вернуться?"
5. **Тест** — дать агенту задачу "спроектируй онбординг", проверить соответствие ограничениям.

## Интеграция в Demiurg

### На каком этапе проверять data pack?

```
ФАЗА 0: ОПРЕДЕЛЕНИЕ ТИПА
↓
Demiurg определяет тип агента → загружает data-packs-blueprint.md →
находит нужный раздел → получает чеклист обязательных данных
↓
ФАЗА 1: DATA PACK AUDIT (НОВАЯ ФАЗА)
↓
Demiurg проверяет: какие данные УЖЕ ЕСТЬ в workspace?
  - Сканирует knowledge/, docs/, agents/
  - Сопоставляет с чеклистом
  - Отмечает: ✅ есть / ❌ нет / ⚠️ частично
↓
Если ❌ > 30% обязательных:
  → СТОП. "Не могу собрать качественного агента. Вот что нужно собрать: [список]"
  → Предлагает процесс сбора (интервью, анкеты, анализ)
↓
Если ❌ < 30%:
  → Продолжаем сборку, помечаем пробелы в MEMORY.md агента
↓
ФАЗА 2: BUILD (как обычно)
↓
ФАЗА 3: REVIEW + DATA QUALITY CHECK
  → Проверяет: все ссылки на данные валидны? Данные актуальны?
```

### Как Demiurg узнает какой data pack нужен?

Добавить в `knowledge/architecture/` файл `data-packs-index.md`:

```markdown
| Тип агента | Ключевые слова в задаче | Data Pack |
|-----------|------------------------|-----------|
| Копирайтер | тексты, посты, голос, стиль, контент | #copywriter |
| Маркетолог | воронка, трафик, конверсия, growth | #marketer |
| Дизайнер | визуал, брендбук, промты, обложки | #designer |
| CTO | код, стек, деплой, архитектура | #developer |
| Коуч | методология, программа, состояние | #coach |
| Community | группа, retention, вовлечение | #community |
| Аналитик | данные, метрики, анализ, CustDev | #analyst |
| Product Designer | UX, интерфейс, экраны, привычки | #product-designer |
```

### Формат хранения

**Один файл** `data-packs-blueprint.md` (этот документ) — справочник.
Причина: типы пересекаются (копирайтер и маркетолог оба используют audience), один файл = один источник правды. Demiurg загружает его целиком, находит нужный раздел по якорю.

Плюс **индексный файл** `data-packs-index.md` — маршрутизатор "ключевые слова → тип → раздел".

## Для продукта AI-OS (продажа клиентам)

### Проблема

Если AI-офис продаётся как продукт, каждый клиент должен "накормить" агентов данными. Но клиент — не аналитик. Ему нужен управляемый процесс, а не чеклист на 50 пунктов.

### Решение: 3-этапный Data Wizard

#### Этап 1: Быстрый старт (15 минут)

**Формат:** анкета / Typeform / Telegram-бот

```
БЛОК 1: О бизнесе (5 мин)
1. Чем занимаешься? (1-2 предложения)
2. Что продаёшь? (продукт, цена, формат)
3. Где твоя аудитория? (Telegram / Instagram / VK / другое)
4. Сколько клиентов в месяц?
5. Бюджет на маркетинг?

БЛОК 2: О стиле (5 мин)
6. Покажи 3 своих лучших поста/текста (ссылки или скриншоты)
7. Покажи 3 поста которые тебе нравятся у ДРУГИХ
8. Какие слова ты НИКОГДА не используешь?
9. Тон: (выбери) дерзкий / тёплый / экспертный / провокационный / спокойный

БЛОК 3: О клиенте (5 мин)
10. Опиши типичного клиента (возраст, чем занимается, что болит)
11. Что клиент говорит ДОСЛОВНО когда приходит к тебе?
12. Почему клиент выбирает ТЕБЯ, а не конкурента?
13. Главное возражение перед покупкой?
```

**Что это даёт:** минимальный жизнеспособный data pack для Копирайтера + Маркетолога.

#### Этап 2: Глубокое погружение (30-60 минут)

**Формат:** голосовое интервью (AI транскрибирует и структурирует)

```
ТЕМА 1: Стиль и голос (15 мин)
- "Расскажи мне историю своего бизнеса как будто мы за кофе"
- "Какой пост ты написал и до сих пор гордишься? Почему?"
- "Что тебя бесит в текстах конкурентов?"

ТЕМА 2: Аудитория (15 мин)
- "Опиши трёх последних клиентов. Что у них общего?"
- "С какой фразой к тебе чаще всего приходят?"
- "Почему люди НЕ покупают?"

ТЕМА 3: Продукт (15 мин)
- "Если бы ты объяснял что делаешь своей маме — как бы сказал?"
- "В чём ты лучше всех конкурентов?"
- "Что ты обещаешь клиенту и всегда выполняешь?"

Опционально — ТЕМА 4: Визуал (10 мин)
- "Покажи 5 картинок/дизайнов которые тебе нравятся"
- "Какие цвета у бренда?"
```

**Что это даёт:** DNA стиля, глубокий портрет ЦА, позиционирование.

#### Этап 3: Обогащение (автоматическое, фоновое)

```
Агент-сборщик (AI) автоматически:
1. Анализирует 30+ постов клиента → извлекает стиль, хуки, структуру
2. Сканирует конкурентов → матрица позиционирования
3. Парсит отзывы / комментарии → голос клиента
4. Генерирует: style-dna.md, anti-patterns.md, audience.md
5. Показывает клиенту: "Вот что я понял о тебе. Поправь если не так."
```

### Минимизация трения

| Принцип | Реализация |
|---------|-----------|
| Не спрашивай то, что можно узнать | Парси посты, канал, сайт клиента автоматически |
| Голос > текст | Интервью голосом, AI транскрибирует |
| Прогрессивный сбор | Начни с анкеты (15 мин), углубляй по мере работы |
| Покажи ценность рано | После этапа 1 уже покажи первый результат (пост / анализ ЦА) |
| Не требуй идеального | "Примерно" лучше чем "ничего". Данные уточняются в процессе работы |
| Клиент не должен знать про data packs | Для него это "настройка AI-офиса", не "сбор данных" |

### Wizard flow для клиента (UX)

```
[Регистрация]
  → "Давай настроим твой AI-офис. 15 минут — и команда начнёт работать"
  ↓
[Анкета: 13 вопросов, 15 мин]
  → "Отлично! Уже могу показать первый результат. Хочешь пост для канала?"
  ↓
[Первый результат: пост / анализ ЦА]
  → "Чтобы команда работала ещё точнее — давай поговорим 30 минут"
  ↓
[Голосовое интервью, 30 мин]
  → AI структурирует → показывает "вот что я понял о тебе"
  ↓
[Клиент корректирует]
  → Data pack собран. Агенты калибруются.
  ↓
[Фоновое обогащение]
  → Агент-сборщик парсит посты, конкурентов, отзывы
  → Data pack растёт автоматически
```

## Резюме: Что меняется в процессе Demiurg

### Было:
```
Задача → Определить тип → Написать CLAUDE.md → Готово
```

### Стало:
```
Задача → Определить тип → DATA PACK AUDIT → Данные есть?
  → Да → Написать CLAUDE.md + подключить данные → Review → Готово
  → Нет → Запросить сбор данных → Собрать → Вернуться к сборке
```

**Ключевое изменение:** фаза DATA PACK AUDIT между определением типа и написанием CLAUDE.md. Это предотвращает создание "пустых" агентов.
