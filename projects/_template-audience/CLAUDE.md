# Stage 1 — Распаковка ЦА (Audience-JTBD)

> Минимальная папка под ОДНУ задачу: глубокая распаковка целевой аудитории через JTBD-фрейм. Без продукта, воронки, бренда. Когда ЦА готова — наслаиваем следующие стадии отдельно.

## Зачем именно так

Клиент пришёл с базовой идеей продукта или направлением. Нужен **фундамент**: кто реально покупает, какие сегменты, какие jobStory, какие барьеры, что мешает купить. Без этого фундамента продукт-оффер-воронка — стрельба наугад.

Папка живёт **в `projects/<slug>/`** под выбранное направление работы (создаётся Алексом автоматически на онбординге).

## Метафора

Это не полный организм — это **диагностика через JTBD**. Только то что нужно для понимания «кто наш клиент и какую работу он нанимает наш продукт делать»:

| Слой | Где | Роль |
|------|-----|------|
| 🍴 Желудок | `inbox/` | приём всех входящих: цитаты клиентов, скрины, голосовые, ссылки на конкурентов, транскрипты диагностик |
| 🩸 Кровь | `inbox/_new/` | сырые материалы — кастдев, диагностики, выступления, отзывы |
| ❤️ Сердце | `hypotheses.md` | гипотезы про ЦА: `[ICP]`, `[SEGMENT]`, `[JOB]`, `[MESSAGE]` |
| 🧠 Мозг | `JTBD_анализ_<main>.md` (создаётся Алексом) | главный артефакт — все сегменты с Big Job, jobStory, Точками А/Б, графом работ, барьерами, оценкой, ранжированием |
| 📚 Гиппокамп | `learnings.md` | уроки про ЦА которые подтвердились 3+ раза |
| 💾 Точка сохранения | глобальный `office/agents/alex-marketer/agent-state.md` | resume-pattern — какой Step активен, какой сегмент, last_checkpoint. Алекс обновляет после каждого Step. Между сессиями `/clear` → новый чат продолжит с этой точки |

Нет 🧠 «чем отличаемся / что продаём / как доставляем» — это следующие стадии.

## Главный артефакт на выходе

После работы Алекса (`/jtbd` 13 шагов на каждый сегмент):

```
JTBD_анализ_<main>.md          ⭐ единый документ — все сегменты с полным разбором
```

Внутри — секции по сегментам:
- Big Job (главная работа клиента)
- jobStory (When... I want... So I can...)
- Точки А (текущее состояние) и Б (желаемое)
- Граф работ (под-задачи и зависимости)
- Consideration Set (что клиент рассматривает кроме твоего продукта)
- Барьеры переключения
- Entry Job + Monetization Job
- Оценка сегмента (по осям размер/доступность/готовность платить/конкуренция/соответствие ресурсам)

Плюс общие секции: ранжирование сегментов → механики ценности → стратегические гипотезы.

## Поток работы

```
1. /alex-onboarding — Алекс читает офис, спрашивает направление, создаёт эту папку
2. /jtbd Step 00 — диагностика готовности (есть ли фактура от платящих?)
3. /jtbd Step 01a-01b — собрать factbase, кластеризация на 1-5 сегментов
4. /jtbd Step 02-10 — per-segment разбор (Big Job → ... → оценка)
5. /jtbd Step 11-13 — ранжирование + механики ценности + стратегические гипотезы
6. /jtbd-critic в новом чате — свежий взгляд проверяет на абстракции и противоречия
7. Stage 1 закрыта когда: документ полный + клиент сказал «принимаю» + критик пройден
```

⚠️ **Если у клиента 0 платящих** — Step 00 предлагает облегчённый путь (1 гипотеза-сегмент, 5-6 шагов) или offramp в кастдев. Не делаем полный JTBD на пустоте.

## Закон голоса

Алекс пишет:
- `JTBD_анализ_<main>.md` — главный документ (Алекс заполняет, клиент читает и принимает блоками)
- `hypotheses.md` — тест-процесс гипотез
- `inbox/_processed/` — переработанные входящие

Алекс не перезаписывает финальный документ блоками которые клиент принял — только дополняет diff'ами в hypotheses.md → Log с пометкой даты.

## Что дальше после Stage 1

После того как JTBD-разбор закрыт, можно **наслаивать** следующие стадии. Каждая — отдельный модуль (появятся отдельно):

- **Stage 2 Product** → распаковка оффера через 6 диалоговых протоколов Хормози (Value Equation, Grand Slam, Lead Magnet)
- **Stage 3 Funnel** → per-segment маршруты, каналы, касания
- **Sales Meetings** → активируется через `/marketer-enable-meetings` — добавит `customers/` и банк возражений из транскриптов
- **Brand** → когда подключится агент-Дизайнер
