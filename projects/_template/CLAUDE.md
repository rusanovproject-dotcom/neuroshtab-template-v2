# Проект — {{PROJECT_NAME}}

Главный проект программы. Здесь живёт всё что относится к маркетингу, продукту, аудитории, бренду и воронке.

## Метафора — папка как организм

Базовый организм (есть у любого проекта):

| Слой | Где | Роль |
|------|-----|------|
| 🍴 Желудок | `inbox/` | приём всех входящих — идеи, цитаты, конкуренты, скрины, метрики, голосовые, любые сырьё |
| ❤️ Сердце | `hypotheses.md` | насос — гоняет реальность через мозг в выводы |
| 🧠 Мозг | `audience/` `product/` `brand/` | три доли: кому, что, чем отличаемся |
| 👄 Уста | `brand/expert-bank.md` + `audience/voc.md` | два голоса — эксперта и ЦА |
| ✋ Руки | `funnel/` | действия — как доставляем оффер |
| 👁 Глаза | `metrics.md` + `progress.md` | измерения |
| 📚 Гиппокамп | `learnings.md` | консолидированные уроки проекта |

**Опциональный орган** (активируется через `/marketer-enable-meetings`):

| Слой | Где | Когда подключать |
|------|-----|------------------|
| 🫁 Печень | `customers/` | бизнес работает с продающими/диагностическими созвонами |

Если у тебя нет встреч 1-на-1 (контент-маркетинг, автоворонка, e-commerce, SaaS) — печень не нужна. Базовый организм работает без неё.

## Поток данных (универсальный)

```
Любой вход (идея, скрин, цитата, цифра, ссылка) → inbox/_new/
                                                    ↓
                                              классификация
                                                    ↓
                            audience/ · brand/ · metrics · hypotheses (Backlog)
                                                    ↓
                                       Active → Validated → Log с предложением diff
                                                    ↓
                                  эксперт принимает → файл вывода обновлён
```

При активации модуля встреч добавляется ветка:
```
транскрипт встречи → customers/{name}/transcripts/ → 7 артефактов параллельно
                     (карточка, voc, expert-bank, конкуренты, возражения, идеи, метрики)
```

## Закон голоса

Файлы выводов (`product/core-offer.md`, `brand/positioning.md`, `audience/NORTH-STAR.md` и т.д.) пишет **владелец проекта** ({{CLIENT_NAME}}), не агент. Агент **предлагает изменения** через diff в `hypotheses.md → Log`. Владелец принимает командой `/accept H{N}` или отвергает.

Безопасные зоны для агента (пишет сам):
- `inbox/`
- `audience/voc.md` (append only)
- `brand/expert-bank.md` (append only)
- `hypotheses.md`
- `metrics.md` (только цифры в AUTO-секциях)
- `_archive/`
- `customers/` (если модуль встреч активирован — накопление, не интерпретация)

## Старт

Если файлы выводов пусты — это норма для дня 1. Первоначальные гипотезы владельца живут в `hypotheses.md → 🧊 Backlog` с маркером `🌱 starter`. По мере прихода данных двигаются в Active или режутся.
