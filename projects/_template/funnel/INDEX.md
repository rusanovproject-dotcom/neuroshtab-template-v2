# Funnel — КАК

✋ Руки. Действия — как доставляем оффер до клиента.

## Файлы

| Файл | Что | Кто пишет |
|------|-----|-----------|
| `channels.md` | Core Four + CAC по каналам + что мертво / что топим | маркетолог |
| `welcome.md` | Welcome flow + лид-магниты + последовательность касаний (включая email-flows для автоворонок) | маркетолог |
| `scripts.md` | (опционально) Скрипты продающих созвонов + банк возражений | маркетолог из транскриптов *— активируется через модуль встреч* |

## Связь с тестами

| Тег H | Обновляет |
|-------|-----------|
| `[CHANNEL]` | `channels.md` |
| `[LM]` | `welcome.md` |
| `[FUNNEL]` | `welcome.md` (+ `scripts.md` если модуль встреч) |

## Принципы

- **Channel-First.** Сначала разобраться какой канал primary, потом строить воронку.
- **Bottleneck.** В каждой воронке найти стопор (где люди отваливаются) — там и работаем, а не везде.
- **Возражения растут снизу.** Скрипты и тексты воронки не пишутся «из головы» — они собираются из реальных реплик ЦА (комментарии, транскрипты, переписки).

## Если бизнес без созвонов

`scripts.md` остаётся пустым или не создаётся. Возражения из автоворонки (отписки, drop-off, ответы в чате) копятся в `audience/voc.md` с тегом «objection» и используются для оптимизации `welcome.md` и текстов лендинга.
