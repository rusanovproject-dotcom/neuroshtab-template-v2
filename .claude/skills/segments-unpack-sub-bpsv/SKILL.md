# Описание сегментов — Первичные гипотезы (БПСВ) — клон под `/segments-unpack`

> **Это клон.** Источник — `knowledge-alchemist/02-segment-description/SKILL.md` (не правится). Здесь — адаптация под пайплайн `audience/segments/`. Методология (БПСВ + страхи Ты/Я + 3 типа возражений) **остаётся как есть**.
>
> **Контракт ввода:** оркестратор `/segments-unpack` запускает клон **по одному сегменту за раз** (hot → warm → cold). Клон читает `audience/segments/{slug}/dossier.md` (секция 0 — профиль уже заполнен Скиллом №1) и при необходимости `audience/segments/{slug}/intel/competitors-{slug}/` для контекста.
>
> **Контракт вывода:** заполненная БПСВ-карточка пишется **прямо в** `audience/segments/{slug}/dossier.md` в секции 1 — гипотезы БПСВ/БПКСВ (соцдем-профиль идёт в секцию 0 если ещё не заполнен; боли/потребности/страхи/возражения — в секцию 1). Шаблон `templates/segment-card.md` используется как структурный референс, не как отдельный output-файл. Список из 2 представителей на сегмент идёт в `audience/segments/{slug}/dossier.md` секцию 0.x (профили представителей).
>
> Берёшь сегмент из ТОП-3 и создаёшь полный психологический портрет: соцдем + боли + потребности + страхи (ты/я) + возражения (ценовые/ценностные/доверительные). Принцип: это гипотезы, не истина — всё проверяется потом через Users Research / `/custdev`. Без этого шага интервью будет вслепую.

## Зона ответственности

Отвечаешь за:
- Создание соцдем-профиля (пол, возраст, доход) для каждого сегмента
- Заполнение 4 блоков БПСВ: боли, потребности, страхи, возражения
- Разделение страхов на "ты как продукт" и "я как потребитель"
- Типизацию возражений: ценовые, ценностные, доверительные
- Подбор 2 реальных представителей на каждый сегмент (6 человек)

НЕ отвечаешь за:
- Само сегментирование → передай 01-segmentation
- Проведение интервью → передай ai-custdev
- Уровни осознанности → передай 04-awareness-levels

## Контекст

При старте загрузи:
1. `references/methodology.md` — фреймворк БПСВ, страхи Ты/Я, 3 типа возражений
2. `references/examples.md` — good/bad примеры заполненных карточек

## Процесс работы

### Фаза 1: Приём сегментов и сбор контекста

```
У тебя есть 3 сегмента из предыдущего шага?
├── Да → "Скинь описание 3 сегментов (горячий, тёплый, холодный)"
├── Частично → "Скинь что есть + опиши свой продукт, добьём вместе"
└── Нет → "Сначала нужно сегментирование. Запусти скилл 01-segmentation"
```

Собери от пользователя:
1. Описание продукта/услуги (1-2 предложения)
2. 3 сегмента с их "теплотой" (горячий/тёплый/холодный)
3. Краткое описание каждого сегмента (кто это, 1-2 предложения)

**Quality gate:** Есть продукт + 3 сегмента с теплотой. Переход к Фазе 2.

### Фаза 2: Соцдем-профиль

Для каждого из 3 сегментов определи:

| Параметр | Правила |
|----------|---------|
| Пол | Если оба — пиши оба. Не сужай без причины |
| Возраст | Вилкой (30-45). Не точное число |
| Доход | Приблизительный. Проверь: доход позволяет купить продукт? |

**Проверка дохода:** если цена продукта > 2× месячного дохода сегмента
и продукт не критично необходим → пересмотри сегмент или цену.

**Quality gate:** У каждого сегмента есть пол + возраст + доход. Переход к Фазе 3.

### Фаза 3: Заполнение БПСВ

Для каждого из 3 сегментов заполни 4 блока:

**Блок 1: Боли (5-10 штук)**
- Формула: "У меня болит / не работает / мешает..."
- Пиши конкретно, не абстрактно
- Первые 2-3 будут очевидные — копай глубже

**Блок 2: Потребности (5-7 штук)**
- Формула: "Я хочу / мне нужно / мне требуется..."
- Каждая потребность вытекает из боли (обратная сторона)
- Проверь: потребность = желаемая трансформация?

**Блок 3: Страхи (5-8 штук, разделённые)**
- "Ты как продукт": сомнения в спикере, продукте, компании
- "Я как потребитель": сомнения в себе, своих способностях
- Формула: "Я боюсь, что..."

**Блок 4: Возражения (5-8 штук, типизированные)**
- Ценовые: "Дорого / за эти деньги можно..."
- Ценностные: "Не нужно / уже проходил / ничего нового"
- Доверительные: "Не верю тебе / не видел кейсов"
- Формула: "Да, но..."

Проделай со всеми 3 сегментами.

**Quality gate:** Каждый сегмент имеет ≥5 болей, ≥5 потребностей,
≥3 страхов (Ты) + ≥3 страхов (Я), ≥2 возражений каждого типа.

### Фаза 4: Подбор представителей и проверка

**Представители:**
Для каждого сегмента найди 2 реальных человека:
- Из существующей базы / CRM
- Из знакомых / знакомых знакомых
- Важно: они должны соответствовать БПСВ-профилю

**Чеклист качества каждой карточки:**
```
□ Соцдем заполнен (пол + возраст + доход)
□ Боли конкретные, не абстрактные (≥5)
□ Потребности вытекают из болей (≥5)
□ Страхи разделены на Ты/Я (≥3 + ≥3)
□ Возражения типизированы: ценовые/ценностные/доверительные (≥2+2+2)
□ Найдены 2 представителя для интервью
□ Доход соответствует цене продукта
БАЛЛОВ: _/7 (требуется 6+)
```

**Exit criteria:** 3 заполненные БПСВ-карточки + 6 представителей для интервью.

## Формат вывода

Для каждого сегмента — заполненная карточка по шаблону `templates/segment-card.md`.
Итого: 3 карточки + сводная таблица представителей.

## Примеры

### Пример 1: Инфопродюсер (happy path)
```
Input: "Курс по автоворонкам. Горячий сегмент — инфопродюсер."
Output:
  Соцдем: М/Ж, 25-45, доход 150-500К/мес
  Боли (10): нет прогнозирования, запуски выжигают, низкие цифры,
    не умеет прогревать, накатанная модель, всё сложно, нет текстов/видео,
    завязано на эксперте, нет системы, устал от марафонов
  Потребности (7): автоматизация, система, не упарываться, влиять на цифры,
    прогревать, линейка продуктов, найти людей
  Страхи-Ты (7): некомпетентен, нет бэкграунда, нет команды,
    новый продукт, не решит проблем, нет ценности, непонятен
  Страхи-Я (8): недостаточно умный, рано, нет опыта,
    негативный опыт, дело во мне, не смогу, ответственность, продукт не подходит
  Возражения: ценностные (4) + доверительные (5)
  Представители: 2 найдены из базы
Путь: Фаза 1 → 2 → 3 → 4 (полный цикл)
```

### Пример 2: Cold start (нет опыта в нише)
```
Input: "Запускаю курс по выпечке хлеба. Сегменты есть, но я новичок —
  не знаю какие у людей боли."
Output:
  → Фаза 1: принял 3 сегмента
  → Фаза 2: соцдем (помог определить доход через вопросы)
  → Фаза 3: БПСВ через наводящие вопросы:
    "Что чаще всего не получается у людей, когда они пекут хлеб?"
    "Почему они бросают после 2-3 попыток?"
    "Чего боятся перед покупкой курса по выпечке?"
  → Фаза 4: предложил где искать представителей (форумы, Instagram, кулинарные группы)
  Все гипотезы помечены как "требует проверки через Users Research"
Путь: Фаза 1 → 2 → 3 (расширенная, с наводящими вопросами) → 4
```

## Что делать если что-то пошло не так

- Пользователь не может сформулировать боли → задай наводящие: "Что чаще всего жалуются клиенты?", "Что бесит людей в этой теме?"
- Боли и потребности путаются → напомни формулу: боль = "болит", потребность = "хочу"
- Страхи не делятся на Ты/Я → задай: "Чего сомневается в ТЕБЕ?" и "Чего сомневается в СЕБЕ?"
- Не может найти представителей → предложи: база, CRM, знакомые, тематические группы, LinkedIn
- Доход не соответствует цене → обсуди: пересмотреть сегмент или пересмотреть ценообразование
