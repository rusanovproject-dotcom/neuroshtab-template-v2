# Source Extract — Урок 2: Описание сегментов (Первичные гипотезы)

> Стадия 2 конвейера Knowledge Alchemist. Полный экстракт из транскрипции.

## Метаданные
- Источник: исходный курс, Модуль 1, Урок 2
- Тип: транскрипция видео
- Объём: ~8500 символов
- Язык: русский, разговорный

---

## Фреймворки (2)

1. **БПСВ-фреймворк** — структура описания сегмента: Боли, Потребности, Страхи, Возражения. Ранее был БПКСВ (с Критериями), но критерии перенесены на 4-й уровень.
2. **Страхи "Ты vs Я"** — деление страхов на две категории: "ты как продукт" (сомнения в спикере/продукте) и "я как потребитель" (сомнения в себе).

## Принципы (4)

1. **Сегменты после первого урока — поверхностные.** Нужно пройти проверку гипотез на адекватность перед интервьюированием.
2. **Гипотезы — не истина, а предположения.** "Как мы считаем" — ключевое слово. Всё требует проверки через Users Research.
3. **Доход привязан к ценообразованию.** Глупо предлагать продукт за 150К человеку с доходом 50К, если продукт не критично необходим.
4. **Возражения — база автоворонки.** Всё, что вытащишь из интервью, это недостаток информации, который нужно восполнить контентом.

## Техники (5)

1. **Соцдем-профиль**: пол + возраст (вилкой) + приблизительный доход. Ничего лишнего.
2. **Заполнение БПСВ**: для каждого из 3 сегментов прописать 4 блока (боли, потребности, страхи, возражения).
3. **Страхи по формуле "Ты/Я"**: разделить страхи на "ты как продукт/спикер" и "я как потребитель".
4. **Возражения 3 типов**: ценовые (дорого), ценностные (не нужно), доверительные (не верю тебе).
5. **Поиск 2 представителей на сегмент**: найти реальных людей из базы, CRM, знакомых — по 2 на каждый из 3 сегментов = 6 человек для интервью.

## Примеры (4)

1. **Инфопродюсер — боли**: нет чёткой уверенности и прогнозирования, запуски выжигают, низкие цифры окупаемости, не умеет прогревать, работает по накатанной модели, всё завязано на эксперте, нет системного подхода.
2. **Инфопродюсер — потребности**: уйти от запусков / автоматизировать, четкая система, влиять на цифры, научиться прогревать, готовая линейка продуктов, нужны люди.
3. **Инфопродюсер — страхи (ты)**: некомпетентен как спикер, нет бэкграунда, нет агентства, не запускал образовательные проекты, продукт новый/неизвестный, не решит проблем.
4. **Инфопродюсер — возражения**: ценностные (мне не нужно, уже проходил, ничего нового, не решит проблему), доверительные (не доверяю спикеру, не видел кейсов, упаковка не внушает).

## Антипаттерны (3)

1. **Писать 5-километровые карты.** Не нужно чрезмерно детализировать — достаточно БПСВ для проверки гипотез на адекватность.
2. **Выдумывать лишнее в соцдеме.** Если и мужчина, и женщина — пиши оба пола. Не изобретай ограничения.
3. **Не указывать доход.** Без привязки к доходу невозможно корректно работать с ценообразованием.

## Определения (3)

1. **Первичные гипотезы** — предположения о болях, потребностях, страхах и возражениях сегмента ДО проведения Users Research.
2. **БПСВ** — Боли (у меня болит), Потребности (я хочу/мне нужно), Страхи (я боюсь что), Возражения (да, но...).
3. **Соцдем** — социально-демографический профиль: пол, возраст (вилкой), приблизительный доход.

## Задания (2)

1. **Заполнить БПСВ для каждого из 3 сегментов** с соцдем-профилем.
2. **Найти по 2 реальных представителя на каждый сегмент** (6 человек) — для последующего интервьюирования.

## Связи (2)

1. **Вход**: результат Урока 1 (3 сегмента: горячий, тёплый, холодный).
2. **Выход**: БПСВ-карты → входные данные для Users Research (Урок 3).
