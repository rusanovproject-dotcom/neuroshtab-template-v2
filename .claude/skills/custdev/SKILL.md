# /custdev — Быстрый синтетический CustDev

Валидируй идеи за 15-30 минут. Не из своих представлений — через OCEAN-персоны
и разведку рынка. Работает для ЛЮБОЙ ниши и ЦА.

## Два режима

| Режим | Когда | Персоны |
|-------|-------|---------|
| **Универсальный** (по умолчанию) | Любая идея, любая ниша | Генерируются с нуля через OCEAN + БПСВ |
| **На базе анкет** (флаг `--наши`) | Идея для текущей ЦА владельца офиса | Из `projects/<slug>/audience/JTBD_анализ_<slug>.md (если есть)` |

Если владелец НЕ указал `--наши` / "используй наших" / "наша ца" — всегда универсальный режим.

## Пайплайн: 5 фаз

```
ВХОД: идея в свободной форме
  → Фаза 1: Фокусировка + Сегментация (ты сам)
  → Фаза 2: OCEAN-персоны (Persona Builder, sonnet) ─┐ параллельно
  → Фаза 3: Разведка рынка (Intelligence)            ─┘
  → Фаза 4: Глубинные интервью (3-5x Interviewer, opus)
  → Фаза 5: Синтез (Producer)
ВЫХОД: CustDev Report
```

---

## Фаза 1: Фокусировка + Сегментация (ты сам, 3 мин)

### 1a. Карточка идеи

```markdown
## Карточка идеи
- Суть: [1 предложение]
- Целевая аудитория: [кто — ниша, профессия, ситуация]
- Боль: [какую проблему решает]
- Формат: [продукт/фича/сервис/контент]
- Отличие: [чем отличается от существующих]
- Рынок: [ниша]
```

### 1b. Быстрая сегментация (3W: What-Why-Who)

Определи 3 сегмента по теплоте:
- **Горячий** — уже ищут решение, тратили деньги на похожее
- **Тёплый** — осознают проблему, но не активно ищут
- **Холодный** — не осознают проблему или не считают её важной

Для каждого сегмента — 1 строка: кто это и чем отличается.

Покажи карточку + сегменты владельцу, дождись подтверждения. Потом Фазы 2+3 параллельно.

## Фаза 2: OCEAN-персоны (параллельно с Фазой 3)

Запусти Agent tool:
- subagent_type: general-purpose
- model: sonnet
- Промт: прочитай спеку `~/workspace/ai-office-v2/demiurg/agents/persona-builder.md`,
  затем шаблон `~/.claude/skills/custdev/references/persona-template.md`.
  Карточка идеи: {вставь}. Сегменты: {вставь}.
  Режим: {универсальный | на базе анкет}.
  Создай 5 персон (3 горячий, 1 тёплый, 1 холодный). Каждая с полным OCEAN-профилем.
  Обеспечь разнообразие по матрице (возраст ≥15 лет разброс, OCEAN-комбинации разные,
  есть фанаты + скептики). Сохрани в `~/workspace/knowledge/custdev/personas/`.

## Фаза 3: Разведка рынка (параллельно с Фазой 2)

Запусти Agent tool:
- subagent_type: intelligence
- Промт: проведи разведку рынка для идеи: {карточка идеи}.
  Найди: конкурентов (кто решает эту проблему), горячие темы (где деньги),
  Gap Map (что не делает никто), Awareness Level, Market Sophistication,
  ценовые ориентиры, Voice of Customer (дословные фразы из Reddit, YouTube, Telegram).
  Используй WebSearch. Выдай краткий разведотчёт (не full deep-research, фокус на идею).

---

## Фаза 4: Глубинные интервью

Дождись результатов Фаз 2 и 3. Затем запусти 3-5 параллельных Agent tool:

Для КАЖДОЙ персоны:
- subagent_type: general-purpose
- model: opus
- Промт: прочитай спеку `~/workspace/ai-office-v2/demiurg/agents/interviewer.md`,
  затем протокол `~/.claude/skills/custdev/references/interview-protocol.md`.
  Карточка идеи: {вставь}. Карточка персоны: {вставь полностью, включая OCEAN}.
  Данные разведки рынка: {краткая выжимка — конкуренты, цены, Gap}.
  Проведи глубинное интервью. OCEAN определяет КАК персона отвечает (стиль, длина,
  скептицизм), а не ЧТО она отвечает. Выдай структурированную карточку результата.

---

## Фаза 5: Синтез

Собери все интервью + разведотчёт. Прочитай фреймворк:
`~/.claude/skills/custdev/references/synthesis-framework.md`

Действуй как Producer. Создай CustDev Report по шаблону из synthesis-framework.
Оцени каждую гипотезу через Value Equation (Hormozi).

## Ограничения (честно скажи владельцу)

- Синтетические персоны !== реальные люди. Для Tier 1 решений — после custdev нужен реальный созвон
- OCEAN-профиль влияет на стиль ответов, но не заменяет реальную психографику
- Не заменяет sales-debrief (custdev = ДО продажи, debrief = ПОСЛЕ)
