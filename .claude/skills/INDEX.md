# Скиллы офиса — индекс

Скиллы — это рецепты «когда / как делать что-то». Запускаются автоматически по триггерам или вручную через `/название-скилла`. Director видит этот индекс и знает что у него под рукой.

> Кастомные скиллы пользователя (положенные руками в `.claude/skills/`) **сохраняются** при `/update-office` — обновляются только перечисленные ниже системные.

---

## Системные (запускаются сами или через Director)

| Скилл | Триггер | Что делает |
|-------|---------|-----------|
| **`/setup`** | первое «привет», есть `{{CLIENT_NAME}}` | Живой онбординг — собирает имя/нишу/Notion/GitHub, заменяет плейсхолдеры, передаёт Стратегу |
| **`/install-agent <name>`** | «подключи помощника», `_agent-packs/<name>` существует | Устанавливает пак из `_agent-packs/` в `office/agents/`, обновляет `AGENTS.md`, добавляет `@-include` в корневой `CLAUDE.md` |
| **`/update-office`** | «обнови офис», «есть обновления?» | Безопасный pull свежей версии из GitHub-шаблона. Бэкапит ВСЕ пользовательские данные, восстанавливает поверх. Не теряет ничего. |
| **`/interview`** | размытая задача, «давай обсудим/подумаем», «нужна концепция» | Режим вопрос-за-вопросом. Собирает чёткое задание из размытого запроса. |

## Архитектор — сборка и аудит (входит в базовый офис)

| Скилл | Триггер | Что делает |
|-------|---------|-----------|
| **`/build`** | «собери агента», «создай помощника», «новый агент», «собери команду», «AI-офис с нуля», «wizard» | Архитектор собирает нового агента (SINGLE) или целый офис (TEAM) по 8-фазному конвейеру `Intake → Brief → Scout → Knowledge → Build → Validate → Iterate → Wire`. Score < 60 = переделка с нуля, ≥ 80 = в команду. Перед сборкой проверяет `_agent-packs/` — если готовый пак есть, направляет в `/install-agent`. |
| **`/audit-project`** | «проверь офис», «наведи порядок», «есть дыры», «аудит офиса» | Универсальный аудит офиса по 5 категориям (структура, агенты, knowledge, скиллы, интеграция) с приоритетами P0/P1/P2. Не интерпретируется «по наитию». |

## Поток сырья (inbox → правильное место)

| Скилл | Триггер | Куда раскладывает |
|-------|---------|-------------------|
| **`/intake`** | «разбери инбокс», файлы лежат в `inbox/` | По таблице из `office/protocols/INBOX.md` → `projects/`, `clients/`, `knowledge/` или оставляет в inbox с пометкой |
| **`/new-project`** | «новый проект», «начать проект» | Создаёт `projects/<slug>/` по шаблону `_template/` |
| **`/new-client`** | «новый клиент», «лид», «контакт» | Создаёт `clients/<slug>/` карточку B2B-клиента |
| **`/new-knowledge`** | «сохрани знание», «положи в базу» | Создаёт `knowledge/<slug>.md` + обновляет `knowledge/INDEX.md` |

## Стратег — 6-недельная программа

Запускаются последовательно. Каждый сохраняет состояние, следующий продолжает оттуда.

| Скилл | Когда | Артефакт |
|-------|-------|----------|
| **`/strategist-intake`** | старт онбординга | базовый каркас проекта, черновик `CLAUDE.md` |
| **`/strategist-unpack`** | после intake | `diagnostic-card.md`, заготовки audience/products/brand |
| **`/strategist-discovery`** | глубокая распаковка | дополненный diagnostic-card |
| **`/strategist-prepare`** | подготовка стратегии | `office/client-profile.md`, обновлённый `AGENTS.md` |
| **`/strategist-roadmap`** | синтез 6-недельного плана | `roadmap.md`, `roadmap.html`, `progress-index.md` |
| **`/strategist-checkin`** | еженедельно | обновлённый progress + `weekly-logs/week-N.md` |

| Скилл | Триггер | Что делает |
|-------|---------|-----------|
| **`/marketer-audience`** | «распакуй мою ЦА», «кто покупает» | Сегментирует аудиторию, собирает портреты |
| **`/marketer-funnel`** | «где деньги протекают», «дыры в воронке» | Карта воронки от трафика до повторных продаж |
| **`/marketer-ladder`** | «продуктовая линейка», «лестница» | Tripwire → core → premium |
| **`/marketer-revision`** | «провёл диагностику» | Ревизия: что работает / что нет |
| **`/marketer-checkin`** | еженедельно | Метрики маркетинга, что докрутить |
| **`/marketer-log-deal`** | «закрыл сделку», «новый клиент пришёл» | Лог сделки + извлечение паттерна |

## Дизайнер (опциональный пак — обычно установлен по умолчанию)

Перечислен в `office/agents/designer/skills/` после установки. Главные: `brand-onboarding`, `brand-revise`, `brand-reference`, `site-extract`, `design-critique`, `claude-design-prompt`, `claude-code-artifact`, `creative-brief`.

---

## Принципы

- **Скиллы для пользователя — невидимы.** Не говори «запусти скилл», просто запускай сам.
- **Один скилл = одна задача.** Если нужно делать что-то новое — пиши новый скилл, не размазывай существующий.
- **Кастомные скиллы пользователя** живут в `.claude/skills/` и **не перезаписываются** при `/update-office`. Обновляются только перечисленные выше системные.
