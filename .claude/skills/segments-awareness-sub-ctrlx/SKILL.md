# Уровни осознанности — Распределение по Шварцу — клон под `/segments-awareness`

> **Это клон.** Источник — `knowledge-alchemist/04-awareness-levels/SKILL.md` (не правится). Здесь — адаптация под пайплайн `audience/segments/`. Методология Шварца (4 уровня + Тема vs Продукт) **остаётся как есть**.
>
> **Контракт ввода:** оркестратор `/segments-awareness` запускает клон **по одному сегменту за раз** после того как Скилл №2 заполнил секции 0–2 dossier. Клон читает `audience/segments/{slug}/dossier.md` секцию 2.x (данные интервью — дословные цитаты респондентов) и `audience/segments/{slug}/awareness-workspace.md` если оркестратор уже подготовил рабочее пространство для ритуала Ctrl+X (см. `audience/segments/_pipeline/awareness-distribution.md`).
>
> **Контракт вывода:** заполненная матрица осознанности пишется **прямо в** `audience/segments/{slug}/dossier.md`:
> - Секция 3 — `#awareness`: 4 уровня с дословными цитатами (по 2-3 якорные цитаты на уровень) и % концентрации
> - Секция 3.5 — доминантный уровень + точка входа + длина воронки + стратегия
>
> Полная разметка тезисов по уровням (workspace до распределения) сохраняется в `audience/segments/{slug}/awareness-workspace.md` — это рабочая «доска Ctrl+X», не финальный артефакт. Шаблон `templates/awareness-matrix.md` используется как структурный референс для секции 3 dossier, не как отдельный output-файл. Сводная аналитика по всем 3 сегментам собирается оркестратором поверх индивидуальных вызовов клона.
>
> Берёшь данные интервью (тезисы от реальных или синтетических респондентов) и распределяешь каждый тезис по 4 уровням: Indifference → Awareness → Consideration → Choice. Принцип: люди не роботы — работай с вероятностями, не с формулами. Каждый тезис = дословная цитата, не перефразировка.

## Зона ответственности

Отвечаешь за:
- Классификацию тезисов респондентов по 4 уровням осознанности
- Различение ТЕМЫ (уровни 1-2) и ПРОДУКТА (уровни 3-4)
- Определение точки входа и выхода для каждого сегмента
- Формирование стратегии воронки на основе матрицы
- Сводную аналитику по всем 3 сегментам

НЕ отвечаешь за:
- Проведение самого Users Research → передай ai-custdev
- Сегментирование → передай 01-segmentation
- Создание контент-стратегии → передай следующий модуль

## Процесс работы

### Фаза 1: Приём данных Users Research

```
У тебя есть данные Users Research?
├── Да, полные (6 респондентов × 3 сегмента) → Фаза 2
├── Частично (есть часть данных) → "Скинь что есть, разберёмся"
├── Есть от AI CustDev (синтетические) → Отлично, берём
└── Нет данных → "Сначала проведи Users Research. Запусти ai-custdev"
```

Собери от пользователя:
1. Название продукта и ТЕМЫ (широкой)
2. 3 сегмента с теплотой
3. Тезисы респондентов (по 2 на сегмент, итого 6)

**Важно:** тезисы = дословные цитаты. Если пользователь пересказывает —
попроси оригинальные формулировки.

**Quality gate:** Есть продукт + тема + ≥3 сегмента с тезисами.

### Фаза 2: Обучение классификации

Перед распределением объясни пользователю ключевые различия:

**ТЕМА vs ПРОДУКТ:**
- Уровни 1-2: говорим о ТЕМЕ (заработок в сети, инвестирование, фитнес)
- Уровни 3-4: говорим о ПРОДУКТЕ (курс по автоворонкам, курс по инвестированию)

**Формула уровней:**
```
1. Indifference: "Мне вообще пофиг на эту тему"
2. Awareness: "Интересно, расскажите про эту тему"
3. Consideration: "Хм, а мне это подойдёт? Хочу ли я?"
4. Choice: "Точно ли этот продукт? Точно ли у этого продавца?"
```

**Маркеры-подсказки для каждого уровня:**

| Уровень | Ключевые слова в речи респондента |
|---------|----------------------------------|
| 1. Indifference | "не нужно", "не думал", "и так норм", "не интересно" |
| 2. Awareness | "а что это?", "слышал", "интересно", "надо бы почитать" |
| 3. Consideration | "мне нужно для...", "смогу ли", "подойдёт ли", "что если" |
| 4. Choice | "точно ли", "почему именно", "а не лучше ли", "выгоднее" |

**Quality gate:** Пользователь понимает разницу между уровнями.

### Фаза 3: Распределение тезисов

Для каждого сегмента:

1. **Возьми все тезисы** от 2 респондентов этого сегмента
2. **Пройди каждый тезис** и задай вопросы:
   - Это про ТЕМУ или про ПРОДУКТ? (определяет 1-2 vs 3-4)
   - Человек безразличен или уже интересуется? (1 vs 2)
   - Примеряет на себя или уже выбирает? (3 vs 4)
3. **Впиши в нужный уровень** дословно, пронумеруй
4. **Определи точку входа** — где больше всего тезисов
5. **Определи стратегию** — сколько уровней нужно пройти до покупки

**Правила классификации:**
- Один тезис = один уровень (не дублируй)
- Если непонятно куда — задай себе: "Человек ЗНАЕТ, ХОЧЕТ или ВЫБИРАЕТ?"
- Если тезис между двумя уровнями — ставь на НИЖНИЙ (консервативная оценка)
- Нумеруй: 1, 2, 3... внутри каждого уровня

**Quality gate:** Все тезисы распределены. Ни один не потерян. Для каждого сегмента
определена точка входа.

### Фаза 4: Аналитика и стратегия

**Для каждого сегмента определи:**

```
Точка входа: уровень {N} — {название}
Доминантный уровень: где больше всего тезисов
Длина воронки: от точки входа до уровня 5
Стратегия:
  Уровень 1→2: образовательный контент (осведомление о теме)
  Уровень 2→3: мотивационный контент (примерка, кейсы до/после)
  Уровень 3→4: доказательства (кейсы, отзывы, гарантии)
  Уровень 4→5: оффер (цена, дедлайн, CTA)
```

**Сводная таблица:** по всем 3 сегментам — точка входа, длина воронки,
приоритетная стратегия.

**Чеклист качества:**
```
□ Все тезисы распределены (ни один не потерян)
□ Тема и Продукт не перепутаны (1-2 = тема, 3-4 = продукт)
□ Awareness и Consideration разделены корректно
□ Точка входа определена для каждого сегмента
□ Стратегия соответствует точке входа
□ Тезисы дословные, не перефразированные
□ Нумерация внутри каждого уровня
БАЛЛОВ: _/7 (требуется 6+)
```

**Exit criteria:** Заполненная матрица осознанности для 3 сегментов + сводная
таблица + стратегия воронки для каждого сегмента.

## Формат вывода

Для каждого сегмента — заполненная матрица по шаблону `templates/awareness-matrix.md`.
Итого: 3 матрицы + сводная аналитическая таблица.

## Примеры

### Пример 1: Инвестирование (happy path)
```
Input: Данные Users Research по 3 сегментам (горячий: предприниматель,
  тёплый: специалист, холодный: трейдер-любитель). 6 респондентов.
  Продукт: курс по инвестированию. Тема: финансовая грамотность.

Output:
  Горячий (предприниматель):
    Indifference: 0 тезисов
    Awareness: 3 тезиса
    Consideration: 5 тезисов ← доминантный
    Choice: 4 тезиса
    Точка входа: уровень 3, длина воронки: 2
    Стратегия: мотивация + доказательства + оффер

  Тёплый (специалист):
    Indifference: 1 тезис
    Awareness: 4 тезиса ← доминантный
    Consideration: 3 тезиса
    Choice: 1 тезис
    Точка входа: уровень 2, длина воронки: 3
    Стратегия: ликбез + мотивация + доказательства + оффер

  Холодный (трейдер):
    Indifference: 3 тезиса ← доминантный
    Awareness: 2 тезиса
    Consideration: 1 тезис
    Choice: 0 тезисов
    Точка входа: уровень 1, длина воронки: 4
    Стратегия: образование + ликбез + мотивация + доказательства + оффер

Путь: Фаза 1 → 2 → 3 → 4
```


## Что делать если что-то пошло не так

- Все тезисы на одном уровне → респондент нерепрезентативен или ошибка классификации. Перечитай ответы, ищи нюансы
- Путаешь Awareness и Consideration → спроси: "Он ИЗУЧАЕТ тему или ПРИМЕРЯЕТ продукт на себя?"
- Нет тезисов на уровне 1 → нормально для горячего сегмента. У него нет безразличия к теме
- Тезис между двумя уровнями → ставь на нижний (консервативная оценка)
- Пользователь перефразирует → попроси оригинал: "Скинь дословно, как респондент сказал"
