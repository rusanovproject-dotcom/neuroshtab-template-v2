# install-agent — универсальный установщик помощников

## Зачем

Базовый офис пользователя минимальный: Director + Стратег + Архитектор. Остальные помощники (Дизайнер, Копирайтер, Продажник, Tech Lead и т.д.) лежат готовыми паками в `_agent-packs/`. Пользователь ставит их когда готов — одной командой, без ручного редактирования конфигов.

Этот скилл — универсальный установщик. Читает манифест пака (`_agent-packs/<name>/install.md`), применяет изменения, оповещает пользователя живым языком.

## Когда запускать

**Явно:**
- `/install-agent <name>` — прямая команда с именем пака

**Неявно (из фразы пользователя):**
- *"установи дизайнера"* / *"подключи копирайтера"* / *"добавь в команду продажника"*
- *"хочу дизайнера"* / *"нужен помощник по <тема>"* — если есть пак под эту тему, предложи установку
- *"какие помощники можно подключить?"* — покажи список доступных паков

**Не запускать:**
- Агент уже установлен (проверь `office/AGENTS.md`) — сказать пользователю «уже в команде», предложить переустановку если пользователь хочет свежие файлы
- Пака не существует в `_agent-packs/` — показать список доступных, честно сказать что такого пока нет

## Алгоритм

### Шаг 1. Определить имя пака

- Если вызвано как `/install-agent <name>` — берём `<name>`.
- Если из фразы — маппинг ключевых слов на имя пака:
  - «дизайнер», «designer», «визуал», «брендбук» → `designer`
  - «копирайтер», «copywriter», «тексты» → `copywriter` (если пак существует)
  - «продажник», «sales», «продажи» → `sales` (если существует)
  - «tech lead», «разработчик», «девелопер» → `tech-lead` (если существует)

Если ключевые слова неоднозначны — спроси живым языком: *«Ты про Дизайнера (визуал, лендинги) или про Копирайтера (тексты, посты)?»*

### Шаг 2. Проверить существование пака

**2a. Проверь локальный пак:** наличие `_agent-packs/<name>/install.md`.

**2b. Если локально нет — скачай с GitHub автоматически:**

- **Репозиторий паков:** `https://github.com/rusanovproject-dotcom/ai-office-packs`
- **Raw base URL:** `https://raw.githubusercontent.com/rusanovproject-dotcom/ai-office-packs/main`
- **API для списка файлов:** `https://api.github.com/repos/rusanovproject-dotcom/ai-office-packs/contents/_agent-packs/<name>`

Алгоритм:
1. Пользователю живым языком: *«Секунду, подтяну с репозитория.»*
2. Создай `_agent-packs/<name>/`
3. Через GitHub API получи рекурсивно список всех файлов пака
4. Каждый файл скачай: `curl -sL <raw-url> -o <local-path>` с сохранением структуры папок
5. Проверь что `_agent-packs/<name>/install.md` теперь локально
6. Если скачивание не удалось (нет сети / репо недоступно / пака нет в репо): *«Не смог подтянуть пак <name>. Проверь интернет или напиши владельцу офиса.»*

**2c. Если пака нет ни локально ни на GitHub:**
1. Через API получи список доступных паков: `https://api.github.com/repos/rusanovproject-dotcom/ai-office-packs/contents/_agent-packs`
2. Пользователю: *«У меня пока нет пака для <name>. Есть готовые: [список]. Что ставим?»*
3. Если список пустой: *«Пока готовых паков нет. Владелец офиса добавит — я подключу.»*

### Шаг 3. Проверить что агент ещё не установлен

Проверь `office/agents/<name>/` — существует ли папка.

**Если агент уже установлен:**
- Прочитай `office/AGENTS.md` на предмет строки про этого агента.
- Скажи пользователю: *«<Human name> уже в команде. Хочешь переустановить свежие файлы? Твои персональные правила и накопленная память я не трону — только обновлю ядро и знания.»*
- Если пользователь соглашается — переходи к шагу 4 в режиме переустановки (флаг `reinstall = true`, `preserve_if_exists` работает).
- Если отказывается — стоп.

### Шаг 4. Прочитать манифест

Открой `_agent-packs/<name>/install.md`. Извлеки:
- `agent_id`, `agent_name_human`, `agent_name_in_chat`, `short_role`
- `files:` — список src→dest + флаги `preserve_if_exists`
- `folders:` — папки которые надо создать если их нет
- Строки для `office/AGENTS.md`, корневого `CLAUDE.md`, `director/core.md`, `routing-patterns.md`
- `first_task` и блок «Message to client»

**Если манифест битый** (нет agent_id / нет раздела files / нет сообщения пользователю):
> *«Пак <name> выглядит неполным — пропущены важные поля. Скажи владельцу офиса, он доправит. Пока устанавливать не буду — не хочу оставить офис в странном состоянии.»*

### Шаг 5. Скопировать файлы

Для каждой записи в `files:`:
- Создай папку назначения если её нет.
- Если `preserve_if_exists: true` и файл назначения существует — **пропусти**, не затирай.
- Иначе — скопируй из `_agent-packs/<name>/<src>` в `<dest>`.

Папки (`knowledge/`, `skills/`, `templates/`) копируй рекурсивно.

Для `folders:` — создай если не существует, оставь существующие README нетронутыми.

### Шаг 6. Обновить `office/AGENTS.md`

Прочитай файл. Найди таблицу «Установленные агенты». Добавь строку из манифеста **после последней строки таблицы** (перед пустой строкой / разделителем).

Если в манифесте указана строка «Прямое обращение» — найди строку `**Прямое обращение** к агенту:` и допиши `@<agent_id>` если его там ещё нет.

Если в манифесте есть блок «Онбординг» для этого агента — вставь его в подходящее место (перед `---`).

Если агент уже был в таблице (защита от дубликатов) — **не дублируй строку**, оставь как есть.

### Шаг 7. Обновить корневой `CLAUDE.md`

**Layered include:** найди блок `@office/agents/.../core.md` (строки идут подряд). Добавь `@office/agents/<name>/core.md` после последней такой строки, если её ещё нет.

**Таблица «Типовые роли»:** найди таблицу в секции «Команда». Добавь строку из манифеста после последней существующей, если её ещё нет.

### Шаг 8. Обновить `office/agents/director/core.md`

**В секцию «Команда офиса»** (список «Типовые роли» — буллеты `- **Название** — описание`) — добавь строку из манифеста.

**В секцию «Роутинг»** — добавь строку из манифеста в «Базовое правило маршрутизации».

**Если в манифесте есть спецсекция (триггер-блок)** — вставь её перед секцией `## Output contract`. Проверь что такая секция ещё не вставлена (защита от дубликатов по заголовку).

### Шаг 9. Обновить `office/agents/director/knowledge/routing-patterns.md`

**В таблицу «Core-роутинг»** — добавь строки из манифеста, если их ещё нет.

**В секцию «Правила»** — добавь пункты из манифеста если они есть.

**В «Параллельный / последовательный режим»** — добавь связки из манифеста.

### Шаг 10. Оповестить пользователя

Покажи сообщение из блока «Message to client» манифеста. Живым языком, без IT-терминов.

Не произноси вслух:
- «скилл», «YAML», «install.md», «layered include», «манифест», «пак», «core.md», «routing-patterns»
- «агент», «artifact», «prompt», «слаш-команда»

Говори:
- «помощник», «подключил», «добавил в команду»
- «шаблон», «инструкция» (вместо промт)
- «живая страница» (вместо артефакт)

В конце сообщения — предложи первую задачу из `first_task.suggestion` одной фразой:
> *«Давай начнём с <first_task.suggestion>? <first_task.why> Скажи "<пример фразы>" и я запущу.»*

### Шаг 11. Самопроверка

- [ ] Папка `office/agents/<name>/` существует и содержит `core.md`
- [ ] В `office/AGENTS.md` есть строка про агента
- [ ] В корневом `CLAUDE.md` есть `@office/agents/<name>/core.md`
- [ ] В `director/core.md` есть упоминание агента в «Команде офиса»
- [ ] В `routing-patterns.md` есть триггер

Если что-то не прошло — скажи пользователю:
> *«Подключил, но заметил что <что именно> не обновилось. Проверь с владельцем офиса — возможно, файл был нестандартной структуры. Помощник в команде, работать должен.»*

## Guardrails

- **Не затирать `overrides.md` / `memory.md` / `failures.md`** если они уже есть. Это персональные данные пользователя и накопленная память агента — потеря = критично. Проверяй флаг `preserve_if_exists`.
- **Идемпотентность.** Повторная установка не должна дублировать строки в `AGENTS.md` / `CLAUDE.md`. Перед вставкой — проверка на наличие.
- **Битый манифест — стоп.** Если полей не хватает — не пытайся «додумать», скажи пользователю связаться с владельцем офиса.
- **Не спрашивай подтверждение на каждый файл.** Пользователь один раз сказал «установи» — ставь весь пак. Подтверждение нужно только при переустановке (будут затронуты ядро и знания).
- **Не упоминай в разговоре технические детали.** Пользователь не должен слышать: «сейчас пропатчу CLAUDE.md», «обновлю routing-patterns», «применю install.md». Говори: «подключаю помощника».
- **Закрытый пак.** Если в `install.md` есть `requires: [<other-agent>]` — проверь что те агенты уже установлены. Если нет — скажи пользователю: *«Для подключения <name> сначала нужен <required>. Хочешь, сначала его поставлю?»*
- **Эскалация при ошибке.** Если в процессе копирования что-то сломалось (нет прав, файл занят) — откати начатое где возможно и скажи пользователю что не получилось и почему.

## Язык пользователю — примеры

**Хорошо:**
- *«Подключаю Дизайнера. Секунду.»*
- *«Готово, в команде появился Дизайнер. Умеет [X]. Начнём с Brand Book?»*
- *«У меня пока нет пака для "аналитика". Есть готовые: Дизайнер. Что из этого?»*
- *«Дизайнер уже в команде. Переустановить свежие файлы? Твои правки и память я не трону.»*

**Плохо (не говорить):**
- «Скилл install-agent сейчас применит install.md пака»
- «Обновляю layered include в CLAUDE.md»
- «Пропатчу routing-patterns и core.md Директора»
- «YAML манифест прочитан, файлы скопированы»
