# Intake — разбор инбокса

Пользователь кидает в `inbox/` всё непонятное. Твоя задача — прочитать, классифицировать, разложить.

## Шаг 1. Прочитай содержимое inbox/

```
ls inbox/
```

Для каждого файла (кроме `README.md`):
- Прочитай содержимое (Read tool).
- Если файл бинарный/изображение — определи по имени и контексту.
- Если ссылка (`.url`, `.webloc`, или просто `.txt` со ссылкой) — посмотри URL.

## Шаг 2. Классифицируй по таблице

Таблица живёт в `office/protocols/INBOX.md`. Проверь её и используй правила оттуда.

Базовая логика:

| Что это | Куда |
|---------|------|
| Задача / ТЗ / "надо сделать X" | `projects/<relevant>/` или новый проект через /new-project |
| Клиент / лид / контакт (B2B-клиент пользователя) | `clients/<slug>/` через /new-client |
| Знание / статья / референс / методика | `knowledge/<slug>.md` через /new-knowledge |
| Бриф / идея проекта | `projects/<name>/brief.md` |
| Скриншот без контекста | оставить в inbox, пометить — спросить пользователя |
| Неясно | оставить в inbox + создать `_questions.md` с вопросом |

Для каждого файла определи слот и обоснование.

## Шаг 3. Покажи план пользователю

НЕ перемещай сразу. Сначала покажи таблицу:

> Вот что нашёл в inbox — **N файлов**. Предлагаю разложить так:
>
> | Файл | Куда | Почему |
> |------|------|--------|
> | note-about-ivan.md | clients/ivan-petrov/ | похоже на карточку лида |
> | gpt-prompts.md | knowledge/llm-prompts.md | коллекция промптов |
> | redesign-idea.txt | projects/redesign/brief.md | бриф нового проекта |
>
> Ок? Если что-то не так — скажи, поправлю.

Жди подтверждения. Если пользователь говорит "да/ок/давай" — двигайся дальше. Если правит — подстройся.

## Шаг 4. Переместить файлы

После подтверждения:

1. Для случаев где нужен новый проект / клиент (B2B) / знание — запусти соответствующий скилл (`/new-project`, `/new-client`, `/new-knowledge`), используй данные из файла как вход.
2. Для остальных — просто перемести (`mv` через Bash или создай новый файл в нужном месте и удали старый).
3. Если в inbox ничего не было — скажи "пусто, всё чисто" и выйди.

## Шаг 5. Обнови индексы

Где обновляем:
- `clients/INDEX.md` — если добавили клиента (B2B)
- `knowledge/INDEX.md` — если добавили знание
- (проекты индексного файла не имеют — пропускаем)

## Шаг 6. Отчитайся

Коротко:

> Разложил. Перемещено — 4 файла. Осталось в inbox — 1 (скриншот без контекста, скажи что это и разложу).
>
> Обновлены: clients/INDEX.md, knowledge/INDEX.md.

## Правила

- Не трогай `README.md` в inbox/ — это инструкция для пользователя, она там живёт.
- Если файл важный, но контекста не хватает — не гадай, спроси пользователя.
- em-dash `—`, не `--`.
- Тон живой, без канцелярита.
