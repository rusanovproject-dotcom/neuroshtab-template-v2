## Топ-5 механик для инфобиза (самые частые)

### 1. Убить работы
Клиенту НЕ надо делать X — продукт это делает за него или устраняет необходимость.
- **Когда применять:** в графе работ есть шаги которые клиент ненавидит или боится, и продукт их убирает
- **Пример инфобиз:** Пример: продукт убивает работу «самому разбираться в рекламе по YouTube и форумам» — даёт пошаговую систему с куратором
- **Пример другая ниша:** ChatGPT убил работу «знать испанский» для перевода текстов

### 2. Упростить работы
Та же работа, но когнитивная нагрузка ниже. Canva vs Photoshop.
- **Когда применять:** клиенты жалуются на сложность текущих решений
- **Пример инфобиз:** Пример: «короткие видео, по шагам, по полочкам» vs другие курсы где «информации много, ничего не понятно»
- **Пример другая ниша:** Tilda vs кастомная разработка сайта

### 3. Убрать налоговые работы
Устранить побочные работы, которые появились из-за плохого текущего решения.
- **Когда применять:** в проблемах текущих решений (шаг 07) есть налоговые работы
- **Пример инфобиз:** Пример: продукт убирает налоговую работу «искать допматериалы в YouTube потому что на курсе не объяснили» — куратор проверяет каждое ДЗ
- **Пример другая ниша:** Notion убрал работу «синхронизировать 5 разных файлов вручную»

### 4. Снизить риск
Гарантия, бесплатный период, «попробуй до покупки».
- **Когда применять:** главный барьер — страх («а вдруг опять не получится»)
- **Пример инфобиз:** Пример: бесплатный мини-курс + практический урок до покупки. Снимает барьер «это обман»
- **Пример другая ниша:** SaaS — бесплатный trial 14 дней

### 5. Выполнить работу качественнее
По критериям КЛИЕНТА (не своим!). Критерии берём из jobStory (блок ХОЧУ, критерии «как»).
- **Когда применять:** клиент уже пробовал решение, оно сработало частично — но не дотянуло по качеству
- **Пример инфобиз:** другие курсы учат, но не учат искать клиентов. Пример: модуль продаж + работа за кейс + чат с вакансиями
- **Пример другая ниша:** премиум-стоматология vs районная поликлиника

📖 ОБЯЗАТЕЛЬНО ПРОЧИТАЙ: `references/value-mechanics.md` (целиком, 198 строк)
Полный список ~20 механик + формула ценности + стратегические ходы + порождённые работы. Прочитай, потом вернись и выбери 2-3 самые применимые.

### Ещё 5 механик (менее частые, но мощные):

**6. Выйти на работу выше** — Big Job вместо Core. Клиент вырос, хочет больше. LiFT: от набора услуг к «бренд под ключ» → чек ×3. Инфобиз: от курса к менторингу.

**7. Выйти на предыдущую работу** — забрать работу ДО. Перехватить клиента раньше. Домклик: «рассчитать кредит» до «оформить ипотеку». Инфобиз: тест «какая профессия» до мини-курса.

**8. Больше работ одним решением** — «всё в одном». Notion = Trello + Docs + Excel. Пример: навык + поиск клиентов + упаковка в одном курсе.

**9. Снизить входной барьер** — бесплатный период, демо. SaaS: trial 14 дней. Пример: бесплатный мини-курс.

**10. Ускорить работы** — та же работа, быстрее. AI-копирайтер: 5 мин vs 2 часа. Пример: «первый результат на 4-5 неделе».

## Пример полного описания механики для B2B (упаковщик)

```
Механика: «Убить работы»

Почему: из графа — владелец школы сам пишет лендинг + оффер + тексты. 
Тратит недели, результат слабый. Из CS: «сделаю сам» = налоговая работа 
«переделывать 5 раз».

Доказательство: клиент 3 мес писал лендинг, конверсия 0.8%.
После упаковки — 3.2%. Время: с 3 мес до 2 недель.

Что даёт: не тратить время на то в чём не эксперт. Фокус на продукте.
```

## Критерии ВЫБОРА механик

Не гадай — привязывай к данным предыдущих шагов:

| Что видишь в данных | Какая механика подходит |
|---|---|
| В графе работ есть шаги которые продукт убирает | **Убить работы** |
| В проблемах текущих решений — налоговые работы | **Убрать налоговые работы** |
| Клиенты жалуются на сложность | **Упростить** |
| Главный барьер — страх/недоверие | **Снизить риск** |
| Конкуренты делают ту же работу, но хуже | **Выполнить качественнее** |
| Можно решить более крупную задачу | **Выйти на работу выше** |
| Можно перехватить клиента раньше | **Выйти на предыдущую работу** |

## Для каждой выбранной механики

- **Название простым языком** (термин в скобках)
- **Почему именно эта** — что в продукте и ситуации клиента указывает на эту механику (конкретная ссылка на данные из предыдущих шагов)
- **Доказательство** — конкретный пример из практики: клиент пришёл с X, до этого Y, не сработало Z, у нас W
- **Что это даёт клиенту** — на его языке

## Пример полного описания механики

```
Механика 1: «Убрать лишние шаги» (Убить работы)

Почему: в Consideration Set главный конкурент — «ничего не делать». 
Побеждает потому что любое действие = напряг. Из проблем текущих решений: 
курсы конкурентов заставляют «самой разбираться после оплаты» (налоговая работа).

Доказательство: Айнагуль купила курс — «вообще не понимаю, куратор говорит: 
я вам продала, сами учитесь». Пример: куратор проверяет каждое ДЗ, 
AI-куратор отвечает 24/7, видео короткие и пошаговые.

Что даёт клиенту: не надо самой искать информацию, не надо гадать 
«правильно я делаю или нет». Всё разжёвано и проверено.
```

## Если не хватает фактуры для доказательств

Вопросы на ДОКАЗАТЕЛЬСТВА, не на процесс:

✅ «Был клиент, который до тебя пробовал [альтернативу] и не получилось? Что именно не получилось и почему у тебя получилось?»
✅ «Есть клиент, который раньше бросал, а с тобой не бросил? Что его удержало?»
❌ «Опиши пошагово что делаешь на первой встрече» — процесс, не ценность
