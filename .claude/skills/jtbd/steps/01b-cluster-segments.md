# Шаг 01b. Группировка в кластеры

**⛔ ПРАВИЛО КОММУНИКАЦИИ: НЕ ПОКАЗЫВАЙ пользователю свои рассуждения, промежуточные классификации. Пользователь видит ТОЛЬКО финальный результат — список групп с описанием + вопрос.**

## Цель

Из собранной фактуры (черновик шага 01a) сгруппировать клиентов в кластеры по похожим запросам и ситуациям. Это предварительные гипотезы сегментов — уточняются на шаге 02.

**Связь назад:** черновик из шага 01a (`сегменты-черновик.md`).
**Связь вперёд:** кластеры → шаг 02 (шапка сегмента + Big Job).

---

## Принцип: сегментация по работам, не по демографии

JTBD-методология: «Можно сегментировать людей по парам работа+решение, и это самая удобная сегментация для принятия продуктовых решений.»

На этом шаге работы ещё не определены — скилл видит только запросы и ситуации. Поэтому кластеризация **предварительная**. Финальная сегментация уточняется на шаге 02.

**JTBD-методология, раздел 9.6.1 — алгоритм группировки:**
1. Для каждого человека: сегмент (интересно) / не-сегмент (с причиной) / не подходит
2. Группировка похожих запросов
3. Усреднение — для каждой группы описать типичную ситуацию

**«Про каждый не-сегмент ищи причину почему он не-сегмент»** — это критично.

Группируем по:
- **Что хотят получить (запрос/работа)** — ГЛАВНЫЙ признак. Это то, ради чего приходят
- В какой ситуации находятся (контекст) — дополнительный признак
- Что пробовали до этого — дополнительный

**Тест:** если два клиента хотят РАЗНОГО (один — сменить профессию, другой — подработку) — это разные группы, даже если оба в декрете. Если хотят ОДНОГО (оба — финансовую независимость) — одна группа, даже если один в декрете, другой в найме. Запрос первичен, ситуация вторична.

НЕ группируем по:
- Полу, возрасту, городу — демография не объясняет причину покупки
- Прошлому опыту с другими курсами — это характеристика клиента, не отдельный сегмент
- Ситуации без привязки к запросу — «все мамы в декрете» ≠ сегмент, если хотят разного

## Алгоритм

### 1. Перечитать черновик

**tool call: view** `сегменты-черновик.md` — прочитать ВСЮ собранную фактуру. **Если файл больше 500 строк — читай порциями по 500 строк (view с range). НЕ пытайся за один вызов. Частичное чтение = мусор на выходе.**

### 2. Сгруппировать по ЖЕЛАЕМЫМ РЕЗУЛЬТАТАМ (сам, молча)

Посмотри в черновике поле «желаемый результат» у каждого клиента. Группируй тех, кто хочет ОДНОГО И ТОГО ЖЕ. Не тех, кто в одной ситуации.

**tool call: str_replace** → дописать в черновик:
```
=== КЛАСТЕРЫ ===

Кластер А: [рабочее название — по запросу, не по ситуации]
- Общее: [какой ЖЕЛАЕМЫЙ РЕЗУЛЬТАТ объединяет]
- Кто входит: [имена + их желаемый результат из черновика]
- Размер: [N человек]

Кластер Б: [рабочее название]
...

Не вошли в кластеры:
- [кто и почему — желаемый результат уникальный]
```

**Если у людей в одном кластере РАЗНЫЕ желаемые результаты — это разные кластеры.** Не сливай их по ситуации. «Все в найме» — это не кластер. «Все хотят уйти и быть с детьми» — это кластер.

### 3. Оценить кластеры (сам, молча)

**tool call: str_replace** → дописать:
```
=== ОЦЕНКА ===
Кластер А: [N] человек. [частотный / единичный].
Кластер Б: [N] человек.
Порядок работы (от самого частотного): [А → В → Б]
```

### 4. Показать пользователю

**tool call: view** `сегменты-черновик.md` — перечитать перед показом.

**Два блока — сначала контекст, потом результат:**

> «Я разобрал всю фактуру и нашёл группы людей с похожими запросами. Это нужно, чтобы понять кто твой самый ценный клиент и как с ним разговаривать — в оффере, на лендинге, в рекламе. Если группы определены неверно, дальше всё поедет. Поэтому мне нужна твоя проверка.
>
> Нашёл [N] групп:
>
> 1. **[Рабочее название]** — [1-2 предложения: кто, ситуация, запрос]. Таких: [N].
>
> 2. **[Рабочее название]** — [1-2 предложения]. Таких: [N].
>
> Проверь: это действительно разные типы клиентов? Может, я кого-то пропустил или неправильно объединил?»

**Не показывать:** номера кластеров, внутреннюю оценку, термин «кластер», рассуждения.

### 5. Учесть ответ пользователя

- Подтвердил → записать в основной документ, перейти к шагу 02
- Добавил группу → дописать, показать обновлённый список
- Объединил/разделил → скорректировать, показать
- «Нет, не так» → спросить: «А как ты видишь своих клиентов? Кто приходит чаще всего?»

**tool call: str_replace** → записать финальный список в `JTBD_анализ_[продукт].md`. Обновить прогресс. Перейти к шагу 02a для первого сегмента.

## ЗАПРЕТЫ

- **НЕ определяй Big Job на этом шаге.** Только группировка. Big Job — шаг 02.
- **НЕ сегментируй по демографии.** Пол, возраст, город — не основа.
- **НЕ группируй по ПРОШЛОМУ ОПЫТУ.** «Пробовали другие курсы» — это характеристика, не сегмент.
- **НЕ выкидывай не-сегменты.** Записывай с причиной.
- **НЕ показывай рассуждения.** Только результат.
