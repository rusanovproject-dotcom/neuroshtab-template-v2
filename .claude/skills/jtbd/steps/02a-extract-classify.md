# Шаг 02a. Извлечь и классифицировать запросы сегмента

**⛔ ПРАВИЛО КОММУНИКАЦИИ: весь этот шаг выполняется МОЛЧА. Пользователь НЕ видит: карточки клиентов, классификации, типы запросов, промежуточные выводы. Единственное что видит — вступительное сообщение (один раз на весь шаг 02).**

## Где ты сейчас (контекст)

Ты определяешь **главную задачу в жизни** (Big Job) для конкретного сегмента. Это то, РАДИ ЧЕГО клиенты этой группы вообще приходят — не то, что продукт для них делает, а чего они хотят достичь В ЖИЗНИ, за пределами продукта.

Определение Big Job состоит из трёх подшагов:
1. **02a (ТЫ ЗДЕСЬ)** — извлечь из фактуры что клиенты этого сегмента говорят, и классифицировать: это цель, процесс, средство или эмоция?
2. **02b** — если целей не хватает → задать пользователю вопросы «зачем? чтобы что?», проверить ответы
3. **02c** — из проверенных целей → сформулировать Big Job, проверить, показать пользователю, записать

Результат этого подшага: в черновике записаны все цитаты клиентов этого сегмента с классификацией + вывод: хватает ли данных или нужны вопросы.

**Связь назад:** кластеры из шага 01 (в файле `сегменты-черновик.md`).
**Связь вперёд:** классификация → шаг 02b (вопросы) или 02c (формулировка).

## Что СКАЗАТЬ пользователю (обязательно, ПЕРЕД началом, один раз на весь шаг 02)

**Визуальное форматирование обязательно.** Сообщение должно быть структурировано так, чтобы НЕ ВЧИТЫВАЯСЬ можно было понять что где: вот заголовок группы, вот объяснение что делаем, вот результат. Не полотно текста.

> **Группа 1 из [N]: [название группы]**
>
> **Что сейчас делаю и зачем**
>
> Смотри, есть то, что твой продукт делает для клиента — это понятно. Но есть ещё то, РАДИ ЧЕГО клиент вообще к тебе пришёл. Это разные вещи. И это «ради чего» всегда **за пределами продукта** — то есть клиент хочет чего-то, что можно решить чем угодно, не только твоим продуктом. Но он выбрал именно тебя.
>
> Простой пример: такси. Такси довозит из А в Б — это что продукт делает. Но клиент вызвал такси чтобы оказаться в нужном месте — на встрече, в аэропорту. «Оказаться в нужном месте» — это за пределами такси. Это можно решить и метро, и своей машиной, и пешком. Но он выбрал такси.
>
> Вот это «ради чего, за пределами продукта» — самое ценное для упаковки. Когда знаем — оффер и лендинг пишутся легко. Не знаем — получается вода.
>
> Сейчас покопаюсь в фактуре, возможно задам пару вопросов. Потом покажу что нашёл.

## Алгоритм

### 1. Узнать кто в кластере

**Этот шаг выполняется ЗАНОВО для каждого сегмента. Не переноси «уверенность» от предыдущего. Для первого сегмента фактуры хватило — для второго может не хватить. Каждый сегмент = чистый лист.**

**tool call: view** `сегменты-черновик.md`

Найти кластер, с которым работаем. Определить: кто в него входит, какие имена/номера из фактуры.

### 2. Перечитать ПОЛНЫЕ КАРТОЧКИ по клиентам этого сегмента

**tool call: view** — открыть ВХОДНЫЕ ФАЙЛЫ фактуры (JTBD_вход_*) и найти полные карточки клиентов из этого кластера. **Если файл больше 500 строк — читай порциями по 500 строк (view с range). НЕ пытайся за один вызов.**

**НЕ бери данные из черновика сегментов — там только имена и короткие описания. Иди в ПОЛНЫЕ карточки.** Из черновика ты знаешь только КТО в кластере. ЧТО они хотят — в полных карточках.

По каждому выписать: кто + ситуация + ЖЕЛАЕМЫЙ РЕЗУЛЬТАТ (дословно из карточки) + цитаты.

**Искать во ВСЕЙ фактуре** — диагностики, кейсы, отзывы, анкеты. Big Job может быть где угодно.

В кейсах обращать внимание: результат (что получил) — это обычно Core Job. Big Job — в контексте ВОКРУГ результата: «теперь не завишу от мужа», «уволилась из найма», «закрыла кредит». Ищи этот контекст.

### 3. Записать запросы в черновик

Если файл `big-job-черновик.md` ещё не существует → **tool call: create_file**. Если уже существует (работали с предыдущим сегментом) → **tool call: str_replace**, дописать в конец.

**ПРАВИЛО: черновик только дополняется. НИКОГДА не удалять, не перезаписывать, не очищать.** Все сегменты, все итерации, все версии — остаются в файле. Каждый блок помечен названием сегмента.

Записать:
```
=== СЕГМЕНТ: [название] — запросы клиентов ===
1. [кто, ситуация]: «[дословная цитата]»
2. [кто, ситуация]: «[дословная цитата]»
...
```

### 4. Классифицировать каждый запрос

**tool call: view** `big-job-черновик.md` — перечитать только что записанные запросы.

По каждой цитате определить тип:

**Процесс** — действие без конечной точки, нельзя сказать «сделано».
Маркеры: «навести порядок», «контролировать», «разобраться», «управлять», «обеспечивать», «развивать», «начать зарабатывать», «накапливать».
→ Нужно поднять через «чтобы что?» до конкретного результата.

**Средство** — инструмент, а не результат.
Маркеры: «получить план», «найти систему», «иметь алгоритм», «составить стратегию», «иметь сумму на счёте».
→ Нужно поднять: «план — зачем? чтобы что?»

**Эмоция** — состояние, а не достижение.
Маркеры: «чувствовать уверенность», «перестать тревожиться», «жить спокойно».
→ Нужно спросить: «а что должно измениться, чтобы он это почувствовал?»

**Core Job** — запрос, который продукт может полностью выполнить.
Тест: «продукт может это полностью сделать за клиента?» ДА → Core Job.
→ Нужно поднять: «а зачем ему это? чтобы что за пределами продукта?»

**Возможно Big Job** — конкретный результат В ЖИЗНИ клиента, за пределами продукта.
Тест: есть момент достижения? Продукт НЕ может полностью сделать?
→ Кандидат на Big Job ✅

**Примеры (из JTBD-методологии, для калибровки):**
- «Доехать из А в Б» → Core Job такси (продукт полностью делает)
- «Оказаться в нужном месте» → Big Job (продукт делает не полностью)
- «Обеспечить комфорт жизни» → слишком абстрактно, процесс
- «Сдать экзамен на гражданство» → Big Job ✅ (конкретный результат, момент достижения, за пределами курса немецкого)

### 5. Записать классификацию в черновик

**tool call: str_replace** → дописать:
```
=== СЕГМЕНТ: [название] — классификация ===
1. «[цитата]» → [ПРОЦЕСС / СРЕДСТВО / ЭМОЦИЯ / CORE JOB / ВОЗМОЖНО BIG JOB] — [почему]
2. «[цитата]» → [тип] — [почему]
...
Вывод: целей из фактуры достаточно для Big Job: [ДА / НЕТ]
Если ДА → переходи к шагу 02c
Если НЕТ → переходи к шагу 02b
```

**Критерий «достаточно»:** есть хотя бы 2-3 цитаты, прошедшие как «ВОЗМОЖНО BIG JOB». Если все цитаты — процессы («хочу уйти», «хочу поменять», «устала»), средства или эмоции — это НЕ достаточно. Иди в 02b и спрашивай. **НЕ ВЫДУМЫВАЙ Big Job из процессов. Не показывай пользователю кривую формулировку с пометкой 🟡 — лучше спроси.**
