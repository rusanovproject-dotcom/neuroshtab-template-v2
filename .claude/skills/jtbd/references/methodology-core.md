## 1. Формула работы

**Когда** [контекст + психологические особенности + прошлый опыт + активирующее знание + триггер]
**Хочу** [ожидаемый результат + критерии результата]
**Чтобы** [результат работы выше уровнем + позитивные эмоции / избегание негативных]

### Градация детализации по бизнес-задаче

**Минимальное описание — «хочу + глагол»:**
- Для микро-функциональности: «хочу закрыть попап», «хочу увидеть скидку»
- НЕ работает для стратегических решений

**Среднее описание — «хочу + глагол + существительное» + контекст:**
- Для продуктовых гипотез внутри существующего продукта
- НЕ достаточно для запуска нового продукта

**Полное описание — вся формула:**
- ОБЯЗАТЕЛЬНО для поиска сегмента, стратегических решений, упаковки продукта (оффер, лендинг, скрипты, воронка)
- Именно этот уровень скилл генерирует по умолчанию

## 2. Полный набор полей работы

**Контекст и триггер:**
- Контекст — в какой ситуации человек находится
- Психологические особенности
- Прошлый опыт
- Активирующее знание (опционально, для спящих работ)
- Триггер — в какой момент начинает действовать

**Ожидаемый результат:**
- Сам результат (что хочет получить)
- Критерии ожидаемого результата (как понимает что результат получен)

**Уровень выше:**
- Ожидаемый результат работы выше уровнем (зачем, чтобы что)
- Позитивные эмоции (как хочет себя чувствовать)
- Негативные эмоции (что испытывает пока не получил результат)

**Метрики работы:**
- Важность работы (1-10)
- Частотность (для частотных работ)
- Тип работы — частотная или последовательная

**Текущее решение:**
- Решение (что нанимает сейчас)
- Удовлетворённость (1-10)
- Ценность, Aha-moment, Стоимость
- Проблемы с решением
- Барьеры к решению
- Альтернативные решения

---

## 3. Два типа работ

**Частотные** — результат получается каждый раз: заказать продукты, доехать на такси, учить слова.

**Последовательные** — проект с этапами: покупка квартиры, обучение на разработчика, релокация.

Разница важна — у них разная структура графа работ и разные формулировки.

## 4. Core Job vs Big Job

**Core Job** — самая высокая работа, которую решение ПОЛНОСТЬЮ выполняет.
**Big Job** — работа выше уровнем, которую решение выполняет НЕ ПОЛНОСТЬЮ.

### Техника определения — итерация «зачем?»

Пример с такси:
- «Заказать такси» → это решение, не работа. Зачем?
- «Доехать из А в Б» → Core Job. Мы это полностью делаем. Зачем?
- «Оказаться в нужном месте» → уже Big Job. Не полностью.
- «Обеспечить комфорт жизни» → точно не наш уровень.

### Критерий Big Job: всегда снаружи продукта, в жизни клиента

Если работа формулируется через продукт («освоить курс», «получить доступ», «пройти модуль») — это НЕ Big Job. Big Job — то, ради чего клиент пришёл за продуктом.

### Тест на булшит: «мы правда это полностью делаем?»

- Если ответ «частично» или «с оговорками» — это не Core Job
- Два признака булшита: обобщающие слова («комфорт», «успех», «свобода», «трансформация») и невыполнимость гарантии

### Следствие для коммуникации

- Core Job → прямые глаголы: «научишься», «освоишь», «соберёшь», «настроишь»
- Big Job → глаголы цели: «чтобы вырасти», «чтобы удвоить доход»

## 5. Entry Job vs Monetization Job

**Entry Job** — работа, через которую клиент ВХОДИТ (для лид-магнита, рекламы, первого касания). Обычно простая, низкопороговая, с быстрым результатом.

**Monetization Job** — работа, на которой происходит МОНЕТИЗАЦИЯ (для тарифов, удержания, повторных продаж). Обычно более глубокая, требует больше ресурсов.

### Кейсы

**Aviasales:** Entry Job = «найти дешёвые билеты» (бесплатно). Monetization = комиссия от авиакомпаний за переход.

**Домклик (Сбер):** Entry Job = «рассчитать кредит, найти квартиру» (бесплатный сервис). Monetization = оформить ипотеку в Сбере.

**Курс по AI (инфобиз):** Entry Job = «узнать что AI может для моего бизнеса» (бесплатный вебинар/статья). Monetization = «освоить систему внедрения AI» (платный курс).

### Применение в скилле

Для каждого сегмента определять:
- Через какую работу клиент входит → материал для лид-магнита и рекламы
- На какой работе монетизация → материал для тарифов и оффера

## 6. Тройная формулировка проблемы

Проблема в Advanced JTBD — это НЕ абстрактное «у клиента проблема». Проблема = следствие плохого текущего решения. Описывается тройной формулировкой:

1. **Просадка дофамина** — клиент ожидал результат, получил меньше. Физиологически неприятно. «Думал будет крутой курс — а тут вода»
2. **Негативные эмоции** — разочарование, фрустрация, стыд, тревога, злость
3. **Налоговые работы** — дополнительные работы, которые клиент вынужден делать из-за недостатков решения. «Приходится самому разбираться в настройках», «нужно нанять ещё одного человека», «трачу 3 часа на то, что должно занимать 15 минут»

### Применение

При описании Consideration Set для каждого текущего решения указывать все три компонента проблемы. Это даёт точки приземления для оффера: «у вас [налоговая работа]? Мы её убираем».

## 7. Диагностический чек-лист: нужна ли JTBD-сегментация?

### При запуске нового продукта (хотя бы 1 из):
- Продукт не продаётся
- Клиенты не понимают зачем покупать
- Нет аргументации против конкурентов
- Клиенты покупают, но быстро отваливаются
- Не сходится юнит-экономика

### При работе с существующим продуктом (хотя бы 1 из):
- Продукт перестал расти
- Конкуренты растут быстрее
- Теряем долю рынка
- Работаем на сложных/убыточных клиентов
- Часть клиентов убыточные

**Правило:** если 2+ симптома → высоковероятно проблема с сегментацией. Если 1 → проверить альтернативные гипотезы. Если 0 → не навязывать JTBD.

**Важно:** симптомы коррелируют с проблемой, но не гарантируют её. «Продукт не продаётся» может быть из-за сломанного платёжного шлюза.

## 8. Границы применимости Advanced JTBD

### Когда Advanced JTBD НЕ работает или работает частично:

**Дофаминовые продукты** — соцсети, гемблинг, порно, аддиктивные игры. Мотивация через зависимость, не через работы. Advanced JTBD покроет только часть мотивации.

**Неосознаваемые продукты** — еда (вкусовые предпочтения), одежда (мода), аксессуары (статус). Большая часть выбора неосознаваема.

**Продукты на чистом статусе** — формат «закрытое комьюнити элитных предпринимателей» имеет признаки неосознаваемой мотивации (принадлежность, статус).

### Когда Advanced JTBD работает полностью:
- Обучение, ПО, услуги, B2B
- Любой продукт, где клиент осознанно решает конкретную задачу
- Инфобиз (основной контекст JTBD-методологии)

**При определении продукта клиента скилл явно классифицирует:** Advanced JTBD полностью применим / частично применим / не рекомендуется.

## 9. Принцип антиинфоцыганства

Правило JTBD-методологии: «Не обещай того, что не можешь выполнить полностью. Останавливайся на том уровне, который полностью можешь выполнить».

### Глаголы для коммуникации:

**Для Core Jobs (полностью выполняем):** «ты освоишь», «ты научишься», «ты разберёшься», «ты соберёшь» — прямое обещание.

**Для Big Jobs (не полностью):** «чтобы», «для того чтобы», «что позволит» — связка через цель, без прямого обещания.

**Красный флаг:** формулировки «ты заработаешь миллион», «ты запустишь успешный бизнес» для продукта, который не может это гарантировать.

### Записи vs живой формат

Записи хуже выполняют Core Job (нет дедлайнов, обратной связи, энергии группы). Для записей:
- Спускать обещания: «ты получишь систему», «ты узнаешь как» (не «ты научишься применять»)
- Запрашивать компенсирующие механики (кураторы, ДЗ, чат, лайвы)

## 10. Декомпрессия абстрактных слов

Абстрактные слова в формулировке работы — ЗАПРЕЩЕНЫ. Каждое абстрактное слово — это компрессированный контейнер, скрывающий 3-5 разных подсегментов.

### Вопросы-помощники для декомпрессии:
- «А [слово] — это какое?»
- «А какой [объект] ты считаешь [слово]?»
- «А какой НЕ считаешь? Почему?»
- «По каким критериям поймёшь что [результат] достигнут?»

### Пример — слово «качественная» (мебель):
- Для одних = прослужит 20 лет → сегмент «долговечность»
- Для других = немецкая фурнитура → сегмент «страна-бренд»
- Для третьих = выдержит арендаторов → сегмент «сдающие квартиру»
- Для четвёртых = итальянская фурнитура → сегмент «верящие в Италию»

### Пример декомпрессии критериев успеха:
- «Восполню компетенции в маркетинге» → «По каким критериям поймёшь?»
  - → «Смогу пересобирать упаковку самостоятельно»
  - → «Смогу понимать кого нанимать и как оценивать»
  - → «Смогу использовать AI по максимуму»

Это конкретные критерии, пригодные для оффера.

## 11. Алгоритм раскопки джобы (9 шагов)

### Шаг 1. Big Job — определить верхнеуровневую работу
Что сегмент вообще хочет в жизни/бизнесе?

### Шаг 2. Core Job — спуститься до работы, которую продукт полностью выполняет
Итерация «зачем?» вверх, пока продукт не перестаёт полностью справляться.

### Шаг 3. Граница Core vs Big
«Наш продукт полностью выполняет эту работу?» Да → Core. Нет → Big, спускайся.

### Шаг 4. Декомпрессия критериев
«По каким критериям поймёшь, что работа выполнена?» Раскапывать каждое абстрактное слово.

### Шаг 5. Контекст
«В какой ситуации находился, когда решил это сделать?»

### Шаг 6. Триггер
«В какой момент начал действовать? Что стало триггером?» Конкретное событие.

### Шаг 7. Позитивные эмоции
«Как хочешь себя чувствовать после выполнения работы?»

### Шаг 8. Негативные эмоции
«Что испытываешь пока не выполнил?» Подсказки: тревога, неполноценность, обесценивание, страх упустить.

### Шаг 9. Собрать в формулу
Все компоненты → формула «Когда / Хочу / Чтобы».

**Если фактуры не хватает для шага — помечать «требует уточнения», не выдумывать.**

## 12. 13 базовых принципов Advanced JTBD

1. **Работа — то, ради чего нанимают продукт.** Не продукт, не фича, не проблема.
2. **Работы → Ценность → Деньги.** Сначала работы, потом ценность, потом продажи.
3. **Лёгкость коммуникации = диагностика правильности джобы.** Тяжело пишется лендинг → плохая JTBD, не плохой копирайтер.
4. **Цена ошибки определяет точность данных.** Маленький магазин может опереться на AI-гипотезы. Крупный ритейлер — нет.
5. **Коммуницируй выполнимый уровень.** Не обещай Big Job как Core Job.
6. **Людям не нужны наши продукты.** Фокус на работах клиента, не на продукте.
7. **Ответы в головах клиентов, не создателей.** Аналитика продукта ≠ источник правды.
8. **Работа ≠ решение.** Работа = цель. Решение = чем выполняет. Одна работа — много решений.
9. **Сегментация по работам, не по демографии.** Демография — дополнительный критерий, не основа.
10. **Привычное решение побеждает по умолчанию.** Новое решение должно быть заметно выгоднее.
11. **Действие — единственный сигнал работы.** Брать тех, кто УЖЕ потратился, не фантазёров.
12. **Выбор сегмента = выбор судьбы.** Маленький убыточный сегмент → потерянные годы.
13. **Работы первичны для абсолютно всего.** Лендинг, фичи, каждая кнопка обслуживает работы фокусных сегментов.

## Уровни достоверности данных

- 🟢 **Факт** — подтверждено фактурой (интервью, CRM, записи)
- 🟡 **Экспертная гипотеза** — на основе знания рынка и аналогий
- 🔴 **AI-генерация** — результат нейросети, требует обязательной валидации

**Всё что генерирует скилл без фактуры от клиента = 🔴 AI-генерация.**
