# New Knowledge — карточка знания

## Шаг 1. Собери или извлеки данные

Два режима:

**Режим А — из контекста.** Если знание уже обсуждалось в текущем чате или лежит в `inbox/` — не допрашивай. Извлеки:
- Тема (о чём)
- Тип (статья/методология/шаблон/кейс)
- Краткое описание (1-2 предложения)
- Ссылка на источник (если есть)

Покажи пользователю что извлёк и уточни только недостающее.

**Режим Б — с нуля.** Если контекста нет — спрашивай живым диалогом, по одному:

1. **Тема.** "О чём знание — коротко?"
2. **Тип.** "Это что — статья/конспект, методология, шаблон для переиспользования, или кейс? (просто скажи ближе всего)"
3. **Описание.** "1-2 предложения сути — чтобы ты через месяц понял о чём это."
4. **Источник.** "Откуда взял? Ссылка, книга, инсайт, или своё? (опционально)"

## Шаг 2. Сделай slug

Из темы — латиница/транслит, lowercase, через дефис.
- "Как писать продающие заголовки" → `selling-headlines`
- "Воронка B2B-продаж Hormozi" → `hormozi-b2b-funnel`
- "Промпт для research" → `research-prompt`

Если slug занят — добавь суффикс или уточни.

## Шаг 3. Создай карточку

Файл — `knowledge/<slug>.md`. Содержимое — из шаблона `office/templates/knowledge-card.md`.

Плейсхолдеры:
- `{{TITLE}}` — заголовок (человекочитаемая тема)
- `{{TYPE}}` — тип
- `{{SUMMARY}}` — краткое описание
- `{{SOURCE}}` — источник или `—`
- `{{DATE}}` — сегодняшняя дата YYYY-MM-DD
- `{{CONTENT}}` — основное содержимое (если есть — вставь, если нет — оставь заголовок "## Суть" и заготовку)

Если знание пришло из inbox — перенеси основное содержимое в `{{CONTENT}}`.

## Шаг 4. Обнови INDEX.md

Файл — `knowledge/INDEX.md`. Таблица:

```
| Тема | Дата | Тег | Путь |
|------|------|-----|------|
```

Добавь:
```
| <TITLE> | <DATE> | <TYPE> | [<slug>](./<slug>.md) |
```

Если в таблице строка-плейсхолдер (`— | — | — | —`) — удали её.

## Шаг 5. Отчитайся

Коротко:

> Сохранил → `knowledge/<slug>.md`
>
> Тема: <TITLE>. Тип: <TYPE>. Индекс обновлён.

## Правила

- em-dash `—`, не `--`.
- Если источник спорный ("слышал где-то") — не придумывай ссылку, ставь `—`.
- Не раздувай карточку — короткая и полезная лучше длинной и размытой.
- Если пользователь дал длинный текст — положи его целиком в секцию `## Содержимое`, а `SUMMARY` сделай коротким.
