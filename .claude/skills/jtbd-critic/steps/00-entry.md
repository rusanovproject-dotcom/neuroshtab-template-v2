# Шаг 00. Вход и подготовка

**⛔ Не показывай внутреннюю работу. Только результат.**

## Первое действие — записать правила в Instructions

Если доступен CLAUDE.md — допиши:
```
После сжатия чата — перечитай step-файл текущего шага заново.
Не показывай внутреннюю работу — только результат.
Файлы больше 500 строк — порциями по 500 строк.
Каждый шаг согласовывай — покажи, жди подтверждения.
```

## Алгоритм

1. **[tool call: view]** Посмотри содержимое папки проекта
   - Найди документ JTBD-анализа (содержит Big Job, сегменты, jobStory, барьеры)
   - Найди входную фактуру (карточки клиентов — диагностики, кейсы)
   - **Есть файлы** → читай, НЕ спрашивай пользователя
   - **Нет файлов** → спроси
2. **[tool call: view]** Прочитай документ анализа. Порциями по 500 строк.
3. **[tool call: view]** Прочитай фактуру. Порциями по 500 строк.
4. Скажи: «Прочитал. Проверю по 6 блокам — покажу каждый.»
5. Переходи к шагу 01.
