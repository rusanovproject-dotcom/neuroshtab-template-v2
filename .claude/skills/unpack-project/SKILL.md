# /unpack-project — мета-оркестратор распаковки

## Зачем

Единая точка входа в маркетинговую распаковку проекта. После онбординга Стратегом у клиента есть скелет (CLAUDE.md, hypotheses.md, пустые папки audience/product/brand/funnel + опционально diagnostic-card.md). Дальше нужно глубоко распаковать ЦА, оффер, воронку и бренд — это делает Алекс Маркетолог через серию специализированных скиллов.

Без этого мета-скилла клиент сам бы запоминал какой скилл когда вызвать. Тут — один вход, 4 фазы по очереди, миграция файлов Стратега как стартовых гипотез, stop+wait чек-поинты между фазами.

## Когда запускать

**Явно:**
- *«распакуй проект»*, *«поехали»*, *«новый проект»*, *«начнём пайплайн»*, *«давай распаковку»*
- `/unpack-project`

**Неявно:**
- Сразу после `/install-agent alex-marketer` — если в `projects/<main>/` есть скелет от Стратега, но `audience/segments/NORTH-STAR.md` не создан → предложи запуск
- Клиент жалуется *«не знаю с чего начать после Стратега»* / *«Алекс установлен, что дальше?»* — предложи `/unpack-project`

**НЕ запускать:**
- Если Алекс не установлен → сначала *«Подключим Алекса: `/install-agent alex-marketer`. Он ведёт распаковку.»*
- Если уже идёт распаковка (есть незавершённая фаза) → продолжаем с того места, не запускаем заново
- Если в `projects/` только `_template/` → *«Создай проект через `/new-project`, потом распакуем.»*

## Pre-flight checks (КРИТИЧНО)

Перед стартом — пройди по пунктам, без них не начинай:

```
1. Алекс установлен?
   ls office/agents/alex-marketer/
   - Нет → «Сначала подключим Алекса: /install-agent alex-marketer. Он ведёт фазы 1-3.»

2. Активный проект?
   ls projects/ (исключая _template и _example-project)
   - Один → берём его как <main>
   - Несколько → спросить: «По какому проекту распаковываем — [перечислить]?»
   - Только _template → «Создай проект через /new-project, потом распакуем.»

3. Миграция от Стратега?
   ls projects/<main>/audience/diagnostic-card.md
   ls projects/<main>/product/ (любые .md кроме INDEX)
   ls projects/<main>/audience/ (черновики)
   - Если есть → читаем, помечаем как starter (НЕ перезаписываем)
   - Если нет → стартуем с нуля

4. Глубокий пайплайн или lite?
   ls projects/<main>/audience/segments/
   ls projects/<main>/product/_protocols/
   - Есть → глубокий пайплайн (полная мощность Алекса)
   - Нет → lite-режим, предупреди владельца: «У тебя упрощённый шаблон проекта.
     Алекс пройдёт по фазам, но без диалоговых протоколов глубины. Если хочешь
     полную мощность — обнови шаблон проекта через Архитектора.»
```

## Старт — план владельцу

После pre-flight, перед фазой 1 — короткое сообщение с планом:

> *«Сейчас проведу через 4 фазы:*
> *1. Аудитория — найдём ТОП-3 сегмента (40-60 мин)*
> *2. Продукт/оффер — соберём Grand Slam через 6 диалоговых протоколов (60-90 мин)*
> *3. Воронка — per-segment маршрут до сделки (40-60 мин)*
> *4. Бренд — пока пропускаем, агента-Брендмейкера ещё нет, оставим на ручное заполнение*
>
> *Между фазами — стоп. Согласишься «продолжаем» — пойдём дальше. Можем сделать всё за один заход или разбить на дни — как удобнее.*
>
> *Поехали с фазы 1?»*

**Без явного «да / поехали / продолжаем» дальше не идти.**

## Алгоритм — 4 фазы с stop+wait

### Phase 1 — Audience (40-60 мин)

```
1. Миграция от Стратега:
   - Если есть projects/<main>/audience/diagnostic-card.md:
     · Прочитай, используй как стартовую гипотезу о ЦА
     · В hypotheses.md → секция 🧊 Backlog добавь:
       «🌱 starter: ЦА из diagnostic-card.md (Стратег, YYYY-MM-DD)»
     · НЕ перезаписывай diagnostic-card.md — он остаётся как исходник
   - Если есть черновики audience/* от Стратега → так же читаем как input

2. Запусти /segments-discover (Алекс ведёт):
   - 8 этапов: контекст → разведка → 12 гипотез → ТОП-3 через 6-осевой scoring
   - Output: audience/segments/NORTH-STAR.md (ТОП-3) + MAP.md (backlog)
   - Stop+wait: «Согласен с ТОП-3? Можем ещё подумать или меняем приоритет?»

3. Для каждого из ТОП-3 сегмента:
   - /segments-unpack {slug} → dossier.md секции 0,1,2,5,6
   - /segments-awareness {slug} → dossier.md 3,3.5,4,7,8 + base.md
   - Stop+wait после каждого сегмента ИЛИ пакетно — спроси владельца:
     «Распаковываем все три по очереди со stop'ами или пакетом подряд?»

4. Финал фазы 1:
   - Запиши в projects/<main>/CHANGELOG.md строку про фазу
   - Сообщение: «Аудитория готова. ТОП-3: [названия]. Артефакты: NORTH-STAR.md,
     3 dossier.md, base.md. Переходим к Продукту?»
```

**Stop+wait → ждём согласия → Phase 2.**

### Phase 2 — Product (60-90 мин)

```
1. Миграция от Стратега:
   - Если есть projects/<main>/product/*.md (заготовки оффера / продукта):
     · Прочитай, используй как input
     · В hypotheses.md → 🧊 Backlog: «🌱 starter: продукт из заготовок Стратега»
     · НЕ перезаписывай — продукт-файлы Стратега остаются исходниками

2. Запусти /unpack-product (Алекс ведёт):
   - Внутри он запустит /product-build
   - 6 диалоговых протоколов из product/_protocols/ (если есть глубокий пайплайн)
     или lite-режим (если протоколов нет)
   - Output: core-offer.md (Grand Slam) + ladder.md + опционально proof.md
   - Stop+wait после каждого протокола (это уже внутри /unpack-product)

3. Опционально:
   - Если у клиента два core-предложения → /product-add для второго Core
   - Спроси: «У тебя один основной оффер или несколько? Если несколько — добавлю.»

4. Финал фазы 2:
   - Запиши в CHANGELOG.md
   - Сообщение: «Оффер готов. Core: [название]. Артефакты: core-offer.md, ladder.md.
     Переходим к Воронке?»
```

**Stop+wait → ждём согласия → Phase 3.**

### Phase 3 — Funnel (40-60 мин)

```
1. Запусти /unpack-funnel (Алекс ведёт):
   - Внутри /funnel-build для каждого ТОП-сегмента (по NORTH-STAR.md из фазы 1)
   - 4 шага: Channel-First → Route → Touchpoints → Conversions
   - Output: funnel/channels.md (Core Four + per-segment) + welcome.md
     + опционально scripts.md (если у клиента активирован модуль продающих встреч)

2. Stop+wait после каждого сегмента (внутри /unpack-funnel)

3. Финал фазы 3:
   - Bottleneck-анализ суммарно по всем сегментам (где узкое горло конверсии)
   - Запиши в CHANGELOG.md
   - Сообщение: «Воронка распакована. Bottleneck: [где именно — какой шаг
     теряет больше всего]. Артефакты: channels.md, welcome.md. Переходим к Бренду?»
```

**Stop+wait → ждём согласия → Phase 4.**

### Phase 4 — Brand (PLACEHOLDER, не автоматизировано)

Фаза не запускается автоматически. Сообщение владельцу:

> *«Фаза 4 — Бренд. Пока агента-Брендмейкера в команде нет, поэтому распаковываем не автоматом.*
>
> *Что делать вручную (или через Дизайнера если он установлен):*
> *— `brand/positioning.md` — Unique Mechanism + Competitive Alternatives. Если установлен Дизайнер, у него есть скилл `/positioning` (8 секторов) и `/brand-wheel` (7 секторов колесо бренда) — могу запустить его*
> *— `brand/competitors.md` — конкурентная карта. Алекс уже наполнил её из competitors-research в фазе 1*
> *— `brand/expert-bank.md` — голос владельца. Алекс уже наполнил из ответов в протоколах фазы 2 и из встреч (если активирован модуль)*
> *— `brand/voice.md` — если установлен Копирайтер, у него есть `/brand-voice`. Иначе оставим заготовку*
>
> *На v1 пайплайна — фаза 4 на ручное заполнение. Когда добавим Брендмейкера — мета-скилл подтянет автоматически. Хочешь сейчас запустить Дизайнера на позиционирование, или оставим бренд на потом?»*

**НЕ ждать согласия для перехода — это финал. Просто информируем.** Если владелец говорит *«запусти Дизайнера»* — handoff Дизайнеру через Director.

> *✅ Аудитория — ТОП-3 сегмента, dossier на каждый*

## Правила миграции от Стратега (закон)

**КРИТИЧНО.** Существующие файлы от Стратега — НЕ перезаписываем. Используем как стартовый input.

**Как:**
1. **Читаем** файл от Стратега в момент старта соответствующей фазы
2. **Помечаем** в `hypotheses.md` секции `🧊 Backlog` строкой формата:
   ```
   🌱 starter: [тема] из [файл-источник] (Стратег, YYYY-MM-DD)
   ```
3. **Используем** как стартовую гипотезу при запуске Алексовских скиллов
4. **Сохраняем оригинал** — файл Стратега остаётся в проекте как исторический исходник

**Записываем в `CHANGELOG.md`** проекта при каждой фазе:

```markdown
## YYYY-MM-DD — фаза N ([Audience/Product/Funnel]) распакована Алексом

Использовано: [diagnostic-card.md / черновики audience-* / product-*] от Стратега как starter.
Output: [список артефактов которые Алекс создал].
Stop+wait чек-поинт пройден: владелец одобрил [что именно].
```

**Что НЕ делать:**
- Перезаписывать `diagnostic-card.md`
- Удалять черновики Стратега из `audience/` / `product/` / `brand/`
- Игнорировать стартовые файлы — они должны быть прочитаны и стать input'ом

## Stop+wait протокол

**Между фазами — ОБЯЗАТЕЛЬНО:**
1. Алекс/мета-скилл объявляет: *«Фаза N завершена. Output: [артефакты]»*
2. Спрашивает: *«Продолжаем к фазе N+1? Или пауза?»*
3. **Без явного «да / продолжаем / поехали»** — НЕ ИДЁМ дальше

**Внутри фазы между скиллами** — тоже stop+wait. Это уже прописано в самих скиллах (`/segments-discover`, `/segments-unpack`, `/segments-awareness`, `/unpack-product`, `/unpack-funnel`) — мета-скилл их просто запускает по очереди и доверяет им.

**Если владелец говорит *«пауза» / «отложим»* посреди фазы:**
- Зафиксируй где остановились (последний пройденный шаг) в `progress.md` проекта
- Сообщи: *«Поймал. Остановились на [шаг]. Когда вернёшься — скажи «продолжаем распаковку» и подхвачу с этой точки.»*

## Output Contract

После полной распаковки в `projects/<main>/` появятся / будут обновлены:

**Аудитория:**
- `audience/segments/NORTH-STAR.md` — ТОП-3 сегмента
- `audience/segments/{slug}/dossier.md` × 3
- `audience/segments/{slug}/base.md` × 3
- `audience/voc.md` — голос ЦА (Voice of Customer, цитаты)

**Продукт:**
- `product/core-offer.md` — Grand Slam Offer
- `product/ladder.md` — лестница офферов
- `product/proof.md` — доказательная база (опционально)

**Воронка:**
- `funnel/channels.md` — Core Four + per-segment каналы
- `funnel/welcome.md` — последовательности касаний
- `funnel/scripts.md` — скрипты продающих встреч (опционально)

**Бренд (заполняется по ходу через Алекса):**
- `brand/expert-bank.md` — голос владельца (наполнен Алексом из протоколов и встреч)
- `brand/competitors.md` — конкурентная карта (от competitors-research в фазе 1)
- `brand/positioning.md` — placeholder, заполняется вручную или через Дизайнера
- `brand/voice.md` — placeholder, через Копирайтера если установлен

**Мета:**
- `hypotheses.md` — обновлённые секции Validated / Active / Backlog
- `metrics.md` — обновлённые цифры по сегментам и воронке
- `CHANGELOG.md` — история фаз с датами и output'ами
- `progress.md` — отметки прохождения фаз

## Guardrails

- **Не перезаписывай файлы Стратега.** Они — исторический исходник. Используй как input, оригинал не трогай.
- **Не пропускай stop+wait.** Между фазами без явного согласия — стоп. Это защита владельца от лавины артефактов которые он не успевает осмыслить.
- **Не запускай фазу 4 автоматически.** Брендмейкера нет — фаза 4 это информирование, не действие.
- **Не сам делай работу Алекса.** Этот скилл — оркестратор. Все маркетинговые скиллы (`/segments-*`, `/unpack-product`, `/unpack-funnel`) — у Алекса. Запускай их, не дублируй.
- **Идемпотентность.** Если фаза уже пройдена (есть артефакты + запись в CHANGELOG) — не запускай повторно без явного *«перезапусти фазу N»* от владельца.
- **Записывай в CHANGELOG.** Каждая завершённая фаза = строка в `projects/<main>/CHANGELOG.md`. Это аудит-трейл, без него невозможно понять что и когда было распаковано.
- **Не упоминай в разговоре технические детали.** Не говори клиенту «запускаю /segments-discover», «парсю diagnostic-card», «миграция от Стратега». Говори: «начинаем с аудитории», «беру наработки которые у тебя уже есть как стартовую точку».

## Язык клиенту — примеры

**Хорошо:**
- *«Поехали с аудитории. У тебя есть наработки от Стратега — возьму их как стартовую точку, не потеряю.»*

**Плохо (не говорить):**
- «Запускаю мета-скилл /unpack-project, фаза 1 — /segments-discover»
- «Мигрирую diagnostic-card.md как starter в hypotheses.md»
- «Stop+wait чек-поинт между фазами 2 и 3»
- «Output contract фазы 3 — funnel/channels.md и welcome.md»
