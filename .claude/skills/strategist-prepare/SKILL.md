# /strategist-prepare — подготовка офиса перед стратегией

После INTAKE + UNPACK + DISCOVERY у тебя много сырых данных. Прежде чем идти в стратегию и roadmap — **остановись и сверь что реально собрано**. Заполни центральную карточку пользователя, обнови офис, план сессии.

**Цель этой фазы:** офис "знает пользователя" — все агенты при следующем обращении получат контекст без повторных вопросов. Пользователь при этом чувствует: "меня распаковали, офис подстроен под меня".

## Когда запускается

- Автоматически после `/strategist-discovery` — проверка и подготовка
- Вручную: "проверь что собрали", "готов к стратегии?", "сверим данные"
- Перед `/strategist-roadmap` если давно не запускался `prepare`

## Пререквизиты
- Прошли INTAKE + UNPACK + DISCOVERY (есть `diagnostic-card.md`, черновики `audience/`, `products/`, `brand/`, `goals-extracted.md`)
- Если чего-то нет — вернуть пользователя на соответствующую фазу

## Шаг 1. Аудит полноты (внутренняя работа, пользователь не видит)

Пройдись по чеклисту. Для каждого пункта — DONE / PARTIAL / MISSING:

```
□ projects/<main>/CLAUDE.md — базовое описание проекта
□ projects/<main>/diagnostic-card.md — заполнен на 80%+, слабая сторона классифицирована
□ projects/<main>/audience/ — минимум 1 сегмент с болями
□ projects/<main>/products/core-offer-draft.md или core-offer.md
□ projects/<main>/brand/voiceprint-v0.md — 5-10 фраз пользователя
□ projects/<main>/goals-extracted.md — цели с цифрами
□ inbox/prework.md — есть ответы на 15 тем
□ inbox/docs/ — документы пользователя (если есть)
```

## Шаг 2. Если есть дыры — добери прямо сейчас

**Если критичный пункт MISSING** (нельзя в стратегию без него):
- Слабая сторона не классифицирована → задай 1-2 быстрых вопроса, классифицируй
- Goals без цифр → один вопрос "давай одну цифру для цели — выручка / пользователи / конверсия?"
- Ресурсы/границы пустые → один вопрос "сколько часов в неделю реально готов вкладываться?"

**НЕ допрашивай.** 2-3 точечных вопроса максимум. Если пользователь устал — принимай как есть, в стратегии пометь `[НУЖНО УТОЧНИТЬ]`.

## Шаг 3. Создай центральную карточку пользователя

Путь: `office/client-profile.md`

Шаблон: `office/agents/strategist/templates/client-profile.template.md`

Заполни плейсхолдеры из собранной информации:
- `{{CLIENT_NAME}}`, `{{NICHE}}`, `{{ROLE}}`, `{{CORE_OFFER}}`
- `{{WEAKNESS_PRIMARY}}`, `{{WEAKNESS_SECONDARY}}`
- `{{MAIN_BLOCKER}}`
- `{{GOAL_PROGRAM}}` — цель на 6 недель
- `{{HOURS_PER_WEEK}}`, `{{BUDGET}}`, `{{TEAM}}`, `{{NO_GO}}`
- `{{VOICE_PATTERNS}}` — из voiceprint-v0
- `{{AUDIENCE_WHO}}`, `{{AUDIENCE_PAIN}}`
- `{{AGENT_HINTS}}` — критично: что полезно знать агентам про этого пользователя

**Это центральный документ** — все будущие агенты при старте читают его первым. Он даёт контекст за 30 секунд.

## Шаг 4. Обнови office/AGENTS.md

Добавь короткий блок в начало `office/AGENTS.md` (после заголовка):

```markdown
> **Контекст пользователя:** {{CLIENT_NAME}}, {{NICHE}}.
> Полная карточка — `office/client-profile.md`.
> Главный фокус: {{GOAL_PROGRAM}}. Слабая сторона: {{WEAKNESS_PRIMARY}}.
```

Чтобы Director и любой агент при открытии AGENTS.md сразу видели контекст.

## Шаг 5. Создай dummy office/strategy/ файлы

Если `office/strategy/strategy.md` не существует — создай placeholder:
```markdown
# Стратегия — собирается в /strategist-roadmap

Этот файл будет заполнен на следующем шаге. Сейчас Director может работать без него — `client-profile.md` даёт базовый контекст.
```

Аналогично для `office/strategy/progress-index.md` если его нет.

**Цель:** Director больше не спотыкается о пустую папку при старте сессии.

## Шаг 6. Составь план сессии стратегии

Внутренний документ для себя — `office/strategy/session-plan.md`:

```markdown
# План сессии стратегии — {{DATE}}

## Что разбираем в /strategist-roadmap

1. **Слабая сторона** — {{WEAKNESS_PRIMARY}}. Есть ли согласие пользователя?
2. **Главный блокер** — {{MAIN_BLOCKER}}. Готов к этому выводу?
3. **3-4 рычага** — примерные направления для обсуждения: [...]
4. **Ресурсы** — {{HOURS_PER_WEEK}}ч/нед, {{BUDGET}} бюджет. Реалистичны ли планы на 6 недель?

## Что нужно УТОЧНИТЬ во время сессии
- [...]

## Риски сессии
- Пользователь устал после intake — не перегружать
- Противоречия в данных — разобрать первым
```

## Шаг 7. Живой отчёт пользователю

Покажи пользователю что офис подготовлен. Живо, без технической занудности:

> *(перефразируй под пользователя, не копируй)* короткая мысль типа:
> - "Окей, я разобрал всё что ты сказал. Вижу X, Y, Z. Главная нагрузка — здесь. Ресурсы — вот такие."
> - "Профиль твой собран, офис настроен. Все помощники теперь про тебя знают."
> - "Готов идти в стратегию? Соберу план на 6 недель с учётом того что увидел."

Пользователь говорит "готов" / "давай" — запускаешь `/strategist-roadmap`.
Пользователь говорит "подожди, у меня вопрос" — отвечаешь, уточняешь, потом roadmap.

## Fallback

- **Критичных дыр нельзя закрыть** (пользователь не отвечает / устал) — запиши `[WAITING]` в `client-profile.md` по нужным полям, пометь `strategist/memory.md` "вернуться к X в check-in", иди в roadmap с тем что есть.
- **client-profile уже существует и заполнен** (повторный запуск) — обновляй, не перезаписывай с нуля. Append новые находки.
- **AGENTS.md тронут пользователем** — НЕ перезаписывай его личные секции, только блок "Контекст пользователя" наверху.

## Память (обязательно)

**После prepare:** в `office/agents/strategist/memory.md` секция "Decisions" — что решил при заполнении профиля, какие гипотезы остались. В "Patterns" — что заметил в пользователе как паттерн (реакции, язык, приоритеты).

## Выход

- `office/client-profile.md` — заполнен, готов к чтению всеми агентами
- `office/AGENTS.md` — обновлён блок "Контекст пользователя"
- `office/strategy/strategy.md` — placeholder или обновлён
- `office/strategy/session-plan.md` — план сессии roadmap
- Короткий отчёт пользователю + предложение запустить `/strategist-roadmap`
