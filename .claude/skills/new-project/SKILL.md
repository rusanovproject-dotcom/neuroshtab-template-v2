# New Project — создание нового проекта

Твоя задача — за 2 минуты превратить идею в правильно оформленную папку проекта, готовую к работе.

## Шаг 1. Собери бриф — живым диалогом

Спрашивай по одному, не кидай список. Если пользователь сразу дал часть данных в триггере ("новый проект лендинг для Ивана") — используй, не переспрашивай очевидное.

1. **Slug папки.** "Как назовём папку? Короткое латинское имя через дефис — например `landing-ivan`, `b2b-outreach`, `course-launch`."
2. **Цель.** "Зачем этот проект — одной фразой?"
3. **ЦА.** "Кто целевая аудитория? Можно коротко — пол, возраст, профессия, боль. Или "не знаю" если не думал."
4. **Метрики успеха.** "Как поймёшь что получилось? 1-3 метрики (например: 50 лидов, CTR 3%, запуск до 15 мая)."
5. **Дедлайн.** "Дедлайн есть? Если да — когда."
6. **Агенты.** "Какие агенты задействованы? Если не знаешь — предложи позвать Director, он разберётся. По умолчанию ставь Director как оркестратора."

## Шаг 2. Создай структуру папки

```
projects/<slug>/
├── CLAUDE.md              (контекст для Claude, по шаблону)
├── README.md              (для человека — что это за проект)
├── brief.md               (исходный бриф — сырые ответы пользователя)
├── audience/
│   └── persona.md         (только если пользователь дал ответ на ЦА не "не знаю")
├── knowledge/
│   └── INDEX.md           (пустой индекс)
├── artifacts/             (пустая папка)
├── sessions/              (пустая папка)
└── status.md              (roadmap, milestones, blockers)
```

Создавай через Write tool — каждый файл с содержимым.

## Шаг 3. Заполни CLAUDE.md из шаблона

Возьми `office/templates/project-brief.md`, подставь ответы пользователя в плейсхолдеры, сохрани как `projects/<slug>/CLAUDE.md`.

Плейсхолдеры шаблона:
- `{{PROJECT_NAME}}` — человекочитаемое имя (из slug, но красиво)
- `{{GOAL}}` — цель одной фразой
- `{{AUDIENCE}}` — ЦА (или "не определено")
- `{{METRICS}}` — метрики списком
- `{{DEADLINE}}` — дедлайн или "без дедлайна"
- `{{AGENTS}}` — задействованные агенты
- `{{DATE}}` — сегодняшняя дата в формате YYYY-MM-DD

## Шаг 4. Заполни остальные файлы

**README.md** — короткое описание для человека:
```markdown
# <PROJECT_NAME>

<GOAL в одну строку>

Контекст для Claude — `CLAUDE.md`.
Бриф исходный — `brief.md`.
Статус и milestones — `status.md`.
```

**brief.md** — сырые ответы пользователя, как есть. Каждый вопрос → ответ.

**audience/persona.md** — только если ЦА не "не знаю". Короткий профиль: кто, боль, чего хочет.

**status.md** — базовый скелет:
```markdown
# Статус проекта

## Roadmap
- [ ] <milestone 1>
- [ ] <milestone 2>

## Сейчас в работе
<пусто>

## Blockers
<пусто>

## Решения
<пусто>

Последнее обновление: <дата>
```

## Шаг 5. Покажи результат

Выведи структуру и путь:

> Готово. Проект `<slug>` создан → `projects/<slug>/`
>
> Структура:
> - `CLAUDE.md` — контекст для работы
> - `brief.md` — твои ответы
> - `status.md` — трек прогресса
> - `knowledge/`, `artifacts/`, `sessions/` — пустые, наполнятся по ходу
>
> Дальше — открой Director и дай первую задачу по проекту.

## Правила

- em-dash `—`, не `--`.
- Если slug конфликтует с существующей папкой — спроси: переименовать или дополнять существующий.
- Не создавай папки которые пустые и не нужны (например `audience/` если ЦА не задана).
- Тон — напарник, а не бюрократ.
