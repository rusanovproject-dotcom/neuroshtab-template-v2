# `/marketer-funnel` — Воронка под лестницу

Методология — `knowledge/classics-compact.md` (Kennedy Magnificent 7, Goldratt TOC) + `knowledge/hormozi-offer-market.md` (Core Four каналы) + `knowledge/audience-framework.md` (типы возражений).

## Пререквизит
`projects/<main>/product-ladder.md` + `audience.md` + `customers.md`. Если чего-то нет — запусти предыдущий скилл.

## Шаги

### 1. Контекст
- `product-ladder.md` — что продаём
- `audience.md` — возражения по 3 типам + 4 уровня осознанности
- `customers.md` — реальная воронка (как приходили покупатели)
- `hypotheses.md` — активные `[CHANNEL]`, `[LM]`, `[FUNNEL]` гипотезы

### 2. Channel-First анализ (Хормози Core Four)
Не выдумывай каналы — смотри **что уже работает у клиента**:
- Warm Outreach (1-к-1 с теми кто знает)
- Content (1-ко-многим с теми кто знает)
- Cold Outreach (1-к-1 холодные)
- Paid Ads (1-ко-многим холодные)

**80% малых бизнесов стартуют с warm + content, не с paid.** Смотри customers.md — откуда реально приходили.

### 3. Если воронка УЖЕ работает — диагностика bottleneck (Goldratt)
Собери конверсию по этапам в цифрах:
```
Трафик → Лид → Диагностика → КП → Оплата
```
Сравни с бенчмарками. Самое узкое место = bottleneck. **Чини только его**, остальное не трогай.

### 4. Welcome-цепочка (3-7 касаний)
Под Awareness Level входящего трафика (из audience.md уровни Schwartz):
- Холодный (1-2 уровень) → 5-7 касаний, история+боль+кейс
- Тёплый (3 уровень) → 3-5 касаний, сравнение+гарантия
- Горячий (4-5 уровень) → 1-2 касания, оффер+цена+CTA

Каждое касание привязано к конкретному тезису из `audience.md`.

### 5. Дожим по 4 типам возражений 
- Ценовые — через Value Stack
- Ценностные — через уникальный механизм
- Доверительные — через кейсы и соцпруф
- Страхи «Я как потребитель» — через Quick Win за 48 часов

Каждому типу — минимум одно касание в цепочке.

### 6. Magnificent 7 проверка (Kennedy)
Каждая коммуникация воронки (пост / лендинг / email / реклама) должна иметь ≥ 5 из 7:
Headline + Reason Why + Offer + Guarantee + Scarcity + CTA + Deadline.

### 7. Запиши в `projects/<main>/funnel.md`
Карта воронки + метрики каждого этапа + welcome-цепочка + дожим по типам возражений + bottleneck (если диагностировался).

### 8. Поставь H-гипотезы в `hypotheses.md`
Минимум 2 новые с тегами `[FUNNEL]`, `[CHANNEL]`, `[LM]`.

## Quality Gate
- [ ] Каналы Channel-First (из реальных данных customers, не из головы)
- [ ] Bottleneck найден (если воронка работает)
- [ ] Welcome-цепочка привязана к Awareness Level
- [ ] Все 4 типа возражений закрыты касаниями
- [ ] Каждое касание прошло Magnificent 7 (≥5 из 7)
- [ ] 2+ новые гипотезы в hypotheses.md

## Выход
Карта воронки + топ-2 bottleneck + *«Воронка собрана. Следующий шаг — тестирование гипотез или углубление одного из этапов?»*
